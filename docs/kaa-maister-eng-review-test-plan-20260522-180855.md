## Affected Pages/Routes

API routes:

- `POST /api/projects` — register a project: validate `maister.yaml` v1 (schemaVersion, project block, flows[] uniqueness, placeholder check), persist row, derive slug from `project.name`.
- `DELETE /api/projects/[slug]` — soft-archive (sets `archived_at`); no hard delete in POC.
- `POST /api/projects/[slug]/tasks` — create task in `Backlog` for the given project; body: title, prompt, flowId.
- `POST /api/runs` — start a run for a task: precondition checks, `git worktree add` under the project's branch_prefix, subprocess spawn; rejects if global `MAISTER_MAX_CONCURRENT_RUNS` cap reached (run → `Pending`, position badge surfaced).
- `GET /api/runs/[id]/stream` — SSE live log stream; `lastEventId` reconnect; tails `.maister/<project-slug>/runs/<run-id>/<block-id>.log`.
- `POST /api/runs/[id]/hitl-response` — accept user response to HITL request, atomic write under the project-scoped subtree, re-invoke Flow with `--resume <block-id>`.
- `GET /api/runs/[id]/diff` — return raw `git diff` of the worktree.
- `POST /api/runs/[id]/merge` — `git merge --no-ff` worktree branch into parent's `main_branch`; abort on conflict; on success, task → `Done`.
- `POST /api/runs/[id]/abandon` — SIGTERM if a block is alive; cleanup worktree; task → `Abandoned`.
- `GET /api/cron/gc` — GC Abandoned/Done worktrees older than 7d across all projects.

UI routes:

- `/` — **Portfolio home** (superset.sh-style grid of active workspaces across all projects). Card = project · branch · status · last activity · quick actions. Filters by project + status.
- `/projects` — list of registered projects; "Add project" button.
- `/projects/new` — form: paste path to dir containing `maister.yaml`.
- `/projects/[slug]` — **per-project board**: 2 columns `Backlog | In Flight`. Done/Abandoned/Crashed in filter tab.
- `/projects/[slug]/tasks/new` — task creation: title + prompt + Flow dropdown (from `flows[]`).
- `/runs/[id]` — run detail: status, live logs, HITL surface, diff view, action buttons (mark ready, merge, abandon). Worktree context shown (project + branch + worktree path).

## Key Interactions to Verify

1. **Register project** via `/projects/new`: paste path to dir with valid `maister.yaml` v1 → project appears in `/projects` list and as a tab/switcher in nav. Invalid yaml → form-level error with line/column.
2. **Create task → see in Backlog column** on `/projects/[slug]`. Persists across page reload. Flow dropdown lists project's `flows[]` only.
3. **Click Launch on a Backlog task** on `/projects/[slug]`: precondition checks pass → workspace created → run spawned → task moves to In Flight column → run-detail link visible. (No drag-and-drop in POC; the button is the only entry point.)
4. **Portfolio home** on `/`: card appears for the new workspace within 1s of run start. Card shows project + branch + status + last-activity. Filtering by project + status works.
5. **Start run → live logs stream** on `/runs/[id]`. Lines appear within 500ms of subprocess emit. Reconnect on tab refresh keeps stream. Log file path is project-scoped: `.maister/<project-slug>/runs/<run-id>/<block-id>.log`.
6. **HITL roundtrip (CRITICAL)** on `/runs/[id]`: Flow writes request file under project subtree → UI surfaces request panel → user submits response → response file appears → Flow resumes → status returns to `Running`.
7. **Diff view** on `/runs/[id]`: `git diff` raw output rendered in `<pre>`. Updates after subprocess writes files.
8. **Merge to main** on `/runs/[id]`: button triggers `git merge --no-ff` on parent repo; success → run + task → `Done`, card disappears from portfolio; conflict → run → `Review`, UI shows "Conflict — resolve manually" with parent repo path.
9. **Abandon mid-run** on `/runs/[id]`: SIGTERM subprocess, worktree marked stale, **run** → `Abandoned`, **task → `Backlog`** (retry-eligible; Launch button reappears on the card). Only explicit "Discard task" sets task → `Abandoned`.
10. **Concurrency cap (3 by default)**: launch 4 runs across 2 projects → 3 → `Running`, 4th → `Pending` with "Position 1/1" badge. On 1st completion, 4th auto-promotes to `Running`.
11. **Retry loop (1:N task→run)**: Launch a task → run #1 → abandon → task → `Backlog` → Launch again → run #2 spawns with `attempt_number=2` and a fresh worktree against the same task. Run-list on the task card shows both attempts.

## Edge Cases

1. **Precondition: dirty parent repo** — `POST /api/runs` returns 400 with clear error; UI surfaces "Commit or stash changes in parent repo first".
2. **Precondition: branch already exists** — same flow; error "Branch `maister/<slug>` already exists. Pick another name."
3. **Precondition: worktree path collision** — same; error "Worktree path `/repos/.../wt-<id>` already exists. Delete and retry."
4. **Concurrent run capped (global)** — launch 4 runs across projects with `MAISTER_MAX_CONCURRENT_RUNS=3` → 4th → `Pending` with position badge; auto-promotes on first slot free. Per-project caps are NOT enforced (global only).
5. **HITL timeout 24h** — Flow asks, user doesn't answer for 24h → run → `Abandoned`, task → `Abandoned`, worktree marked stale.
6. **Next.js process crash mid-run** — kill server; restart; reconcile (per project): `Running` runs with no live PID → `Crashed` status; UI prompts "Recover or discard" on the relevant project board AND portfolio card.
7. **Subprocess spawn failure** — `uv` missing or Flow framework not installed → `MaisterError({code: 'SPAWN'})`, clear UI message with install hint.
8. **API key missing/invalid** — Claude Code 401 surfaced in logs + status → `Failed`, UI hints "Check `.env`".
9. **Merge conflict** — `git merge --no-ff` fails → no merge, run stays in `Review` state with conflict file list visible.
10. **Empty Flow output** — Flow exits 0 with no diff → run + task → `Done`, diff view shows "No changes" message.
11. **Subprocess produces >10MB stdout** — does SSE keep up? Does server OOM? Verify pipe-to-disk works (project-scoped log path).
12. **`maister.yaml` invalid YAML** — registration error with line/column from YAML parser; project not persisted.
13. **`maister.yaml` schemaVersion mismatch** — error "Config version N, expected 1. Migrate or downgrade."
14. **`maister.yaml` duplicate `flows[].id`** — registration error; project not persisted.
15. **`maister.yaml` Flow command missing `{prompt}` or `{workspace_path}` placeholder** — registration error; project not persisted.
16. **Slug collision on register** — second registration with the same derived slug (e.g. `MyApp` + `my-app` both → `my-app`) → `MaisterError({code: 'CONFLICT'})` with message identifying the existing project. Existing row is **not** touched.
17. **`repo_path` collision on register** — second registration whose `project.repo_path` equals an existing project's `repo_path` → `MaisterError({code: 'CONFLICT'})`. One repo = one project.
18. **`MAISTER_PROJECTS_DIR` recursive auto-discovery** — on startup, the env-pointed directory tree is walked **recursively**; every `maister.yaml` found is registered (slug + repo_path uniqueness enforced — duplicates are skipped with a logged warning, not a hard crash).
19. **Project archive (DB only — no UI)** — `DELETE /api/projects/[slug]` sets `archived_at`; archived projects disappear from nav, portfolio, and `/projects` list; existing runs stay queryable by direct URL. Their `repo_path` is **not** freed for re-registration in POC (avoid silent re-use).

## Critical Paths

1. **End-to-end Flow execution (single project)**: register project → create task on board → click Launch → worktree created → subprocess runs → logs streamed → no HITL → diff visible → merge to main → run + task → `Done`. Single test covering happy path.
2. **HITL roundtrip**: click Launch on a Backlog task → Flow writes HITL request under project subtree → UI panel appears (project board + run detail) → user submits response → response file written atomically → Flow consumes → continues to completion.
3. **Recovery after crash**: start run → kill Next.js mid-run → restart → per-project reconcile flags run as `Crashed` → user discards → worktree cleaned up → can start fresh run.
4. **Abandon and cleanup**: start run → abandon mid-execution → SIGTERM successful → worktree GC'd within 7d → no orphan files in `.maister/<project-slug>/runs/`.
5. **Multi-project portfolio**: register 2 projects with ≥2 Flows each → launch 1 run per project → portfolio home shows both cards with correct project/branch/status → 3rd run on either project starts immediately (cap=3) → 4th queues with position badge.

## Test Framework

No tests exist yet (greenfield). Recommended stack:
- **Unit/integration**: `vitest` (Next.js-friendly, fast)
- **E2E**: `playwright` (auto-includes for Next.js, runs against `next dev` or `next start`)
- **Eval**: ad-hoc script that runs aif with fixed prompt, hashes resulting diff, compares against baseline

Test plan does not include implementation — see Implementation Tasks file for ordering.

## Notes

- POC will likely ship without 100% coverage. Critical paths (#1, #2) MUST have tests before dogfood begins. Edge cases (#1-#14) can be added incrementally.
- aif eval is the only LLM-related test; user can defer if aif determinism is already validated externally.
- Performance tests (#11 — large stdout) deferred unless friend reports OOM.
