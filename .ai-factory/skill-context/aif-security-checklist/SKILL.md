### Path-traversal via validator-bypass — Zod `z.string()` flowing into `path.*` is CRITICAL
**Source**: 2026-05-26-12.45.md, 2026-05-26-12.53.md
**Rule**: During injection audits, add a dedicated step: trace every `path.resolve` / `path.join` / `fs.*` / `child_process.spawn` argv (including `cwd`) back to its Zod (or other boundary) schema. If the field that produced the value is `z.string()`, `z.string().min(N)`, or any other unconstrained string without a regex constraint (`^[A-Za-z0-9._-]+$` for filesystem path segments) or a path-shape `.refine()` (`startsWith("/")` + no `..` for absolute paths), record the finding as CRITICAL `#path-traversal`. Severity remains CRITICAL even when the affected service is documented as internal-network only (no auth, internal topology) — internal services routinely gain auth later or get exposed by mistake, and the defense costs ~5 minutes per field. When one field in a schema gets a regex, audit all siblings — partial-coverage waves are the common failure mode.

### SSRF / egress trust — a member-writable destination + a viewer-readable response is a cloud-metadata read primitive
**Source**: 2026-06-11-11.54.md
**Rule**: Audit every outbound HTTP feature (webhooks, pings, subscriptions, agent fetch). Evaluate the trust claim against the **LOWEST role that can write the destination URL** PLUS **every read path of the response** — "operator-trusted" only holds if operators are the only writers AND readers. A member-created subscription/ping to `http://169.254.169.254/latest/meta-data/…` whose response snippet is persisted and viewer-readable = a cloud-IMDS credential read (SSRF → credential exfil). Required defense, all four layers:
1. A `node:net` `BlockList` over loopback / private / link-local (incl. the `169.254.169.254` metadata range) / multicast / unspecified — and the v4-mapped-v6 forms of each.
2. **Write-time** IP-literal rejection in `assertUrl` (reject when the host is a literal in a blocked range).
3. **Send-time** `dns.lookup` of ALL records with refuse-if-ANY-blocked → nothing is sent.
4. A pinned `undici` dispatcher (`Agent` answering only from the vetted addresses) to close the DNS-rebind TOCTOU between resolve and connect.
Flag `://user:pass@host` userinfo in any body-supplied URL as a secret-at-rest decision (store / redact / reject — keep the "no secrets at rest" spec claim consistent).

### Authored-package path-safety — reject `..` on original bytes, decode fatally, refuse symlinks, validate every external id before a sink
**Source**: 2026-06-08-12.02.md, 2026-06-08-12.15.md, 2026-06-08-13.05.md, 2026-06-08-14.41.md, 2026-06-01-15.31.md
**Rule**: For any code that imports/installs/exports a package, archive, or upload:
- Reject `..` in the **ORIGINAL** slash/backslash path segments BEFORE normalization — `schemas/../setup.sh` is accepted if you check the normalized result (`setup.sh`). Test both normalized and original forms, and both POSIX `../` and Windows `..\` separators regardless of host OS.
- Read package bytes as bytes, THEN decode with **fatal UTF-8** validation — `readFile(path, "utf8")` silently replaces invalid bytes before validation can reject a binary payload.
- Reject non-regular entries (symlinks): the generic installer copies the full tree, so a symlinked `setup.sh` enters the cache as untrusted bytes and later becomes executable. Log the refused path.
- Reject ancestor/descendant (file-vs-directory) path collisions, not just exact duplicates — they fail at export time.
- Validate EVERY external identifier (package slug, filename, `flow.yaml.name`) server-side before it reaches DB identity, YAML, SQL, or a filesystem sink; never reuse one manifest field as both display label and sink-safe slug.
- Hidden form fields (`capabilitySlug`) and body fields are locators / concurrency tokens ONLY, never authority for resource identity — load the server-owned value after authorization.

### Fail-open at the trust boundary + authz scope must match write scope
**Source**: 2026-06-04-17.50.md, 2026-06-12-12.56.md
**Rule**: Validate the **wire**, not the schema — a zod range (`z.number().min(0).max(1)`) on the producer side does NOT hold for an untrusted producer (agent stdout `{"confidence":2}`, `Infinity`). Re-check the domain at every untrusted sink. **Out-of-range ≠ missing**: a present-but-invalid value must fail-closed on its own branch and must never fall through to a "missing / allow-default" branch (a fail-OPEN promotion gate). Every `typeof x === "number"` before a comparison that gates promotion needs `Number.isFinite(x)` + the domain bound in the SAME expression. Separately: an endpoint's authz scope MUST match the **write scope** of what it mutates — a project-URL route (`POST .../packages/{attachmentId}/trust`) gated on project-scoped `managePackages` but whose handler fans `setup.sh` trust across EVERY project on a shared platform `package_installs` row is cross-project arbitrary-code-execution; it needs `requireGlobalRole("admin")`. If a project-URL route updates platform-scoped rows (no `projectId` in the WHERE), red-flag it.
