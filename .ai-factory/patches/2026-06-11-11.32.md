# Codex adversarial-review findings on the model-discovery branch (5 fixes)

**Date:** 2026-06-11 11:32
**Files:** supervisor/src/acp-client.ts, supervisor/src/model-catalog/{sources/acp-probe.ts,harvest.ts,resolve.ts,cache.ts}, supervisor/test/fixtures/mock-acp-models.mjs, web/lib/flows/graph/runner-graph.ts, docs/decisions.md (+24 ADR-renumber files), and the matching tests
**Severity:** high

## Problem

A Codex adversarial review of `feature/acp-runner-model-discovery` (vs main) surfaced five issues, all re-verified against the code before fixing:

1. **Claude wrong-model execution.** A capability-less `ai_coding`/`judge` node never wrote `settings.local.json`, so its configured model was never applied (claude pins ONLY via that file).
2. **Codex apply-gap.** `applyAndVerifyModel` returned early when the adapter omitted `currentModelId`, silently skipping `setSessionModel` for codex.
3. **Probe process leak.** The ACP model-probe sent SIGTERM with no wait/escalation — a SIGTERM-ignoring adapter could orphan.
4. **Catalog cache downgrade.** Passive harvest `cache.set`-replaced the resolved catalog with an `agent_observed`-only subset for the full 1h TTL.
5. **ADR collision.** The branch reused `ADR-073`, which main already owns (different decision) — a guaranteed merge collision across 39 refs.

## Root Cause

- **#1 — coupling.** The model-pin write was a side effect of *capability* materialization (`materializeNodeCapabilities` early-returned on `!declares`). Two orthogonal concerns (run-model vs declared capabilities) shared one gate, so the common "bare node" case silently lost the model. Scratch was safe only because it happened to materialize unconditionally.
- **#2 — a deliberate-but-wrong guard.** `!observed → return` was intentional AND unit-tested, but conflated claude's *verify-only* channel with codex's *active-apply* channel. A test encoding the buggy behavior made it look correct.
- **#3 — "send signal" mistaken for "ensure exit."** `child.kill('SIGTERM')` returns immediately; the code's own comment admitted the promise "keeps running," and the test only asserted `kill` was *called*.
- **#4 — replace where merge was meant.** Harvest's observed list is a SUBSET of the merged resolve; a replace shrank it. The route serves cache-first, so the shrink persisted for the whole TTL.
- **#5 — fork-time number allocation.** The branch forked before main's ADR-072/073/074 landed, so its "next free" 073 was already taken by merge time.

## Solution

- **#1** Made claude model materialization unconditional: `materializeNodeCapabilities` now falls through to `materializeCapabilityProfile` with an **explicit-empty** profile (`selectedMcpIds: []` — NOT undefined, which pulls default MCPs) when `agent==='claude' && executor.model`. Reused the materializer's settings.local.json backup/owned-marker/reclaim path (never hand-write it → `#data-loss`). Codex bare nodes still skip (they pin supervisor-side).
- **#2** Split the guard by channel: codex calls `setSessionModel` even on absent/empty `observed`; claude stays verify-only (no advisory we cannot substantiate). Kept the documented no-readback design (env-router slot-remap is legitimate; `cost.jsonl` is billed truth).
- **#3** SIGTERM → bounded wait → SIGKILL → destroy stdio; a `teardownGraceMs` option + an `ignore-sigterm` fixture mode prove the child actually exits.
- **#4** Added `cache.setMerged` that PRESERVES the live entry's `insertedAt` (enrich without extending a stale row's TTL); harvest unions via the now-exported `mergeModels` (DRY) and de-dupes the `agent_observed` source row.
- **#5** Renumbered `ADR-073 → ADR-075` (two case-sensitive passes: `ADR-073` and the lowercase `adr-073` anchor slugs) across 25 files + added the index row; `validate:docs` green.

## Prevention

- **Decouple run-config from feature-config.** If exactly one launch path must apply a setting on EVERY launch, don't gate it behind an unrelated feature flag/declaration. Audit all launch paths (legacy + graph + scratch + resume/recover), not just the one in front of you.
- **A test that pins buggy behavior is not coverage.** When a finding says "X is wrong," check whether the existing test *asserts* X — flipping the assertion is part of the fix.
- **"Killed" ≠ "exited."** Teardown of a spawned child must wait-for-exit and escalate to SIGKILL; assert the process is gone, not that a signal was sent. (Repo already had this pattern in `ccr-manager` — reuse it.)
- **Cache writes from a partial observer must merge, never replace** — and decide TTL semantics explicitly (preserve `insertedAt` so enrichment can't resurrect stale rows).
- **Allocate ADR numbers against the merge target's HEAD, not the fork point.** Re-check `git show main:docs/decisions.md` for the true next-free immediately before merge. The lowercase anchor slug needs its own renumber pass.
- **Pre-existing, out-of-scope (flagged, not fixed):** `recover` (Crashed→resume) reuses the snapshot `adapterLaunch` and does NOT re-materialize, so a capability cleanup sweep before recover drops ALL capabilities (model+MCPs+skills) — broader than this fix.
