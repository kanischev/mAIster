# Flow Studio node-form row controls persisted empty list entries → invalid manifest

**Date:** 2026-06-26 12.17
**Files:** web/lib/flows/editor/manifest-normalize.ts (new), web/lib/flows/editor/__tests__/manifest-normalize.test.ts (new), web/components/flows/flow-editor-tabs.tsx, web/lib/flows/__tests__/flow-dsl-grammar.test.ts, web/lib/capabilities/package-catalog.ts, web/lib/flows/editor/reference-sources.ts
**Severity:** medium

## Problem

The Flow Studio node editor replaced comma-separated text fields with structured
row controls (`StringListField`) for `restrictions`, `roles`, `assignees`,
`decisions`, `material_axes`, `rework.allowedTargets`, and
`hooks.pathGuard.allowedPaths`. `StringListField`'s "+ Add" appends a blank row
(`onChange([...values, ""])`), and its DOM test asserts exactly that. The
`node-side-form` handlers only guarded `next.length` (not the contents), and
nothing downstream filtered: `stringifyYaml(manifest)`, `saveAction`,
`working-dir-save`, and the server `PUT /files/[...path]` route all skip
`flowYamlV1Schema` validation. A blank/whitespace row was therefore written
verbatim into `flow.yaml` and failed `z.array(z.string().min(1))` at
compile/launch with a `CONFIG` error.

## Root Cause

The legacy comma inputs filtered empties for free as a side effect of their
shape: `value.split(",").map((s) => s.trim()).filter(Boolean)` reparsed the whole
field on every change, so `""` segments never survived. The new row controls hold
each entry as a discrete array element, so the same "filter empties" guarantee had
to be re-established explicitly — and it wasn't. The contributor reasonably tried
the obvious spot (filter inside the field's `onChange`), but that fires
per-keystroke and would erase the very blank row the author just added, so the
filtering belongs at the serialize boundary, not the control. The gap was invisible
because the unit tests explicitly defer onChange/serialize wiring to e2e, and e2e
doesn't exercise "leave a blank row and save."

## Solution

Added `pruneEmptyListEntries(manifest)` (`manifest-normalize.ts`): a pure
recursive transform that, for every string array in the manifest, trims kept
values and drops empty/whitespace-only entries, omitting the array if it becomes
empty (matching the legacy `filter(Boolean)` + sparse-omit semantics). Applied it
at the single canvas→YAML boundary in `flow-editor-tabs.handleCanvasChange`,
pruning the serialized YAML *and* `canvasManifestRef`/`liveManifest` together so
the `parse(yaml) === canvasManifestRef` invariant holds (otherwise the yaml→canvas
reseed diff would spuriously remount the canvas — a known trap in that file). The
live canvas keeps the blank row via the graph editor's own state, so editing UX is
unchanged; only the persisted/validated manifest is cleaned. A generic string-array
prune (not a 7-field enumeration) was chosen because every list field in the DSL is
`z.array(z.string().min(1))`, so an empty element is invalid everywhere — this is
the "sweep the whole class, not the cited line" rule.

Also hardened the grammar drift-guard test to derive node `settings` keys from the
node union dynamically (so a future node type can't slip past it) with a
vacuous-derivation guard, and de-duplicated the `^skills/<slug>/SKILL.md$` regex
into one exported `SKILL_PATH_RE`.

## Prevention

- When replacing an input that did implicit normalization (split/trim/filter,
  coercion, dedupe) with a different control, re-establish that normalization
  explicitly — diff the OLD field's onChange against the NEW one for dropped
  behavior, don't just match the rendered output.
- Normalize collection inputs at the **serialize boundary**, never per-keystroke
  (per-keystroke filtering erases transient in-progress rows).
- A control whose own test asserts it emits `[""]` is a flag: prove the emitted
  manifest still parses under the authoritative schema in a test, not just that
  testids render. Acceptance criteria like "emitted manifest parses under
  `flowYamlV1Schema`" must have a test that actually parses a realistic edited
  manifest — the render-only node-form smoke can't catch serialize-shape bugs.
- For any value that flows into a `z.*.min(1)` array, empty elements are invalid
  everywhere it occurs — fix the class generically rather than the one field cited.
