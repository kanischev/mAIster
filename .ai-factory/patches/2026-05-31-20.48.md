# aif-fix patch: scratch run service and contract cleanup

## Context

Task 11 stayed logically incomplete even after the scratch run feature existed:
route handlers still owned launch/message state transitions, prompt completion
overwrote event-derived states, API docs drifted from actual response shapes,
and supervisor lifecycle integration still posted the removed `prompt` field to
`POST /sessions`.

## Applied fix pattern

- Move scratch launch and message orchestration into `web/lib/scratch-runs/service.ts`.
- Keep route handlers to request parsing, auth session checks, service calls,
  and HTTP error mapping.
- Complete prompt turns by transitioning only `Starting`/`Running` to
  `WaitingForUser`; preserve `NeedsInput`, `Crashed`, and other states already
  projected from supervisor events.
- Recheck and lock run rows on PostgreSQL before appending a user message.
- Keep project active workspaces visible on the project page, including scratch
  dialog links and recovery/discard action labels.
- Keep docs truthful about V1 capability materialization: current runtime writes
  `profile.json` and `instructions.md` and passes paths via env; adapter-specific
  MCP config/settings/skill-loader materialization remains designed follow-up.
- Use an ACP-speaking supervisor lifecycle fixture now that `POST /sessions`
  no longer accepts a prompt body.
- Guard scratch stop/discard terminal states: `Done` runs are no longer
  resurrected to `Review` or destroyed by discard, and `Abandoned` discard is
  idempotent.
- Add scratch dialog live refresh through the existing run SSE bridge so
  assistant/tool/HITL projections appear while the agent turn is still running.
- Keep supervisor and ACP session handles server-only in scratch detail/recovery
  responses and OpenAPI.
- Map scratch permission timeout failures to `Crashed`, matching the scratch
  state table rather than introducing `Failed` as a scratch dialog terminal.
- Make capability SET/CLEAR source-complete (`platform`, `project`,
  `flow-package`) and include `source` in the capability uniqueness/upsert
  boundary; resolver collisions now attach all matching source records
  deterministically instead of last-wins overwriting.
- Restrict scratch local promotion to the launch-time base branch, serialize
  local merges per parent repo, and add defensive `--`/ref validation around
  git positional arguments.
- Align `adapterLaunch` OpenAPI limits to supervisor Zod (`32` args, `1024`
  chars), add `NEEDS_INPUT` to web supervisor error-code parsing, and document
  scratch error columns plus removed redundant indexes.
- Write materialized `instructions.md` atomically via `atomicWriteText`.

## Follow-up rule for future agents

When changing supervisor session shape, update integration fixtures to speak the
current ACP boundary instead of keeping pre-ACP stdout-only fakes alive. A
schema-only fix can hide stale tests until they start hanging rather than
failing.

When adding scratch run lifecycle endpoints, treat `runs.status` and
`scratch_runs.dialog_status` as a single contract: terminal states must be
idempotent, and user-visible APIs must not return supervisor/ACP handles.
