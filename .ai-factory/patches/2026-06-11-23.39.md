# MiMo ACP Verification Warnings

**Date:** 2026-06-11 23:39
**Files:** supervisor/src/__tests__/adapter-compatibility.integration.test.ts, web/lib/db/__tests__/migration-0045.integration.test.ts, docs/api/supervisor.openapi.yaml, docs/supervisor.md
**Severity:** low

## Problem

The MiMo ACP adapter implementation passed the shared runtime path, but
verification found two coverage gaps and two documentation drifts:

- MiMo did not have an adapter-specific prompt permission round-trip fixture.
- The 0045 MCP supported-agents migration was not directly tested for default
  backfill behavior.
- The supervisor OpenAPI adapter description omitted MiMo even though the enum
  included it.
- The supervisor docs still showed legacy compose commands for supervisor
  startup.

## Root Cause

MiMo was added to the existing multi-adapter structures, but some validation
surfaces were updated only at the enum/list level. The test suite proved
registration and resume behavior, but it did not replay the permission path
for MiMo. The migration was simple enough to inspect manually, but no focused
test asserted that only the exact legacy default list is backfilled while
custom subsets stay unchanged.

## Solution

Added a shared ACP compatibility helper that sends a prompt, waits for
`session.permission_request`, resolves the pending permission, and asserts the
prompt completes. OpenCode keeps using the same helper, and MiMo now exercises
the same permission contract before its resume assertion.

Added a focused Postgres integration test for migration 0045. The test creates
the minimal legacy `platform_mcp_servers` table shape, seeds default,
explicit-old-list, and custom-subset rows, applies the migration SQL, and
asserts MiMo is added only where intended.

Updated the supervisor OpenAPI description and local supervisor docs so MiMo
and the host-run/Postgres-only compose topology are described consistently.

## Prevention

When adding an ACP adapter, verify the adapter-specific fixture covers the full
shared lifecycle: new session, prompt permission, model advisory, and resume
or checkpoint strategy. Migration tests should assert both the changed default
and the preservation case for rows that intentionally diverge from the old
default.
