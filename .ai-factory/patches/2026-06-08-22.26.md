# Fix: HITL Success Audits In Durable Transactions

## Problem

Remaining HITL success branches recorded token success audits outside the
transaction that committed the corresponding durable state change. If the audit
write failed, a request could leave a run resumed or an assignment completed
without the required success audit row.

## Changes

- Added focused ext HITL regressions for:
  - idle permission resume audit failure keeping the run in `NeedsInputIdle`;
  - already-delivered form retry audit failure rolling back assignment
    completion;
  - permission concurrent-winner audit failure rolling back assignment
    completion.
- Extended `markResumed`/`resumeRun` with an internal success-audit callback so
  the idle resume `NeedsInputIdle -> NeedsInput` write and the 202 audit commit
  atomically.
- Moved HITL assignment completion and success audit writes into the same
  transaction for delivered, already-delivered, and concurrent-winner success
  branches.
- Wrapped remaining 202 success audit branches in a transaction instead of using
  the outer DB handle directly.
