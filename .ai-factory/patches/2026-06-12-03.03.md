# Delivery policy execution and UI contract completion

**Date:** 2026-06-12 03:03
**Files:** `web/lib/runs/promote.ts`, `web/lib/runs/auto-delivery.ts`, `web/lib/services/runs.ts`, `web/app/api/runs/[runId]/promote/route.ts`, `web/app/api/runs/[runId]/delivery-policy/route.ts`, `web/components/board/launch-popover.tsx`, `web/components/runs/review-panel.tsx`, `docs/api/web.openapi.yaml`
**Severity:** high

## Problem

Delivery policy work was partially surfaced but not fully executable: launch stored
legacy promotion modes, promotion did not honor policy snapshots consistently,
`ai_rebase_merge` could be confused with local merge behavior, cancel-auto-delivery
accepted an implicit body shape, OpenAPI drifted from production route payloads,
and several delivery-policy controls still used native elements instead of HeroUI
surface patterns.

## Root Cause

The implementation crossed API, service, UI, and documentation boundaries, but
the route contracts and service layer still treated delivery policy as a UI hint
instead of the durable execution input. That left compatibility aliases, policy
snapshots, unsupported strategy behavior, and route schemas evolving separately.

## Solution

Centralized policy-to-promotion mapping, resolved promotion from the run snapshot
or explicit override, added push-on-success execution, added auto-on-ready
delivery through the run event path, and made unsupported `ai_rebase_merge`
explicitly degrade to manual with structured logging. The route schemas,
OpenAPI, HeroUI controls, i18n strings, and focused route/component/e2e tests
were aligned with the production behavior.

## Prevention

- Treat policy snapshots as execution contracts, not only display metadata.
- Keep compatibility aliases at route boundaries and convert immediately into
  canonical strategy names in service code.
- When a strategy is designed but not executable, expose a typed degradation
  state and test it instead of silently mapping to another strategy.
- Add route tests whenever OpenAPI request or response shapes change.
- Verify UI controls with the actual HeroUI surface and Playwright selectors.
