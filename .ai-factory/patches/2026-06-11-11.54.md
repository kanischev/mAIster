# Outbound-webhooks adversarial-review fixes: tx-atomicity, env-ref headers, replay reset, kill-switch skip, SSRF egress policy

**Date:** 2026-06-11 11:54
**Files:** web/lib/flows/graph/gate-store.ts, web/lib/webhooks/{subscriptions,replay,send,destination}.ts, web/lib/scheduler/handlers/webhook-delivery.ts, docs/{decisions,configuration}.md, docs/system-analytics/outbound-webhooks.md, docs/api/web.openapi.yaml, docs/api/async/outbound-webhooks.asyncapi.yaml, web/playwright.config.ts, .env.example
**Severity:** high

## Problem

Codex adversarial review of `feature/outbound-webhooks` raised 6 verified
findings: (1) usage-guarded subscription delete raced concurrent fanout — the
ON DELETE CASCADE could erase a just-committed audit row; (2) gate-store
`transition()` wrote the gate flip and the `gate.decided` outbox INSERT as two
autocommits (crash loses the event) with no idempotency guard (re-transition
re-emits); (4) `assertHeaders` accepted literal header values, contradicting
the documented env:NAME-only contract — secrets could persist and echo to
viewers; (5) `replayDelivery` left `delivered_at`/`last_http_status`/
`last_error_*` from the prior cycle on the re-queued row; (6) the global
kill-switch buffered (early return left events un-fanned → re-enable delivered
the whole disabled-window backlog) although docs said "skip, not buffer";
(3, owner-approved revisit) no outbound destination policy — a member-created
subscription/ping to `http://169.254.169.254/…` plus the persisted, viewer-
readable response snippet formed a cloud-IMDS credential read primitive.

## Root Cause

- **#1/#2 (atomicity class):** multi-statement invariants ("check then act",
  "write + emit") executed as separate autocommit statements on the plain `db`
  handle. The correct tx+CAS pattern already existed in-repo
  (`state-transitions.ts`, `casLiveTransition`) but `transition()` and
  `deleteSubscription` predated/skipped it. The delete additionally relied on
  a read-then-write usage check with no lock while the FK was CASCADE —
  the check could not see an in-flight INSERT it was about to erase.
- **#4:** the validation was written value-shape-deep ("is a string") instead
  of contract-deep ("is an env ref like its sibling `signing_secret_ref`") —
  the sibling-field sweep that caught `signing_secret_ref` was not applied to
  `headers`, and a comment rationalized the gap ("safe to echo back").
- **#5:** the reset enumerated the *scheduling* columns (status/attempts/
  lease) but not the *outcome* columns; nothing forced the writer to clear
  every field that becomes semantically false after the state change.
- **#6:** code shipped the cheap branch (early return) while the doc specified
  skip semantics; "disabled" was implemented as "do nothing" instead of
  "consume and drop", leaving an unbounded re-enable backlog.
- **#3:** v1 deliberately deferred SSRF blocking on an "operator-trusted"
  rationale that ignored the *member* write surface + the persisted response
  snippet read path (the exfil channel was in the audit feature itself).

## Solution

- **#2:** `transition()` now wraps the CAS UPDATE
  (`WHERE id = $1 AND status <> $target`) and `emitWebhookEvent` in one
  `db.transaction`; repeat same-status calls update 0 rows and emit nothing,
  `failed→overridden` still moves. Verified callers only ever pass a plain
  `db`, so owning the tx never nests.
- **#1:** `deleteSubscription` is one transaction: `SELECT … FOR UPDATE` on
  the subscription row (a fanout INSERT's `FOR KEY SHARE` on the FK parent
  serializes against it), then the usage check, then the DELETE. Deterministic
  two-racer test holds an uncommitted INSERT while deleting.
- **#4:** `assertHeaders` requires `isEnvRef(v)` per value → `CONFIG`/422;
  UI hint strings (EN/RU) and OpenAPI schema/examples flipped to match.
- **#5:** replay UPDATE also nulls `delivered_at`, `last_http_status`,
  `last_error_kind`, `last_error_message`.
- **#6:** disabled tick runs a batched skip pass — claim `fanout_at IS NULL`
  events (`FOR UPDATE SKIP LOCKED`) and stamp `fanout_at = now()` with zero
  deliveries and NULL payload; summary gains `skippedEvents`; drain/prune stay
  skipped, pending deliveries pause and resume.
- **#3:** new `web/lib/webhooks/destination.ts` — `node:net` BlockList over
  loopback/private/link-local(+metadata)/multicast/unspecified (v4-mapped v6
  classified by the embedded v4); write-time IP-literal rejection in
  `assertUrl`; send-time `resolveAllowedDestination` (dns.lookup all-records,
  ANY blocked answer refuses → `errorKind:"config"`, nothing sent) +
  `pinnedDispatcher` (undici Agent whose connector answers from the vetted
  addresses — DNS-rebind TOCTOU closed, proven by fetching a `.invalid` host
  through the pin). `MAISTER_WEBHOOK_ALLOW_HOSTS` exact-host allowlist for
  local consumers (required by dev/e2e at 127.0.0.1). `send.ts` switched to
  undici fetch; dispatcher closed per send. ADR-075 + system doc + OpenAPI +
  AsyncAPI + configuration.md + .env.example synced in the same change.

## Prevention

- Any "check then act" or "write + emit" pair against the DB is a transaction
  with a lock or CAS by default — autocommit pairs on the plain `db` handle
  are review-reject. Grep for `emitWebhookEvent` outside a `tx` and for
  usage-guard SELECTs not under `FOR UPDATE` when the FK is CASCADE.
- When a field family shares a sink/contract (secret refs: `signing_secret_ref`,
  `headers`), enforce the contract on EVERY sibling in the same pass — a
  comment explaining why a sibling is exempt is a smell, not a justification.
- A state-reset writer must enumerate every column the new state makes false
  (outcome fields, not just scheduling fields); test the round-trip
  (delivered→replay→dead) and assert the stale field is gone.
- When docs say "skip" and code says "return early", treat the buffer/skip
  distinction as a data-semantics bug, not wording — an early return before a
  consume step silently builds an unbounded backlog.
- For egress features, evaluate the trust claim against the LOWEST role that
  can write the destination plus every read path of the response; "operator-
  trusted" only holds if operators are the only writers and readers.
- The two-racer test recipe (hold an uncommitted FK-child INSERT on one pg
  client, run the mutation on another, commit, assert) makes cascade races
  deterministic — reuse it for future usage-guarded deletes.
