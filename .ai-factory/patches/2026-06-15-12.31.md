# Patch: Dirty Review Diff Filters Materialized Runtime Files

## Problem

The run diff and right inspector treated every dirty worktree path as a
reviewable change. Flow-materialized `.claude/skills/**` files flooded the
dirty diff and the full git-diff-view preparation could fail with
`Invalid string length`, leaving the review gate with only a file list or a
generic diff-load error.

## Root Cause

The `uncommitted` diff path used the correct dirty-tree source, but it did not
distinguish user-authored changes from runtime-materialized bundle files before
building the prepared diff DTO. Dirty inspector shortcuts also pointed to the
source file view, which cannot reliably open untracked files because no git blob
exists yet.

## Fix

- Added a server-side reviewable-change filter for materialized Claude bundle
  paths.
- Applied the filter to `uncommitted` change summaries and the `/diff` route
  before diff bundle preparation.
- Added a summary-only fallback when full diff bundle preparation fails.
- Changed dirty inspector file shortcuts to open diff anchors instead of the
  source file view.
- Defaulted the run workbench to the timeline tab when no explicit `wb` query
  is present.

## Verification

- `pnpm --filter maister-web exec vitest run lib/diff/__tests__/prepare.test.ts lib/runs/__tests__/reviewable-changes.test.ts lib/runs/__tests__/run-query-state.test.ts components/runs/__tests__/run-inspector.test.ts components/workbench/__tests__/diff-view-review.test.ts components/workbench/__tests__/run-diff.test.ts`
- `pnpm --filter maister-web exec tsc --noEmit`
- `git --no-pager diff --check`
- Manual production-like git check with a copied run index confirmed the
  concrete run filters from 35 raw dirty files to `.ai-factory/PLAN.md`.

## Prevention

Any future dirty-tree review surface must filter runtime-materialized files
before computing user-facing review counts or building heavyweight render
payloads. Dirty/untracked file actions should open diff anchors, not git-ref
source blobs.
