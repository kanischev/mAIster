# Guardrail liveness hole + non-durable agent hook_trip resume + stale HITL contract

**Date:** 2026-06-24 02:29
**Files:** supervisor/src/acp-client.ts, supervisor/test/fixtures/mock-acp-guardrail.mjs, supervisor/src/__tests__/guardrail-interceptor.integration.test.ts, web/lib/services/hitl.ts, web/lib/services/__tests__/hitl-hook-trip.integration.test.ts, docs/api/web.openapi.yaml, docs/system-analytics/hitl.md
**Severity:** high

## Problem

Codex adversarial review of the guardrail-hook-engine branch surfaced three
defects:

1. **Liveness hole** ŌĆö a `path_guard` deny `return`ed `cancelled` BEFORE the
   repetition counter ran, and `no_progress` is only fed from `sessionUpdate`
   turns. A repeated out-of-lane write (modeled as a permission request, no
   sessionUpdate) reached NEITHER breaker ŌåÆ an unattended agent could retry the
   same denied write forever (record-only denials, never the promised halt).
2. **Non-durable agent resume** ŌĆö the `hook_trip` agent resume CAS-flipped the
   run to `Running` inside the response tx and committed `respondedAt`, then
   spawned post-commit. A crash before the live session left a fake `Running`
   run; the already-delivered retry only re-drove `NeedsInput` runs, so the
   agent resume was consumed and lost (recovered only by reconcile ŌåÆ Crashed).
3. **Stale contract** ŌĆö `hook_trip` was absent from `HitlResponseBody` /
   respond-route docs and the `hitl.md` kind union.

## Root Cause

1. Interceptor ORDER: deny short-circuited before the liveness ticks, and the
   liveness model assumed every denied write would later produce a `sessionUpdate`
   turn ŌĆö false for a permission-only denied call.
2. The agent path put the `NeedsInputŌåÆRunning` CAS in the RESPONSE TX. That
   destroyed the durable re-drive signal: the flow path stays `NeedsInput` and
   self-heals because *status* (not `respondedAt`) is the re-drive signal; the
   agent path broke that symmetry. Violated backend rule 41 (two-phase commit:
   idempotency marker AFTER the side-effect) and "the runner owns NeedsInputŌåÆ
   Running, never the response route."
3. The docs were never updated when the branch added the `hook_trip` kind.

## Solution

1. Reordered `repetition` ABOVE `path_guard` (a denied write now ticks the
   repetition counter), and added a `no_progress` non-progress tick INSIDE the
   `path_guard` deny branch (catches varied denied writes + the
   path_guard+no_progress-only config). Both guarded by `hooksConfig.*`
   presence, so a path_guard-only node keeps pure deny-and-continue.
2. Left the agent run `NeedsInput|NeedsInputIdle` through the response tx; moved
   the `NeedsInputŌåÆRunning` claim into an idempotent `claimAndResumeAgentRun`
   (CAS `.returning()` ŌåÆ 0 rows = already advanced = no double-spawn) called off
   the response path AND from the already-delivered retry. Agent resume now
   returns `202 { runStatus: "NeedsInput", ŌĆ” }` ŌĆö re-aligned with the
   already-documented contract.
3. Added `hook_trip` to `HitlResponseBody` (top + `optionId`), the respond-route
   dispatch + example, and the `hitl.md` kind union (├Ś2).

## Prevention

- **A `return` inside a pre-tool-call interceptor skips every check below it.**
  When a guard "denies and continues," the liveness/Ō¢ócounter logic that is
  supposed to eventually stop the loop MUST run on the denied path too ŌĆö or move
  it above the early return. Verify with a test that arms the guard + the
  breaker TOGETHER and asserts the loop halts.
- **A resume/idempotent handler must use STATUS (not the idempotency marker) as
  the re-drive signal, and keep the resource in the awaiting state until the
  runner claims it.** Mirror the sibling kind that already self-heals (flow
  `runFlow`) rather than inventing an in-tx flip. Grep every sibling resume path
  (`infra_recovery`, `budget_breach`, flow) before diverging.
- **A new HITL kind / enum value fans out to the wire contract too** (OpenAPI
  `HitlResponseBody` + route description + `hitl.md` kind union ├Ś2). Grep the
  kind union string across `docs/` when adding a kind. Sibling gap noted:
  `infra_recovery`/`budget_breach` were also never added to `HitlResponseBody`
  prose (pre-existing, named in the reworded `optionId` to stop the contract
  being wrong).
- **Close-out grep recorded:** swept all `hook_trip` test files
  (schema/scratch/runner-agent-hooks/route/agent/escalate ŌĆö all green),
  `resume-in-progress` response-shape consumers, and the kind-union restatements.
