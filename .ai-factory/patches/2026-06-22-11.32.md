# Cost-budget governance: flow-only escalate, run.escalated in-tx, webhook contract sync

**Severity:** high

## Problem

`/aif-review` of the ADR-101 budget axis surfaced three real defects in an
otherwise-strong feature:

1. **`Raise & resume` could strand non-flow runs.** The budget watchdog shipped
   escalate adapters for `flow | agent | scratch` (all tested at the PAUSE), but
   the raise-and-resume completion (`handleBudgetBreachResponse` →
   `scheduleResume` → `runFlow`) is flow-only: `loadRun` throws `PRECONDITION
   "flow not found"` for a run with `flowId=null` (agent/scratch). An escalated
   agent/scratch run would sit in `NeedsInput` forever (HITL responded,
   `budget_state` raised, but never resumed — the throw is swallowed by
   `scheduleResume`'s `.catch`). Latent today (agent/scratch carry no budget axis
   at launch) but the adapters were shipped + tested, so the gap was real.
2. **`run.escalated` was emitted OUTSIDE the pause transaction**, unlike the
   terminate path's in-tx `run.failed`. A crash in the post-commit window lost the
   escalation event with no retry (the paused run is no longer a watchdog
   candidate) → Observatory undercount + missed webhook.
3. **The webhook wire contract rejected the feature's own events.** AsyncAPI
   `DataRunNeedsInput.reason` was `[permission, form, human]` +
   `additionalProperties:false`, so a `budget_breach` `run.needs_input` failed
   validation; `run.escalated` had NO payload schema at all. Two canonical docs
   also stated `MAISTER_DEFAULT_UNATTENDED_BUDGET_TOKENS` seeds `run` scope when
   the code seeds `tree`.

## Root Cause

A **kind-agnostic enforcement mechanism whose completion path is kind-specific.**
The watchdog candidate set spans all run kinds and the *terminate* path correctly
branches `run_kind` (flow CAS / `finalizeAgentRun` / `markScratchCrashed`). But
*escalate* pauses to `NeedsInput` and defers completion to the HITL handler, whose
resume (`runFlow`) only re-enters a flow graph. The per-kind branch was applied to
the irreversible half (terminate) but not the reversible half (escalate→raise→
resume). "Each kind has a verified escalate adapter" was asserted for the PAUSE
only — the half-A+half-B≠A∘B rule missed the resume leg.

For #2, "all persistent writes in ONE transaction" (the outbox) and "notify after
commit" (the runner wake) were conflated — the durable `run.escalated` outbox row
was placed in the after-commit bucket alongside `scheduleResume`.

For #3, a new payload variant on an existing event kind (`run.escalated`
budget variant; `run.needs_input` `budget_breach` reason) was added in code without
updating the `additionalProperties:false` AsyncAPI schema it had to satisfy.

## Solution

- **Scope escalate to flow (owner decision: flow is a must, non-flow deferred).**
  Extended the existing tree-escalate→terminate promotion in
  `evaluateBudgetForCandidate` to also promote a non-flow run/task escalate-band
  verdict to terminate. So `actBudgetEscalate` only ever runs for flow runs (its
  dead scratch branch removed); agent/scratch budget breaches terminate via the
  kind-dispatched terminate path. The two agent/scratch escalate tests were
  converted to assert terminate.
- **Moved `run.escalated` (domain + webhook) inside the pause tx** next to
  `run.needs_input` (both accept `db: tx`); only `logExecPolicyAction` +
  `scheduleResume` stay post-commit. Flattened the redundant nested `meter` object
  to flat `{ reason, scope, meter }` strings (no consumer read the nested form).
- **Synced the contract:** added `budget_breach` to `DataRunNeedsInput`, added a
  `DataRunEscalated` schema (both B3 `{nodeId,onStuck}` and budget
  `{reason,scope,meter}` variants) + oneOf member; fixed `outbound-webhooks.md`
  emit-sites/data-shapes; `run`→`tree` in the two env-var docs. Made
  `DEFAULT_BUDGET_WARN_PCT` a single exported const (was duplicated with a
  `// mirrors` comment).
- **Test gaps:** added the E5 "404-proceeds" terminate arm (terminal deleteSession
  failure still flips Failed) and an E11 "reconcile-is-called" assertion
  (spy-through-to-real).

## Prevention

- For any **periodic-sweep enforcement** action that has a reversible rung
  (pause→resume) and an irreversible rung (terminate): verify the COMPLETION/
  resume path per `run_kind`, not just the entry/pause. Grep the resume entry
  (`runFlow`/`startAgentSession`/scratch turn) against every kind that can reach
  the pause. A test must exercise the full pause→resume round-trip per kind, not
  just the pause.
- A durable outbox row (`emitDomainEvent`/`emitWebhookEvent`) is a **persistent
  write** — it belongs in the same tx as the state change (ADR-086 exactly-once),
  never the "notify after commit" bucket (that bucket is for `scheduleResume` and
  other in-process wakes only). If a sibling terminal path emits in-tx, the
  escalate path must too.
- When code adds a new payload variant to an existing event kind, update the
  `additionalProperties:false` AsyncAPI schema in the SAME change — grep the event
  `type` enum for a matching `Data<Kind>` schema and a `oneOf` member; an enum
  value with no payload schema is a contract hole.
- A `// mirrors X` constant comment is a confessed DRY drift — export the const
  from the client-safe module and import it.
