# Cost-rollup backstop sweep: durable reconcile marker (no starvation, pre-migration backfill)

**Date:** 2026-06-29 17:25
**Files:** web/lib/db/schema.ts, web/lib/db/migrations/0084_runs_cost_reconciled_at.sql, web/lib/runs/cost-reconcile-sweep.ts, web/lib/runs/__tests__/cost-reconcile-sweep.integration.test.ts, web/lib/db/__tests__/check-migrations.integration.test.ts, docs/decisions.md, docs/system-analytics/{observatory,scheduler}.md, docs/database-schema.md, docs/db/{runs-domain,erd}.md
**Severity:** high

## Problem

ADR-117's `system_sweep` cost-rollup backstop (the advertised "completeness
guarantee" for `run_cost_rollups`) keyed its candidate set on `run_cost_rollups`
ROW STATE: `rollup row missing OR rollup.updated_at < ended_at + SETTLE_GRACE`.
Two high-severity holes (found by adversarial review):

1. **Pre-0083 rollups never get `by_runner`.** Migration 0083 set `by_runner = {}`
   for every existing rollup. The dominant pre-existing trigger (run-detail open)
   reconciles a run well AFTER it ended → `updated_at >> ended_at + grace` → the
   sweep treats it as "settled" and never re-reconciles it. The event consumer is
   `startFrom:"now"` and won't replay old terminals. Result: historical runs show
   correct flat totals + by_model but permanently empty by_runner.
2. **Unreconcilable runs starve newer ones.** A run with `ended_at` set but no
   `cost.jsonl` (crashed before cost, remote-host cost, etc.) has no rollup → it
   is `rollup missing` → candidate EVERY tick. The sweep is oldest-first, capped
   at 50/tick, and writes no durable marker on a `missing-cost-file`/`CONFIG`
   outcome, so a block of >50 such old runs monopolizes the scan forever and
   newer no-event scratch successes behind them are never reconciled.

## Root Cause

The sweep's progress was inferred from the OUTPUT artifact (`run_cost_rollups`
row), which cannot represent two real states: "attempted but produced no rollup"
(missing cost) and "rollup exists but is stale w.r.t. a new column" (empty
by_runner on a long-settled row). A scan whose termination depends on a data
state it can't always reach is non-terminating for those states.

## Solution

Introduced a durable per-run progress marker `runs.cost_reconciled_at`
(timestamptz NULL, migration 0084), stamped in a `finally` on EVERY sweep attempt
(reconciled / missing-cost / error). New predicate keys on the marker, not rollup
state: `ended_at IS NOT NULL AND ended_at > now − lookback AND
(cost_reconciled_at IS NULL OR cost_reconciled_at < ended_at + SETTLE_GRACE)`.
- Missing-cost run → attempted once, marker stamped past grace → settled → frees
  the scan (fixes starvation).
- Pre-0083 rollup → NULL marker → re-reconciled once → `by_runner` backfilled
  (fixes completeness). NULL is the natural "never swept" seed for existing rows;
  no data migration needed (a SQL default can't derive per-runner splits anyway).
- SETTLE_GRACE semantics preserved: a marker within `ended_at + grace` still
  re-selects (late-flush capture), then settles.
Rejected the review's lighter "treat empty by_runner as stale" OR-clause: it
would have AMPLIFIED finding 2 (a missing-cost run with empty by_runner stays a
candidate forever). The marker is the single root-cause fix for both.

## Prevention

- A bounded oldest-first scan whose candidacy is "work not yet done" MUST persist
  a durable per-item attempt marker, independent of whether the work produced an
  output row — otherwise unprocessable items starve the queue.
- When a migration adds a NOT-NULL-default column that needs per-row computation
  (not a constant), plan the backfill path explicitly; a `default '{}'` is a
  silent "looks populated but isn't" trap.
- Regression tests added: (a) pre-0083 rollup (empty by_runner, settled
  updated_at, NULL marker) → backfilled; (b) >limit missing-cost old runs before
  a newer healthy run → healthy reconciled within two ticks (starvation guard).
- Adversarial review caught this; the original test suite only covered the
  happy/within-grace/settled cases with seeded rollups, never the no-rollup or
  pre-migration states.
