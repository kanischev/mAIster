# Flow Studio Phase 2 verify blockers: frontmatter gate over-scope, EISDIR dir target, ADR renumber

**Date:** 2026-06-11 09:20
**Files:** web/lib/flows/artifact-validate.ts, web/lib/flows/package-content.ts, web/lib/flows/__tests__/artifact-validate.test.ts, web/lib/flows/__tests__/package-content.test.ts, web/app/api/projects/[slug]/flow-packages/[flowRefId]/revisions/[revisionId]/fork/__tests__/fork.integration.test.ts, web/components/flows/node-form/__tests__/node-side-form.test.ts, web/e2e/flows-authoring.spec.ts, docs/decisions.md, docs/system-analytics/{flow-studio,flow-packages}.md, docs/api/web.openapi.yaml, .ai-factory/ROADMAP.md
**Severity:** high

## Problem

`/aif-verify` failed the Flow Studio Phase 2 branch with four blockers:

1. The per-kind content gate raised BLOCK `frontmatter_missing` for EVERY file
   under `skills/` or `agents/` — a forked package carrying skill aux files
   (`skills/x/references/notes.md`, the standard AIF package shape) could never
   save its draft (CONFIG 422 on every save).
2. `?file=<existing-directory>` on the package viewer passed the whole
   confinement chain, then `readFile(dir)` threw an unhandled EISDIR → RSC 500
   for any member, violating the typed-degraded-state contract.
3. The plan-listed fork-then-first-save integration test was never written —
   exactly the test whose realistic bundle would have caught (1).
4. The new ADR was numbered 072, but main took ADR-071–074 while the branch
   was in flight — a guaranteed merge collision in the globally-sequential
   ADR registry.

## Root Cause

1. **Editor classifier reused as a contract scope.** `classifyPackageFilePath`
   is prefix-only BY DESIGN (it picks which *editor* opens a file); the
   validator borrowed it as if it also defined which files carry the runtime
   frontmatter *contract*. Two different concepts. The severity table said
   `skills/**/SKILL.md` / `agents/*.md`, but every test pinned only canonical
   paths — no aux-file negative test existed, so the over-broad gate shipped
   green.
2. **Confinement checked path SAFETY, not target TYPE.** The chain handled
   escape (zod/lexical/realpath) and absence (ENOENT), but a directory passes
   realpath and `stat`, then explodes at `readFile`. The traversal test matrix
   (`../`, absolute, NUL, leading `-`, symlink, missing) had no
   directory-target row.
3. **Numbering verified against the wrong ref.** The implementer re-verified
   the ADR max at implementation time against the BRANCH BASE (max 070). On a
   long-lived parallel branch the only ref that matters is MAIN at
   verify/commit time (`git show main:docs/decisions.md`).

## Solution

- `artifact-validate.ts`: frontmatter checks now run only when
  `isSkillDefinitionPath` (basename `SKILL.md` under `skills/`, any depth) or
  `isAgentDefinitionPath` (`agents/*.md`, single segment) matches; editor
  dispatch untouched. Doc severity table already stated this scope — code
  converged to doc (plus `schemas/*.json` → `schemas/**/*.json` wording fix).
- `package-content.ts`: `if (!stats.isFile()) return { state: "not-found" }`
  after `stat`, mirroring the silent ENOENT sibling. No new logging: the
  module's pino boundary logs confinement REJECTS; a benign dir target degrades
  exactly like a missing file (repo `no-console` rule + existing convention
  override the generic `[FIX]` console pattern).
- Two fork-then-first-save integration tests: aux-file bundle forks AND saves;
  SKILL.md-missing-description bundle forks (no gate at fork) but first save is
  CONFIG-blocked with `[frontmatter_field_missing]`, draft_version untouched.
- ADR renumbered 072→075 across all 9 references (decisions.md index row +
  anchor slug + heading + context prose, flow-studio.md ×2, flow-packages.md,
  OpenAPI, ROADMAP, plan), the stale "latest migration stays 0037" claim
  replaced with "this feature adds no migration".
- Extras: node-side-form presentation inputs got their first unit coverage;
  `flows-authoring.spec.ts:113` now opens the YAML tab after reload (the saved
  manifest compiles → graph tab is the designed default), turning the whole
  spec green where base had 4 reds.

## Prevention

- When a gate's scope is defined by a doc table, write the NEGATIVE-scope test
  (the file that must NOT trigger) in the same change as the positive one.
- An editor/display classifier is not a validation scope. If a validator needs
  "which files carry contract X", give it its own predicate named after the
  contract, next to the gate.
- Add "existing directory" to the standard path-confinement test matrix,
  alongside `../`, absolute, NUL, leading `-`, symlink escape, and missing.
  Every `stat`→`readFile` sink needs an `isFile()` type check, not just escape
  checks.
- ADR/migration/M-numbers on parallel branches: grep MAIN at verify and again
  at commit time, never the branch base; budget a renumber pass into any
  long-lived branch (`git show main:docs/decisions.md | grep "^### ADR-"`).
- Any "copy now, validate later" flow (fork, import, restore) gets a
  copy-then-first-save test with a REALISTIC fixture, not a minimal one.
