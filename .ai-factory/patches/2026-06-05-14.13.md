## Problem

A full-branch `/aif-review` of `feature/m22-workbench-visibility` found no
merge-blocking issues but seven non-blocking ones. Fixing them surfaced five
recurring classes worth recording:

1. The `GET /api/runs/{runId}/diff` route returned **409** (`PRECONDITION`,
   "run not found") for a non-existent run, while its sibling M22 routes
   (`/files`, `/graph`, `/graph-status`) return **404** â€” and the route's own
   OpenAPI documented the 409 for not-found, so the contract was internally
   honest but inconsistent across the surface.
2. `flow-graph-view.tsx` threaded a fully-translated `labels.node`
   (`Pending/Running/...` status map) from the server page down through props,
   but **never rendered it** â€” node status was conveyed by color only (also a
   WCAG "color alone" gap).
3. `useRunStream` unconditionally accumulated **every** SSE event into a state
   array. A new consumer (`flow-graph-view`) only read `events.length` as a
   change-tick, so a long-lived run grew the array without bound.
4. ADR-053 and `workbench.md` documented the listing command as
   `git ls-tree --name-only -z ...`, but the code runs `ls-tree -z` **without**
   `--name-only` (it needs the `<mode> <type> <sha>` token to derive
   `file|dir`). The documented command, if followed, would break the feature.
5. The workbench file tree (`<ul role="tree">`) had no `aria-label`.

## Root Cause

1. **Per-route not-found semantics drift.** The diff route predates the M22
   file/graph routes and reused a shared `loadRun` helper that threw
   `PRECONDITION`. When the newer routes standardized on bare-404 for
   not-found, the older route was not re-aligned â€” and nothing forced the
   reconciliation because each route's own contract was self-consistent.
2. **Plumb-then-forget i18n.** A translated label map was added to the props
   contract and wired end-to-end before the render site that would consume it
   existed; the render site shipped reading the raw enum (via color) and the
   now-orphaned map was never pruned or hooked up.
3. **Shared primitive, single-shape assumption.** `useRunStream` was written
   for a consumer that displays the event list (the dev fixture). A second
   consumer needing only a count inherited the full-retention cost because the
   hook offered no opt-out.
4. **Docs written from intent, not from the final argv.** The `--name-only`
   flag was the obvious way to "list names," but the implementation needed the
   type token, so the prose and the code diverged at the detail level.
5. Standard tree markup shipped without the WAI-ARIA accessible name.

## Solution

1. `loadRun` now returns `run | null`; the GET handler returns a bare **404**
   for a missing run (both scratch and flow paths), keeping **409** only for
   the genuine preconditions (workspace missing/removed, base unavailable).
   OpenAPI gained a `404 â†’ NotFound` ref and the 409 description was narrowed.
   Added a regression test asserting 404 (not 409) for an unknown run id and
   that `requireProjectAction`/git are never reached.
2. `FlowNodeBody` gained an **optional** `statusLabel` prop rendered as the
   chip label's `title`; `makeFlowNodeView` passes `labels.node[status] ?? status`.
   Optional so the standalone presentational render test (which omits it) still
   typechecks and passes. Added an assertion that the translated label renders
   as `title`.
3. `useRunStream(runId, { retain })` â€” `retain` defaults `true` (dev fixture
   unchanged); `flow-graph-view` passes `retain: false` and reads a new
   `eventCount` field. The effect depends on the primitive `retain`, not the
   options object, to avoid a re-subscribe storm.
4. Dropped `--name-only` from both ADR-053 and the `workbench.md` mermaid node.
5. Added an optional `treeLabel` to `FileTreeLabels`, applied as `aria-label`
   on the depth-0 `<ul role="tree">`, wired from both mount points (run page +
   project board) with a new EN/RU `workbench.files.treeLabel` key.

Also hardened `extractFileSection` (`.includes(' b/<path>')` â†’ `.endsWith`) â€”
equivalent for well-formed git headers but states the "new path is the final
token" intent; and switched the diff fetch-error state from the reused
`labels.empty` ("No changes") to a distinct `labels.error`.

## Prevention

- **Enumerate not-found semantics across a route family.** When adding routes
  that share a resource shape, fix the not-found status code (404 vs 409) once
  for the whole family and reconcile older siblings in the same change. A
  per-route-consistent contract can still be surface-inconsistent â€” grep the
  family before shipping.
- **A label map plumbed through props must have a consuming render site in the
  same change** â€” or don't add it yet. A typed-but-unread prop is dead weight
  and a latent untranslated-enum leak. Reviewers: trace each `labels.*` field
  to a JSX render site.
- **A shared SSE/stream hook that accumulates state needs an opt-out** the
  moment a count-only/long-lived consumer appears. Default to retain for
  back-compat; gate retention on a primitive flag (stable effect dep).
- **Docs that quote a command must quote the real argv**, including flags the
  parser depends on (`--name-only` strips the type token `ls-tree` output
  parsing needs). When a doc shows a CLI invocation, diff it against the code's
  actual args array.
- **Tree/tab/list ARIA markup ships with an accessible name** (`aria-label`)
  from the first version; thread it as an optional i18n label so all mount
  points can supply it.
