# Verify-blocker fixes on the model-discovery branch (guard hole, resume coverage, dead i18n, undocumented env, stale docs)

**Date:** 2026-06-11 12:25
**Files:** supervisor/src/model-catalog/harvest.ts (+test), supervisor/src/__tests__/model-application.integration.test.ts, supervisor/test/fixtures/mock-acp-models.mjs, web/components/settings/model-autocomplete.tsx (+test), web/lib/flows/graph/runner-graph.ts (format), .env.example, docs/{configuration,supervisor,architecture,decisions,CLAUDE}.md, docs/system-analytics/model-catalog.md, docs/api/async/supervisor-sse.asyncapi.yaml, docs/spikes/2026-06-11-acp-model-discovery-spikes.md
**Severity:** high

## Problem

`/aif-verify` on `feature/acp-runner-model-discovery` found five blockers the
green suites had missed: (1) `harvestSessionModels` could throw a `TypeError`
into the live session-spawn path on a malformed adapter `models` payload;
(2) the resumed-session model-application path (plan T3.2's "dedicated
RESUMED-session case") had zero test coverage; (3) the suggestion-group origin
badges rendered server-hardcoded EN labels while the EN/RU
`modelSuggestions.sources.*` catalog keys existed unconsumed (RU users saw EN);
(4) `provider-api.ts` read `OPENAI_API_KEY` at runtime but no `.env.example`/
`configuration.md`/domain-doc entry mentioned it — the openai discovery source
was undiscoverable-by-operators (always `skipped`), and the plan had claimed
"no new env var"; (5) five docs spots described superseded mechanisms
(SIGTERM-only teardown, harvest "feed", codex advisory read-back loop,
"content-only" claude fix, missing docs/CLAUDE.md glossary row).

## Root Cause

- **(1) Guard placed outside the try it was guarding for.** The
  `models.availableModels.length === 0` early-return ran before the `try` that
  implements the function's own "MUST NEVER throw into the session flow"
  contract — and the ACP SDK applies no response schema to `session/new`, so the
  typed `SessionModelState` is a fiction at the wire boundary. The unit tests
  fed only well-formed literals, so the hole was invisible.
- **(2) Plan-promised test trimmed during implementation.** T3.2's TDD bullet
  named the resumed-session case; the implementation pass landed the resume
  wiring but only the `session/new` integration test. Nothing made the
  test-list survive the hand-off ("tests have a habit of being trimmed").
- **(3) Translation keys created in one task (T4.4), consumption forgotten in
  the sibling task (T4.3).** The server route shipped its own EN `label`
  fallback, the component rendered it verbatim, and the EN/RU parity test only
  checks key PRESENCE — dead keys are invisible to it.
- **(4) "Plain provider kinds carry no env-ref field"** forced a hardcoded
  conventional-name fallback, but the House-Rule "no new env var" claim from
  the plan was never re-checked against the implementation.
- **(5) The five adversarial-review fixes changed behavior after the docs were
  frozen** — code and docs diverged in the same branch because the fix pass
  did not sweep the spec.

## Solution

- **(1)** Guard became `!models?.availableModels?.length` (optional-chain the
  whole unvalidated path) + a RED-first regression test feeding `{}` and a
  non-array `availableModels`. Sibling-field sweep (per skill-context rule):
  `applyAndVerifyModel` already uses `models?.currentModelId` and the probe
  already uses `resp.models?.availableModels ?? []` — no siblings needed fixing.
- **(2)** `mock-acp-models.mjs` now advertises `sessionCapabilities.resume` and
  returns `models` from `resumeSession()`; a new integration case drives
  `POST /sessions` with `resumeSessionId` for a codex runner and asserts the
  `model_advisory` (`channel: set_session_model` — the SDK's deterministic
  `methodNotFound` rejection proves the apply attempt ran on the resume
  response), non-fatality, and the harvested cache entry. `openai_compatible`
  drafts are refused by `provisionRunnerLaunch` pre-spawn — the test runner
  uses the plain `openai` kind.
- **(3)** `model-autocomplete.tsx` maps `group.source` →
  `t("modelSuggestions.sources.<key>")` with the server `label` as
  unknown-source fallback; component test asserts the keys (mock-returns-key
  convention) + a fallback case. EN values equal the old hardcoded labels, so
  e2e stayed green while RU activated.
- **(4)** Documented `OPENAI_API_KEY` in `.env.example` + the
  `configuration.md` env table (+ a model-discovery clause on the
  `ANTHROPIC_API_KEY` row) + the domain doc Edge cases + supervisor.md prose +
  ADR-075 §3/§7 ("no new **required** env-var").
- **(5)** Swept the spec to as-built: teardown = SIGTERM → bounded grace →
  SIGKILL (5 spots), harvest = merge-not-replace with `insertedAt`-anchored
  expiry (state machine formula + entity + flow + ADR §4), codex advisory fires
  on a FAILED `setSessionModel` call (no read-back loop — asyncapi summary +
  `observedModelId` description + ADR §6 + application diagram), ADR §5/spike
  corrected for the explicit-empty materialization, docs/CLAUDE.md glossary row
  added. Plus the 2 prettier warnings in runner-graph.ts via scoped
  `eslint --fix <file>`.

## Prevention

- **Guards protecting a no-throw contract belong inside (or before via
  optional chaining) the try — and trust-boundary inputs (adapter wire
  responses) get optional-chained access even when the SDK type says the field
  is required.** A type is not validation.
- **When a plan's TDD bullet names a test case, the implementing task is not
  done until that named case exists** — re-derive the test list from the plan
  at verify time, not from the suite's green-ness.
- **A new i18n key is only done when something consumes it** — parity tests
  check presence, not wiring; grep for the key's usage before calling an i18n
  task complete.
- **Every `process.env.X` added in a diff must appear in `.env.example` + the
  `configuration.md` table in the same change** — "no new env var" claims in
  plans are re-checked against `grep process.env` of the final diff.
- **A behavior-changing fix pass sweeps the frozen spec in the same pass** —
  grep the domain doc/ADR/asyncapi for the mechanism words the fix changed
  (here: "SIGTERM", "read back", "feeds", "content-only").
- **Mermaid note/message text must not contain `;`** — it is a statement
  separator; use an em dash.
