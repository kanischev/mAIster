# Close /aif-review findings on G4 guardrail-hooks (test gaps, DRY, wire-symmetry, doc honesty)

**Date:** 2026-06-23 18:25
**Files:** supervisor/src/guardrail-hooks.ts, supervisor/src/acp-client.ts, supervisor/src/types.ts, supervisor/test/fixtures/mock-acp-guardrail.mjs, supervisor/src/__tests__/{guardrail-hooks,guardrail-interceptor.integration,types}.test.ts, web/lib/agents/__tests__/definition.test.ts, docs/api/supervisor.openapi.yaml, docs/api/async/outbound-webhooks.asyncapi.yaml, docs/system-analytics/guardrail-hooks.md
**Severity:** low

## Problem

A `/aif-review` of the G4 guardrail-hooks branch (Phases 0–2 of M40) surfaced
non-blocking findings on otherwise-sound code: (1) the halt path's
`pendingPermissions.cancel` loop was only "tested" under `autoApprove`, where the
deferred count is already 0 — so the no-leaked-deferred guarantee was effectively
unverified; (2) `hookHalted` short-circuit, `repetition` re-trip-after-reset,
`no_progress` reset-on-write, and the kind-only WARN-once guard lacked coverage;
(3) `WRITE_KINDS` (guardrail-hooks.ts) was byte-identical to
`READ_ONLY_MUTATING_KINDS` (acp-client.ts) with a `// same closed set as …`
comment confessing the duplication; (4) the supervisor wire Zod accepted
`pathGuard.allowedPaths: []` / `[""]` while the web authoring schema forbade them
(asymmetric contract); (5) a webhook-spec line described an unbuilt Phase-3 emit
in live present tense, and a domain-doc cross-ref pointed at a file that never
lists the cited enum value. The migration-0063 renumber was explicitly deferred
(owner: "leave renumberings for later rebase" — correct per the project rule that
renumbering is its own post-rebase pass).

## Root Cause

- **Vacuous assertion under the wrong fixture.** The integration mock ran every
  scenario with `autoApprovePermissions=true`, so permission requests resolved
  inline and no deferred was ever pending when a halt fired. `expect(size).toBe(0)`
  passed for the wrong reason. The cancel-on-halt branch only matters when a real
  HITL deferred is open — which, in production, happens on the async `no_progress`
  path (a human is mulling a permission while the watchdog trips).
- **Two ADRs grew the same constant independently.** ADR-078 (L2 read-only-turn)
  and ADR-104 (path_guard/no_progress) each needed "is this tool kind mutating?";
  the second author noticed the duplication but left a comment instead of sharing.
- **Emitter-stricter-than-acceptor drift.** The resolver (web) always fills
  `allowedPaths` (>= `["**"]`), so the lax supervisor schema never bit in practice
  — the gap was invisible until a reviewer compared the two wire ends directly.

## Solution

- **Deterministic deferred-cancel test via stream FIFO.** Added a `deferred_cancel`
  mock scenario that fire-and-forgets a real write `requestPermission` (autoApprove
  OFF → a deferred registers), then emits idle turns to trip `no_progress`. Because
  the ndjson stream is FIFO, the supervisor registers the deferred before processing
  the idle turns, so the halt's cancel loop deterministically releases it — the mock
  `await`s the cancelled call, so a leak hangs the prompt past the test timeout
  (the hang IS the assertion). Also added `repetition` short-circuit (count>max),
  `no_progress_reset`, and `path_guard_kindonly` (WARN-once via a capturing pino
  logger threaded through a parametrized `boot(logger)`), plus unit cases for
  re-trip-after-reset and the new `allowedPaths` rejections.
- **DRY:** made `WRITE_KINDS` the single source of truth (kept in the pure leaf
  `guardrail-hooks.ts`); `acp-client.ts` now imports it and the L2 path uses it —
  the confessing comment is gone.
- **Wire symmetry:** tightened the supervisor Zod `pathGuard.allowedPaths` to
  `z.array(z.string().min(1)).min(1)` and mirrored `minItems/minLength` in
  `supervisor.openapi.yaml`.
- **Doc honesty:** tagged the webhook `hook_trip` emit `(Designed — M40 Phase 3,
  not yet emitted)`; repointed the domain-doc Schema cross-ref from
  `db/assignments-domain.md` (no enum list) to `db/erd.md` (carries the value);
  added a `no_progress` edge case for the zero-tool-call stall (covered by the
  ADR-101 budget, not the hook engine).
- **NO `[FIX]` logging added** — the changes are tests + a pure-const move + a Zod
  constraint + docs, none with runtime logic; the project bans `console.log`
  (`no-console` + pino boundary), so the skill's generic logging mandate yields to
  the project convention.

## Prevention

- **A `size === 0` / "no leak" assertion is only meaningful if the resource could
  be non-zero in that fixture.** When testing a release/cleanup path, first arrange
  the state it cleans up; assert the count was >0 before, or use a fixture
  (autoApprove OFF) where the resource genuinely exists.
- **Exploit stream FIFO for deterministic cross-channel ordering tests** instead of
  sleeps/polling: enqueue the request before the trigger on one ndjson stream and
  the consumer processes them in order.
- **A `// same set as X` / `// mirrors X` comment is a must-fix, not a note** —
  extract to one SSOT in the pure leaf module and import it (the project
  skill-context calls this out explicitly).
- **Wire schemas have two ends; review them as a pair.** Even when the emitter is
  stricter today, make the acceptor symmetric so a direct-POST can't smuggle a
  shape the authoring layer forbids.
- **Tag not-yet-emitted wire shapes `(Designed)`** in OpenAPI/AsyncAPI prose, and
  make every doc cross-ref resolve to a file that actually carries the cited value.
- **Defer namespace renumbering to a post-rebase pass** — never reconcile a
  migration/ADR number against a stale branch baseline.
