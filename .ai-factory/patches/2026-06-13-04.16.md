# Fix Patch: Platform agents review fixes

## Summary

- Fixed worktree-backed agent completion so a clean agent exit lands in `Review`
  and must be explicitly promoted through the existing workspace promotion path.
- Reused the durable workspace promotion service for agent worktrees while
  preserving the Flow evidence gate only for `run_kind=flow`.
- Moved task verdict PATCH and HITL response body parsing behind authentication
  and project authorization gates.
- Made agent finalization the single cleanup boundary for agent abandon/discard:
  token revoke, open HITL close-out, assignment close-out, terminal outbox, and
  workspace GC deadline now happen in the same status-guarded path.
- Made declared agent MCP profile resolution fail closed instead of silently
  launching without requested MCPs.
- Updated current-state docs/copy for package-owned agents and the current
  `MAISTER_MAX_CONCURRENT_RUNS=6` default.
