# Schema reference edit stayed outside schema boundaries

**Date:** 2026-06-26 03:12
**Files:** `web/lib/flows/editor/schema-ref-actions.ts`, `web/lib/flows/editor/reference-sources.ts`, `web/components/flows/node-form/schema-ref-field.tsx`, `web/lib/flows/editor/__tests__/schema-ref-actions.test.ts`, `web/components/flows/node-form/__tests__/schema-ref-field.test.ts`
**Severity:** high

## Problem

Flow Studio schema edit accepted arbitrary free-text refs as package paths and
could overwrite non-schema package files through the shared draft state. The
schema JSON editor also kept stale content after the selected schema ref
changed, so editing `./schemas/review.json` after `./schemas/intake.json` could
write intake content into review.

## Root Cause

Schema edit reused the general schema ref normalizer (`trim` + remove `./`) as
if it were a schema path validator. The UI also seeded `jsonDraft` from selected
schema content only at component mount and never synchronized it when
`value`/`schemaFiles` selected a different existing schema.

## Solution

Added `isRootSchemaFilePath` to centralize the `schemas/*.json` predicate and
made `buildSchemaWriteFromRef` reject edit refs outside
`./schemas/<name>.json`. Synced `SchemaRefField`'s JSON draft whenever the
selected schema file content changes. Added RED-first tests for invalid edit refs
and changed-selection edit content.

## Prevention

When a helper converts a UI ref into a package-file write path, pair
normalization with a namespace validator in the same module. For state
initialized from props, add an interaction test that rerenders with changed
props before performing the action that consumes the state.
