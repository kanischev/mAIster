# Silent partial diff promotable in the review panel; stale 413/415 file-open docs

**Date:** 2026-06-09 13:01
**Files:** web/lib/worktree.ts, web/lib/diff/prepare.ts, web/app/(app)/runs/[runId]/layout.tsx, web/app/api/runs/[runId]/diff/route.ts, web/app/api/runs/[runId]/takeover/return/route.ts, web/app/api/runs/[runId]/artifacts/[artifactId]/payload/route.ts, web/components/workbench/diff-view.tsx, web/components/workbench/run-diff.tsx, web/components/runs/review-panel.tsx, web/messages/{en,ru}.json, docs/api/web.openapi.yaml, docs/system-analytics/{workbench,manual-takeover}.md, docs/error-taxonomy.md, docs/configuration.md, docs/decisions.md, web/lib/instance-config.ts
**Severity:** high

## Problem

Two codex review findings on the ADR-066 editor/diff branch:

1. **[high]** The promotion review panel could render a *silently truncated*
   diff. `buildReviewPanelData` fed `diffRange()` ŌåÆ `prepareDiff()` ŌåÆ the panel.
   On a diff over the 4 MiB `EXEC_MAX_BUFFER` bound, `diffRange` fell back to a
   streaming reader that appended an in-band marker string and returned a partial
   diff. `prepareDiff` returned only `{ files, perFile }` with **no structured
   truncation state**, so the marker was swallowed into the last file section and
   the **Promote button stayed fully enabled** ŌĆö a reviewer could promote a run
   having seen only a prefix of the change, with no blocking indication.
2. **[medium]** `docs/system-analytics/workbench.md` (+ error-taxonomy,
   configuration, ADR-053/066) still claimed oversized blobs return HTTP `413`
   and binary blobs `415`. ADR-066 retired the `ŌĆ”/files/content` HTTP route and
   moved blob opens to the `?file=` RSC path, which renders `file-too-large` /
   `file-binary` **page states** ŌĆö the documented contract was invalid.

## Root Cause

Finding 1: truncation was signalled **in-band** (a magic marker appended to the
diff text) rather than as **structured metadata**. An in-band signal is invisible
to any consumer that parses the text into a typed DTO ŌĆö the parser dropped it, so
the "this is incomplete" fact never reached the UI. The graceful-degradation
fallback (truncate instead of throw) is correct, but degradation without a
machine-readable flag turns a safety feature into a silent-data-loss hazard at
the most dangerous decision point (promotion). Also asymmetric: the workbench
`/diff` reader (`diffRunWorkspace`) *threw* 409 on the same condition, so behavior
diverged between two readers of the same git delta.

Finding 2: a capability migration (ADR-066 retiring `ŌĆ”/files/content`) updated the
code and the *primary* doc paragraph but not the Expectations bullets, the mermaid
node labels, the edge-case list, the error-taxonomy callout, the env-var table, or
the source comment ŌĆö the contract surface was wider than the one paragraph touched.

## Solution

Finding 1 ŌĆö structured `truncated` flag end-to-end (user chose the "unify" scope):
- `diffRange`/`diffRunWorkspace` now return `{ text, truncated }`; both share one
  `streamGitDiffTruncated` fallback so the workbench `/diff` reader also degrades
  gracefully (no longer throws 409 on size). The in-band marker is dropped from
  the structured data path (kept only where a plain-text surface ŌĆö `returned_diff`,
  the git-range artifact payload ŌĆö re-appends it, since they have no flag channel).
- `prepareDiff(raw, truncated)` threads the flag onto `DiffPrepResult`; it rides
  `RunDiffResponse` (+ OpenAPI), `ReviewPanelDiff`, and `DiffView`.
- `DiffView` renders a truncation banner (both workbench + review surfaces).
- `ReviewPanel` BLOCKS Promote behind an explicit "Promote anyway (diff truncated)"
  acknowledgement ŌĆö mirrors the existing `driftDetected` escape-hatch idiom, so a
  legitimately huge diff is not stranded but cannot be promoted silently.
- Regression tests: `prepareDiff` truncation unit, diff-route `truncated:true`
  propagation, ReviewPanel gated-promote render test, real-git `diffRange`
  truncation shape.

Finding 2 ŌĆö rewrote every 413/415 mention to the RSC page-state contract across
workbench.md (prose, mermaid, Expectations, edge cases, cross-ref), error-taxonomy,
configuration, the ADR-066 wording, and the stale source comment. Distinguished the
surviving `ŌĆ”/files?path=` tree API (HTTP) from the `?file=` RSC blob render.

## Prevention

- **Degrade with a flag, never a marker.** When a reader truncates/caps/samples
  instead of failing, the result MUST carry a structured boolean/enum the typed
  consumer can branch on. An in-band sentinel string is invisible to any DTO
  parser and WILL be dropped silently. Grep test: does the partial-result path set
  a field, or only mutate the payload text?
- **Symmetry check on duplicated readers.** Two functions reading the same thing
  (`diffRange` vs `diffRunWorkspace`) must agree on the failure/degradation mode.
  When you touch one, diff its error path against its sibling.
- **Block irreversible actions on incomplete evidence.** Any "you can't fully see
  X" state at a promote/merge/approve gate must gate the action (block, or
  block+explicit-override), never just annotate. Reuse the existing escape-hatch
  pattern (`driftDetected`) for consistency.
- **A retirement is a contract-wide sweep, not a paragraph edit.** When code drops
  a route/status/field, grep the WHOLE docs tree for the old token (`413`, `415`,
  `files/content`) ŌĆö Expectations bullets, mermaid node labels, edge-case lists,
  error-taxonomy, env tables, ADR wording, and source comments all drift
  independently of the primary prose.
- **`.ai-factory/ROADMAP.md` is a historical milestone log** ŌĆö leave its record of
  what shipped in a past milestone even when later superseded (and it is read-only
  for `/aif-fix`); fix the *current-contract* docs instead.
