# acp-runner-crud review fixes: conflict-check race, contract drift, auth-first, server-derived readiness

**Date:** 2026-06-08 13:13
**Files:** web/app/api/admin/acp-runners/route.ts, web/app/api/admin/acp-runners/[runnerId]/route.ts, web/app/api/admin/acp-runners/__tests__/route.test.ts, web/lib/__tests__/i18n-acp-runner-keys.test.ts, docs/api/web.openapi.yaml, supervisor/src/acp-client.ts
**Severity:** high

## Problem

`/aif-review` of the acp-runner-crud feature surfaced:

1. **Create conflict-check not race-safe.** POST `/api/admin/acp-runners` did `db.select().from(platformAcpRunners)` (full-table scan, no WHERE) + `.some(r => r.id === id)`, then `insert`. Two concurrent POSTs with the same id both pass the SELECT; the second hits the id PK constraint as a raw Postgres `23505` → 500 instead of the documented 409.
2. **OpenAPI response contract lied.** GET declared `platformDefaultRunnerId` (code returns `defaultRunnerId`, + undocumented `adapters`/`sidecars`); POST 201 & both PATCH 200 declared `{runner: object}` (code returns `{ok,id}` / `{ok}`). The settings PATCH (`PATCH /api/admin/acp-runners`) was entirely undocumented.
3. **Auth-ordering.** POST/PATCH parsed + Zod-validated the body BEFORE `requireGlobalRole("admin")`.
4. **Server-derived fields accepted from the body.** `readinessStatus`/`readinessReasons` were in the request Zod schemas (overridden server-side, but a trust-smell).
5. **`model: z.string().min(1)`** accepted whitespace-only server-side (client trimmed).
6. **i18n test** omitted the `cancel`/`save`/`saving` keys the runner modal uses.
7. (supervisor) resume fallback silently `newSession`'d when an adapter lacks `sessionCapabilities.resume` — the same empty-session class as the original resume bug.

## Root Cause

- The conflict check was a read-then-write (TOCTOU): correctness was delegated to application code that cannot see an uncommitted concurrent insert. The DB unique constraint is the only race-safe arbiter, but its raw error was not translated to the typed 409.
- "Code lands, docs follow" drift: the route shapes evolved (to `{ok,...}`, which is also the *secure* shape — it avoids returning the stored row) but the OpenAPI was not re-derived from the code; the settings PATCH shipped with no path entry.
- Defense-in-depth ordering and server-authoritative-field hygiene were not enforced at the schema/handler boundary.

## Solution

- **Race-safe create:** dropped the pre-SELECT; `insert(...).onConflictDoNothing().returning({id})` — an empty returning yields the typed `CONFLICT` (409); a real duplicate can no longer surface as 23505/500.
- **Contract:** aligned `web.openapi.yaml` to the code ("code wins" + the `{ok,...}` shape avoids `#response-leak`): GET `defaultRunnerId` + `adapters`/`sidecars`; POST 201 `{ok,id}`; `{runnerId}` PATCH 200 `{ok}` + added its requestBody; added the missing `PATCH /api/admin/acp-runners` (set-default) path; dropped `readinessStatus`/`readinessReasons` from the POST request schema.
- **Auth-first:** `await requireGlobalRole("admin")` is now the first statement in POST/PATCH (collection) and `{runnerId}` PATCH; body parse/validation moved inside the post-auth try.
- **Server-derived only:** removed `readinessStatus`/`readinessReasons` from both Zod schemas (`.strict()` now rejects them, 422); readiness stays recomputed server-side on every write.
- **`model: z.string().trim().min(1)`** in both schemas.
- **i18n test** extended with `cancel`/`save`/`saving`.
- **Supervisor:** the resume branch now `throw new SupervisorError("CHECKPOINT", …)` when `sessionCapabilities.resume` is absent — fail loud, never a silent empty session.
- Updated unit tests: fake DB now models `onConflictDoNothing().returning()`; readiness tests assert server-derivation without sending the field; added a 422-on-caller-readiness test. Added a **two-racer integration test** (`route.concurrency.integration.test.ts`, testcontainer postgres): two concurrent same-id POSTs MUST yield exactly one 201 + one typed 409, never 500, with exactly one row persisted (Docker CI only).

## Prevention

- For any create with a unique/PK constraint, use `onConflictDoNothing()`/`ON CONFLICT` + empty-returning → typed 409. Never gate uniqueness on a read-then-write SELECT. A real two-racer integration test (testcontainer) is the contract for the 409-vs-500 path — unit fakes cannot prove it.
- On every HTTP wire change, re-derive `docs/api/*.openapi.yaml` from the handler (shape, field names, status codes) AND confirm every state-changing route has a path entry. A response shape of `{ok,...}` (not the stored row) is the secure default.
- Auth gate (`requireGlobalRole`/`requireActiveSession`) is the first `await` in every handler, before body parse.
- Server-authoritative fields (readiness, timestamps, ids) must not appear in request Zod schemas; `.strict()` then rejects them.
- Resume must use the ACP `session/resume` protocol call; if unsupported, fail loud (CHECKPOINT) — never `session/new` (empty context).
