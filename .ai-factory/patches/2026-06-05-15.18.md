# Patch: M25/M24 Review Finding Fixes

**Date:** 2026-06-05 15:18
**Scope:** Catalog auth and draft concurrency, scheduler tick contract/status, DB documentation drift

## Problem

Review findings identified three contract violations:

- Catalog API project lookup happened before active-session authorization, so unauthenticated callers could trigger project existence probes.
- Authored draft updates checked `draft_version` before mutation but did not guard the write itself, leaving concurrent same-version edits able to fall through to raw database conflicts.
- Scheduler tick responses drifted from the OpenAPI shape (`jobs` vs `attempts`) and returned `200` for skipped claimed work.

The scheduler and authored catalog ERD documentation also contained stale columns/index names that no longer matched the Drizzle schema.

## Root Cause

- Shared catalog route authorization resolved the project slug before any session gate.
- `updateAuthoredDraft` used a read-time version assertion but did not make the capability update a compare-and-swap operation.
- The scheduler service type and route tests evolved with an internal `jobs` field while the API spec documented the public field as `attempts`.
- Documentation was not updated from the final migration/schema shape after implementation.

## Fix

- Added an active-session gate before catalog project lookup and regression coverage that asserts unauthenticated requests do not touch the DB.
- Added a CAS `UPDATE authored_capabilities ... WHERE draft_version = expectedDraftVersion RETURNING id` before revision-row mutation; stale concurrent writers now receive typed `CONFLICT`.
- Added integration regression coverage for two same-version draft edits racing to exactly one success and one typed conflict.
- Renamed scheduler tick summary output to `attempts` and changed the cron tick route to return `207` for failed or skipped claimed attempts.
- Updated OpenAPI, scheduler ERD, global ERD, and capabilities ERD notes to match implemented schemas and DTOs.

## Prevention

- Auth-sensitive route helpers must perform the session gate before any resource lookup.
- Optimistic concurrency must guard the first write with `UPDATE ... WHERE version = expected RETURNING`, not only a prior read.
- Route DTO names in service types, route tests, and OpenAPI must use the same public shape.
