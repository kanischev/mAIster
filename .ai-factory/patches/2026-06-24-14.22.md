# Codex adversarial-review fixes: budget-raise retry hole + agent policy/branch-base inertness

**Date:** 2026-06-24 14.22
**Files:** web/lib/services/hitl.ts, web/lib/services/runs.ts, web/lib/agents/launch.ts (+ 4 integration test files)
**Severity:** critical

## Problem

Three confirmed findings from a Codex adversarial review of the M39 branch:

1. **(critical)** A budget-`raise` retry whose first post-commit resume was lost
   returned `{ok, idempotent}` WITHOUT resuming ‚Äî permanently stranding the run.
   The already-delivered self-heal only re-fired `scheduleResume` (flow + only
   `NeedsInput`), but `terminate_restorable` parks the run in `NeedsInputIdle`,
   and an agent run was mis-driven through `runFlow`.
2. **(high)** Agent launches resolved their execution policy with NO `base`, so
   `resolveAgentExecutionPolicy` fell back to bare `{preset:"supervised"}` ‚Äî
   dropping the project budget axis for standalone agents, and (for flow-driving
   agents) passing the agent policy as a wholesale launch override that SHADOWED
   the task/project base entirely. Configured budget ceilings silently didn't bind.
3. **(high)** `branchBase` was persisted (link + `recommended.branch_base`) and
   contractually required (ADR-106) to fork the worktree, but launch hard-coded
   `ctx.project.mainBranch` in both worktree paths AND the `workspaces` row ‚Äî the
   override was inert; an agent always branched/promoted against main.

## Root Cause

1. **Self-heal used a different (weaker) dispatcher than its own happy path.** The
   raise happy-path called the status-aware, run-kind-aware `scheduleBudgetBreachResume`;
   the retry self-heal called the flow-only `scheduleResume`. Whenever a recovery
   path and its primary path resolve the same effect through DIFFERENT mechanisms,
   they drift ‚Äî the recovery silently covers fewer states (here: no agent branch,
   no `NeedsInputIdle`).
2. **An optional `base` parameter that defaults to a SAFE-looking value hides a
   missing wire-up.** `resolveAgentExecutionPolicy({base?})` defaulting to
   `defaultExecutionPolicy()` made a call-site that forgot to pass `base` compile
   and "work" ‚Äî fail-OPEN (unlimited budget), invisible without an assertion that
   the project axis survives. The contract (agents.md) said `‚Ä¶ ‚Üí project/platform`;
   the code did `‚Ä¶ ‚Üí supervised`.
3. **Persist-then-ignore.** A field was added to the schema/API/UI (write path) and
   to the resolver contract, but the launch read path was never wired ‚Äî there was
   no test asserting the worktree's actual base branch, so the gap was invisible.

## Solution

1. Route the budget already-delivered self-heal through the SAME
   `scheduleBudgetBreachResume({db, runId, runKind, stepId})` the raise path uses
   (runKind + status branch internal), guarded on `NeedsInput | NeedsInputIdle`.
   `runRow`/`hitlRow` were already in scope ‚Äî no outcome threading needed.
2. Standalone: pass `base: resolveExecutionPolicy({projectDefault})` and wrap in
   `applyDefaultBudgetForUnattended` (mirrors the flow launch). Flow-driving: add
   `agentPolicyOverlay` to `LaunchRunInput`; `launchRun` resolves its OWN task/project
   base, then folds the agent axes on top (overlay, not override) ‚Äî inherited budget
   survives.
3. Resolve `resolvedBranchBase = link.branchBase ?? recommended.branch_base ??
   project.mainBranch` once at the top of the worktree block; validate a DECLARED
   base against `listBranches` (‚Üí PRECONDITION before any git side-effect); use it
   for `resolveBaseCommit` baseRef, `addWorktree` startPoint (both shared + standalone),
   and the `workspaces` `baseBranch`/`targetBranch`.

## Prevention

- **A recovery/retry path MUST re-use its happy-path's dispatcher, not a hand-rolled
  subset.** When reviewing a self-heal block, diff its effect call against the primary
  path's ‚Äî if they call different functions, that's a drift bug waiting to happen.
- **An optional parameter with a safe default is a fail-open trap.** When a resolver
  takes `base?` and defaults to the floor, add a test that asserts an inherited axis
  (budget) SURVIVES through every call site ‚Äî a pure-function test of the resolver
  passes vacuously and proves nothing about the call sites.
- **Every persisted/contract field needs a read-path test.** "Accepted + stored" ‚âÝ
  "used". For any override that flows to a side effect (worktree base, promote target),
  assert the OBSERVABLE outcome (the `workspaces.base_branch`, the forked `base_commit`),
  not just that the column was written.
- **Sweep the sibling, but verify reachability before fixing it.** The `hook_trip`
  handler has the same self-heal shape, but its flow runs never reach `NeedsInputIdle`
  (no `terminate_restorable`; a Running run's `keepaliveUntil` is null so the idle
  sweep skips it) ‚Äî so its flow/`NeedsInput`-only self-heal is correct. Trace the
  reachability (here: who sets the status + the sweep predicate) before expanding scope.
