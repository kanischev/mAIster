## Problem

A full `/aif-review all` of the ADR-122 branch surfaced 10 merge blockers the
prior adversarial round missed: (1) the reinforce UPDATE carried no
`status='active'` re-check, so a racing decay sweep silently consumed the
domain event while the lesson landed on an expired, invisible row; (2)
schema-valid-but-empty distill output wedged the harvest cursor in a paid
re-distill loop; (3) the F1 fix's own enable-check sat OUTSIDE the ambient
try — a transient DB error on a decorative read crashed opted-in runs; (4) a
non-retryable provider 4xx permanently stalled a project's reindex job on the
same item every tick; (5) a generation switch was a 3-step non-atomic
sequence, and dims > 2000 passed validation then failed HNSW DDL after the
settings commit with no recovery path; (6) the KNN CTE took the global top-K
across ALL projects/statuses before filtering — cross-project recall
starvation; (7) `content`/`tags`/`q` were unbounded on paid embedding paths
while sibling `title` was capped; (8) task/review text entered the distiller
prompt unfenced and ambient injected without a confidence floor —
`state_fact` never expires → durable cross-run prompt injection; (9)
`brainContext`/`canReadBrain`/`canWriteBrain` were absent from
`additionalProperties:false` OpenAPI schemas; (10) a wide band of
Implemented-tagged docs described behavior that did not exist
(brain_index_jobs FSM, state_fact supersede, snapshot provenance, @chonkiejs
citation, "lineage 0001" cites), and 0002/consumer/sweep/column fanout was
missing from sibling analytics. Plus: the brain migration journal was outside
the journal-integrity lint AND 0002's `when` was future-dated (the known
silent-skip class); the branch's own settings-page change had broken a
pre-existing page-contract test nobody ran.

## Root Cause

Recurring shapes, not one bug: **a guard enforced at one layer but not at
every path to the same effect** (enable-predicate copy-pasted ×3 instead of
living inside `recall()`/`retain()`; the reinforce predicate checked in the
SELECT but not re-checked in the UPDATE under READ COMMITTED); **validation
at the wrong boundary** (distill validated JSON shape, not retain's actual
content contract; dims validated as "positive int", not against the pgvector
2000 HNSW cap the DDL enforces); **partial-coverage waves** (title capped,
siblings not; main journal linted, brain journal not; agent launch path
tested, flow path waived with a "compile error catches it" argument that only
covers typos, not deletions); and **"docs follow later"** — five phases of
implementation with the analytics/contract reconciliation deferred to a
final flip commit that under-swept.

## Solution

Code: kind-scoped near-dup + `status='active' RETURNING` with fall-through to
insert; distill-side empty/oversize rejection + `max_tokens` + fenced
untrusted sections; ONE kill-switch predicate in `guard.ts` enforced inside
`recall()`/`retain()` with ambient's check moved inside its try + 60s
negative cache + 0.4 ambient confidence floor + one-snapshot-per
(run,query_hash,model); deterministic 4xx → `CONFIG` (vs
`EMBEDDING_UNAVAILABLE` = timeout/429/5xx/network/malformed-200) + `data[].
index` ordering + finiteness checks; reindex marks `failed` on per-item
CONFIG (error recorded in `resumable_cursor`), skips disabled projects, and
recovery = reconcile-enqueue on every settings save AND project enable (also
fixes re-enable stranding); `updateBrainSettings` = one transaction with
`FOR UPDATE` + dims ≤ 2000 + post-commit index assert (sweep re-asserts as
belt); KNN scoped inside the CTE by project/active/unexpired; caps
(content ≤ 32k, q ≤ 2k, tags ≤ 10×64) mirrored route-Zod ↔ MCP TOOL_SPECS ↔
both OpenAPI; snapshot `run_id` wired from token `boundRunId`; terminal
run.json keeps the last brain projection + prompt-pointer untrusted caveat;
journal lint parameterized over BOTH lineages + not-in-future guard + 0002
redated; `db:check` covers both lineages; 21 docs files reconciled
(validate:docs + redocly green). Every blocker got a regression test;
full web unit 5655/5655, mcp 58/58, all brain integration suites green on
real pgvector.

## Prevention

- **A kill switch/authz gate belongs INSIDE the service that performs the
  gated operation**, not in each caller — callers keep their checks only for
  ordering/UX. Copy-pasting the predicate is how the F1 class recurs.
- **An UPDATE that acts on a row selected earlier must re-assert the
  selecting predicate in its WHERE (+ RETURNING) whenever another writer can
  flip it** — inside one transaction under READ COMMITTED the SELECT's view
  is not the UPDATE's view unless the lock covers both writers.
- **Validate producer output against the CONSUMER's contract** (distill vs
  retain), and validate config against the ENGINE's real limits (pgvector
  2000-dim HNSW cap), not against generic shapes.
- **When capping/validating one field, sweep every sibling on the same sink
  in the same pass** — and the "sink" includes paid APIs (embedding tokens),
  not just fs/SQL/argv.
- **A "compile error catches it" test waiver only covers typos.** Deleting a
  nullable-field passthrough compiles; only a test pins it.
- **Run the FULL unit project before calling a branch green** — the broken
  settings page-contract test was invisible to every feature-scoped suite
  run across five phases.
- **Hand-authored migration journals need the same lint as generated ones,
  and `when` must never be in the future** — a later `Date.now()` migration
  would be silently skipped.
- Close-out sweep greps recorded: enable-predicate definitions = 1
  (guard.ts); vector-literal homes = codec.ts + the annotated customType
  driver; zero `// mirrors X` confessions in brain; `assertBrainProvisioned`
  present at all 3 route surfaces; caps present in route Zod + TOOL_SPECS +
  web/external OpenAPI.
