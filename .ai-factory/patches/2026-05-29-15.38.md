# Branch-wide /aif-review fixes: doc drift, dead links, portfolio fan-out, stream auth-first

**Date:** 2026-05-29 15:38
**Files:** docs/architecture.md, docs/system-analytics/projects.md, docs/api/web.openapi.yaml, web/lib/queries/portfolio.ts, web/app/api/runs/[runId]/stream/route.ts, web/lib/config.schema.ts, web/app/api/runs/route.ts, web/app/api/runs/[runId]/hitl/[hitlRequestId]/respond/route.ts, web/components/chrome/left-rail.tsx, web/components/portfolio/needs-you-strip.tsx, web/components/board/hitl-inbox.tsx, web/components/chrome/status-bar.tsx, web/app/(auth)/layout.tsx, web/app/(app)/page.tsx, web/app/(app)/projects/[slug]/page.tsx, web/app/dev/run-stream/[runId]/page.tsx, web/messages/{en,ru}.json, web/lib/__tests__/config.schema.test.ts
**Severity:** medium

## Problem

A whole-branch `/aif-review` of the M9 milestone (5 parallel review slices)
found no code-critical or security blockers, but a spread of real defects:

1. **Doc drift (×2, treated as must-fix per the docs' own "code wins" policy):**
   - `architecture.md` persona note said "no auth, no RBAC on POC" — contradicted
     by shipped M9 auth/RBAC (and by `ROADMAP.md`).
   - `system-analytics/projects.md` register sequence diagram emitted
     `409 PRECONDITION` for the slug/repo_path collision, but the route throws
     `CONFLICT` — and the same doc's Edge-cases section already said `CONFLICT`.
2. **Dead nav/links → 404 on click** (same class as a prior shipped bug): the
   left-rail rendered `inbox/flows/agents/mcps/settings` as real `<Link>`s to
   routes that do not exist; the portfolio Needs-You strip and the board HITL
   inbox linked to a nonexistent `/inbox`; the status bar + auth layout linked
   to a nonexistent `/docs` route.
3. **`portfolio.ts` workspace fan-out:** the recent-merge and Needs-You queries
   omitted the `isNull(workspaces.removedAt)` filter that the active-workspace
   query uses. A run with a stale + live workspace (the retry case, since
   `workspaces.runId` is not unique) double-counts merge cards and inflates
   `pendingHitlCount` / `totalNeeds`.
4. **`GET /api/runs/[runId]/stream` info-leak / ordering:** the run lookup ran
   before the RBAC gate, so unauthenticated / must-change callers could probe
   run existence (404-vs-403) on a state-bearing endpoint.
5. **Contract + convention gaps:** OpenAPI `/api/runs` omitted the `400 CONFIG`
   (malformed body) the handler emits; `repo_path` was an open `z.string()`
   (sink-guarded but not parse-validated); two `db` casts lacked the project's
   `// FIXME(any):` annotation; hardcoded English in the LiveTicker, the
   Needs-You "paused … ago", and the board breadcrumb.

## Root Cause

- **Doc drift:** docs authored before the code stabilized and never re-derived
  from the implementation; the architecture persona note predates M9.
- **Dead links:** the UI was built to the full designed nav while the routes for
  deferred features were intentionally not built — the links were never gated to
  match the shipped surface. A link to a route is a contract; if the route is
  absent the link must be non-navigating.
- **Fan-out:** the `removedAt` filter was applied to the active query but not
  copy-pasted to the sibling merge/need queries that join the same non-unique
  `workspaces.runId` — partial coverage of a one-to-many join guard.
- **Stream ordering:** the read route loaded the resource before authenticating
  (the same auth-first gap fixed earlier in the write routes, not yet applied to
  the SSE read route).

## Solution

1. Rewrote the `architecture.md` persona note to state credentials auth +
   global/project RBAC shipped in M9; fixed `projects.md` to `409 CONFLICT`.
2. Dead-destination nav/links made **non-navigating** rather than stubbing
   deferred features: left-rail deferred sections render as muted, `aria-disabled`
   "coming soon" items (matching the existing cursor/aider "coming soon" agent-chip
   precedent in the same file); removed the redundant `/inbox` affordances from the
   portfolio strip and board inbox (items are individually linked); pointed `/docs`
   at the GitHub `docs/` tree (external, matches the `↗` affordance).
3. Added `isNull(workspaces.removedAt)` to the recent-merge and Needs-You queries.
4. Added `requireActiveSession()` before `loadRunLite` in the stream route, with
   auth-error → HTTP mapping; project-role check still runs after server-deriving
   `projectId` from the run row.
5. Added `400 CONFIG` to the OpenAPI `/api/runs` responses; added a path-shape
   `.refine()` to `repo_path` (absolute, no `..`) mirroring the `dir` body rule;
   annotated the two `db` casts with `// FIXME(any):`; moved the LiveTicker text to
   a `t.rich("ticker", …)` message, the "paused … ago" text to `portfolio.pausedAgo`,
   and the breadcrumb to `nav.crumbProjects`. Added `nav.comingSoon`. EN/RU parity
   preserved (214/214 keys). Guarded the dev-only `/dev/run-stream/[runId]` fixture
   with `notFound()` in production.

Two open review *questions* were investigated and found to be non-bugs:
`installFlowPlugin` is always called without a `workspaceRoot` (defaults to
`process.cwd()`, exactly what the registration compensation cleans), and human
HITL rows always carry the loaded form `schema`, so `assertHitlResponse` cannot
falsely reject. One SUGGESTION (parallelizing two independent `activity.ts`
queries) was deliberately skipped: a pure micro-latency nicety not worth the
multi-line reorder churn.

## Prevention

- **A link to a route is a contract.** During review, grep the app tree for
  `href="/x"` targets and confirm a matching `page.tsx`; if the feature is a
  deferred, render the affordance non-navigating ("coming soon") rather than as a
  live link. Reuse the existing disabled-affordance precedent instead of stubbing
  routes for unbuilt features.
- **One-to-many join guards must cover every sibling query.** When a filter
  (`isNull(removedAt)`) dedups a non-unique join, apply it to *all* queries over
  that join, not just the first — the same "sweep siblings" rule as security
  refinements.
- **Auth-first applies to read routes too**, not only state-changing ones: gate
  authentication before any resource lookup so existence cannot be probed.
- **Re-derive doc contracts from code, never trust the doc's own status tag.**
  A shipped milestone must reconcile persona/architecture prose, sequence-diagram
  status codes, and the OpenAPI error set against the implementation.
- Tests: added negative `config.schema` cases (relative `repo_path` and `..`
  traversal both rejected). Recommended follow-ups (Docker-backed integration):
  a stream-route auth-first regression and a portfolio `removedAt` fan-out
  no-double-count assertion.
