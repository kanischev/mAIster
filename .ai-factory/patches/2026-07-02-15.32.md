# Project Brain — four robustness holes under disablement / degraded deps / redelivery

**Date:** 2026-07-02 15:32
**Files:** web/lib/brain/ambient.ts, web/lib/brain/openai-compatible.ts, web/lib/brain/retain.ts, web/lib/brain/reindex.ts, web/lib/domain-events/memory-harvest.ts, web/lib/brain/schema.ts, web/lib/db/brain-migrations/0002_brain_review_fixes.sql
**Severity:** high

## Problem

An adversarial review of the new Project Brain (ADR-122) surfaced four defects that
unit/integration tests missed because they all exercise the *happy* path:

1. Ambient recall injected memory into a run even after an admin disabled the
   project's Brain (`projects.brain_enabled=false`).
2. The embedding client had no request deadline — a provider that accepts the
   connection then stalls hung the recall/retain routes and the harvest/reindex
   sweeps indefinitely.
3. The reindex worker could write duplicate `brain_embeddings` for the same
   generation under overlapping sweeps / multi-instance (no uniqueness guard).
4. Harvest redelivery double-counted confidence/TTL on the *reinforce* path — the
   idempotency check only covered the *insert* path.

## Root Cause

A single theme: **a guard/contract was enforced on the primary path but not on every
path that reaches the same effect.**

1. Enablement was checked in the harvest consumer and the explicit ext route, but
   ambient (`getAmbientBrainProjection`) is a *second* recall consumer and only
   checked `runs.brain_context` — a launch-time boolean persisted independently of
   the live `brain_enabled` switch. "Disabled → no-op" was true for one consumer, not all.
2. The "bounded retry on timeout" contract handled HTTP status + thrown network
   errors, but "timeout" was assumed to come from `fetch`'s implicit behavior. A
   stall produces *neither* a status nor a throw, so nothing fired. Missing an
   explicit `AbortSignal` deadline.
3. Idempotency was enforced by a pre-`INSERT` `NOT EXISTS` filter — a check-then-act
   race with no DB constraint behind it. Two workers both pass the filter, both insert.
4. The idempotency key (`brain_items.source_domain_event_id`) is only written when
   retain *inserts*. The reinforce/exact-dup branches produce no such row, so a
   redelivered event that reinforced was invisible to the "already harvested?" check.

## Solution

- F1: `getAmbientBrainProjection` re-queries `projects.brain_enabled` before
  embedding/recall; returns `undefined` when false.
- F2: per-attempt `signal: AbortSignal.timeout(timeoutMs)` (default 30s); the abort
  is caught in the existing transient branch → retried → `EMBEDDING_UNAVAILABLE`.
- F3: `UNIQUE (item_id, split_ordinal, embedding_model, embedding_dimensions)` +
  `ON CONFLICT DO NOTHING` in both insert sites (retain + reindex) — the DB is the
  real guard, the filter is just an optimization.
- F4: a `brain_harvested_events (project_id, domain_event_id)` ledger written in
  retain's transaction for EVERY outcome; the harvest pre-check queries it.

## Prevention

- When a feature has a **disable switch or an authorization gate, enumerate EVERY
  consumer** of the gated resource and assert the gate in each — not just the one the
  happy-path test drives. Ambient vs explicit recall were two doors to one room.
- **A check-then-act idempotency filter needs a DB uniqueness constraint behind it**,
  or it is only correct single-threaded. Ask "what if two workers / two instances run
  this at once?" for every sweep/worker.
- **Idempotency keyed on a side effect only covers the branch that produces it.** If a
  handler has multiple outcomes (insert / update / no-op), record consumption in a
  dedicated ledger written on ALL branches, in the same transaction as the effect.
- **Any outbound `fetch` in a request or sweep path needs an explicit
  `AbortSignal` deadline** — do not rely on the runtime's implicit timeout; a stall is
  neither a status code nor a throw.
- Test the *degraded* path: disabled-project, never-resolving fetch, concurrent
  double-fire, and event redelivery-after-reinforce are the four that caught these.
