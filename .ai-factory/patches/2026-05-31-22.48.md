# Scratch run contract verification cleanup

**Date:** 2026-05-31 22:48
**Files:** docs/api/async/web-runs.asyncapi.yaml, docs/api/web.openapi.yaml, docs/error-taxonomy.md, docs/system-analytics/hitl.md, docs/system-analytics/runs.md, docs/system-analytics/scratch-runs.md, .ai-factory/DESCRIPTION.md, CLAUDE.md, web/CLAUDE.md, web/app/api/runs/[runId]/hitl/[hitlRequestId]/respond/__tests__/route.test.ts
**Severity:** high

## Problem

The scratch-runs verification gate found that public contracts and project
context overstated or contradicted the implementation. AsyncAPI documented
`scratch.dialog_*` SSE payloads that were not emitted, HITL timeout docs said
all runs transitioned to `Failed` while scratch timeout code transitions to
`Crashed`, and M8 checkpoint/resume was described as both designed and
implemented in different context files. Read-only lint also failed on
formatting/import-order warnings.

## Root Cause

The feature evolved through implementation and follow-up fixes faster than the
contract docs were normalized. Some docs preserved planned "Designed" wording
after code landed, while the scratch dialog implementation chose a simpler
SSE-as-invalidation model instead of emitting scratch-specific SSE payloads.
The timeout state split was fixed in code but not covered by a scratch-specific
regression test, allowing stale docs to survive review.

## Solution

Updated the AsyncAPI contract to document the implemented session-event stream
and detail-refresh pattern. Updated Web OpenAPI, error taxonomy, and system
analytics to distinguish flow `HITL_TIMEOUT -> Failed` from scratch
`HITL_TIMEOUT -> Crashed`. Normalized M8 checkpoint/resume wording in project
context, added scratch workspaces to the current product description, and added
a unit regression for scratch permission timeout. Ran ESLint auto-fix to clear
the warning gate.

## Prevention

- When a verifier flags contract drift, audit every public surface for the same
  concept: OpenAPI, AsyncAPI, error taxonomy, system analytics, and AI context.
- If code intentionally implements a simpler event model than a plan proposed,
  remove the unimplemented contract variants instead of leaving them as
  "Designed" inside implemented docs.
- Add a run-kind-specific regression whenever the same route maps one error code
  to different persisted statuses.
