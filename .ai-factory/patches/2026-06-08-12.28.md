# Authored Package Contract Documentation Sync

**Date:** 2026-06-08 12:28
**Files:** `docs/configuration.md`, `docs/system-analytics/capability-catalog.md`
**Severity:** medium

## Problem

Two authored-package contract documents still described package file validation
as safe relative paths with no duplicates or binary payloads. The current
validator also rejects duplicate normalized paths, file-vs-directory path
collisions, and any file content that is not valid UTF-8 text.

## Root Cause

The implementation contract was tightened, and `docs/flow-dsl.md` was updated,
but the remaining package contract references were not synchronized.

## Solution

Updated the configuration and capability catalog docs to state the full current
validation behavior: safe relative paths only, no `..` segments, no duplicate
normalized paths, no file-vs-directory collisions, valid UTF-8 text content, and
binary payload refusal.

## Prevention

When authored-package validation changes, update all package contract docs in
the same pass and compare them against `docs/flow-dsl.md` before review.
