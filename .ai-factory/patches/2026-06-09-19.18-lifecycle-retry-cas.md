# Patch: Workbench lifecycle retry/CAS/precondition fixes

**Date:** 2026-06-09
**Skill:** aif-fix
**Scope:** M27 workbench lifecycle replay on current `main`

## Problem

Review found three blocking lifecycle correctness issues:

- Handoff branch retry could collide with the local branch created by the first
  failed push attempt.
- Drop could mark a run `Abandoned` from stale context after another actor moved
  the run to a different status.
- Export accepted a syntactically valid but missing remote until after lifecycle
  preflight work had started.

The lifecycle work was also based on an older detached HEAD and had to be
replayed onto current `main`.

## Root Cause

- Handoff preflight used boolean branch-exists checks, so it could not
  distinguish "same branch at the same workbench HEAD" from a real collision.
- Drop persistence updated run state by id only, without carrying the expected
  run status from context into the DB transaction.
- Export validated remote syntax but did not require the selected remote to be
  present before the lifecycle claim.
- The worktree HEAD was behind `main`, and dirty overlapping files prevented a
  direct fast-forward without replay.

## Fix

- Added exact local/remote branch-head helpers in `web/lib/worktree.ts`.
- Made handoff branch creation idempotent for same-head local/remote refs:
  retry reuses a same-head local ref after transient push failure, and a
  same-head remote ref completes a lost-ack retry.
- Kept different-head local/remote refs as `CONFLICT`.
- Added export remote existence preflight before status checks and lifecycle
  claims.
- Added `expectedRunStatus` to drop persistence and locked the run row in the
  transaction before recording removal or status updates.
- Added a stale-status DB regression for `recordDrop`.
- Replayed the lifecycle work onto `main` commit `cccd85c2` via stash +
  fast-forward + conflict resolution.
- Moved run-detail lifecycle actions into the new current-main persistent
  `runs/[runId]/layout.tsx` header.

## Prevention

- Branch lifecycle retries must compare exact refs, not boolean existence.
- Lifecycle operations that remove or preserve external state must carry the
  expected DB state into transactional persistence.
- Body-controlled git remotes must be checked before lifecycle claims and git
  side effects.
- After replaying old work onto `main`, regenerate Next route types before
  trusting `tsc`.
