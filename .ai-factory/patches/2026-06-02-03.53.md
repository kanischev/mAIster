# M12 evidence-gating + artifact-integrity fixes (adversarial-review findings)

**Date:** 2026-06-02 03:53
**Files:** web/lib/flows/graph/runner-graph.ts, web/lib/flows/graph/evidence-readiness.ts, web/lib/queries/board.ts, web/lib/config.ts (+ tests: review-refusal, rework-readiness, immutable-git-payload, config-artifacts)
**Severity:** high

## Problem

A Codex adversarial review of the M12 (typed artifacts / evidence graph)
branch surfaced four real defects, all verified against the as-built contract
in `docs/system-analytics/artifacts.md`:

- **F1 (high)** — `assertEvidenceReady("review")` had **zero production
  callers**. The documented review-refusal flow (artifacts.md flowchart +
  Expectations) requires it to gate the review-approval, but the only evidence
  check at review was each node's own `pre_finish` `artifact_required` gate,
  which checks only its declared `inputArtifacts`. A flow declaring
  `requiredFor:[review]` artifacts but no matching gate could be approved with
  stale/missing required evidence (false readiness).
- **F2 (high)** — `assertEvidenceReady` Check 2 and board `mergeBlocked` both
  scanned **every** `artifact_required` gate row (`status IN ('failed','stale')`).
  `markDownstreamStale` flips a prior attempt's passed gate to `stale` and
  leaves it; the re-run writes a fresh `passed` row on a new attempt. The old
  `stale` row then falsely blocked readiness + kept the board badge on forever.
- **F3 (medium)** — the runner recorded `commit_set` artifacts with
  `locator.baseRef = workspace.branch` (a branch name) while `headRef` resolved
  to the branch-tip SHA. The payload route renders `git log baseRef..headRef`,
  so `branch..branchTipSha` collapsed to empty/wrong output.
- **F4 (medium)** — `validateArtifacts` rejected only absolute / `..` paths.
  An empty `""` (or `.`) `produces[].path` passed, joined to the run directory,
  `access()` accepted it, and the payload route then 500'd reading a directory
  as a file.

## Root Cause

- **F1**: the guard function was built + unit-tested but never wired into the
  runner. The team assumed the `pre_finish` gate alone enforced the contract;
  it does not — the gate is per-its-own-`inputArtifacts`, while the
  `requiredFor:[review]` rule is run-global (per-def-current).
- **F2**: the artifact-def path was fixed for rework (per-def-current via
  `supersedePrior` retiring ALL prior rows), but the **gate** path never got an
  equivalent. Gate rows are append-only per node attempt, so a stale row
  outlives its re-produced replacement. Both consumers used an any-row scan.
- **F3**: the `commit_set` branch was copy-adjacent to the `diff` branch but
  never adopted the `diff` branch's `resolveBaseRef` (merge-base) call — it kept
  the branch name as the base. The takeover-return route does it correctly; only
  the runner-inline path drifted.
- **F4**: the validation predicate was written as a deny-list of two bad shapes
  (absolute, `..`) rather than an allow-list of "non-empty relative file path".
  Empty string is neither absolute nor `..`, so it slipped through.

## Solution

- **F1**: at the human-review node's non-rework, terminal (`resolveTransition
  === null`) finish — the run's review-approval chokepoint — call
  `assertEvidenceReady(runId, "review")` before `markNodeSucceeded`. On
  `!ready`, refuse exactly like a blocking-gate failure (`markNodeFailed`,
  PRECONDITION, run → Failed) and log the documented `review refusal` pino line.
  Gated on `artifactEnforcementActive` (engine ≥ 1.2.0); legacy flows are
  vacuously ready, so behavior is unchanged for them.
- **F2**: filter blocking gates to the **latest attempt per node** in BOTH
  `evidence-readiness.ts` (reusing `getNodeAttemptsForRun` + `latestAttemptByNode`)
  and `board.ts` (batched `node_attempts` query). Keyed on node-attempt
  lineage, not gate id, so it is immune to gate-id reuse across nodes.
- **F3**: resolve `commit_set` `baseRef` via `resolveBaseRef({branch, mainBranch:
  "main"})` (the merge-base), mirroring the `diff` branch; branch-name fallback
  only when git is unavailable (synthetic-flow test envs).
- **F4**: `validateArtifacts` now rejects empty / `.` / trailing-slash paths
  (allow-list semantics); the runner uses `stat().isFile()` instead of
  `access()` as defense-in-depth so a directory path is never recorded as a
  file artifact.

Each fix has a RED→GREEN regression test; F2 and F3 were empirically
RED-verified by reverting the fix and observing the new test fail.

## Prevention

- **Wire-up audit for guard functions**: a guard that exists + is unit-tested
  but has zero production callers is a silent contract gap. When a doc flowchart
  / Expectations bullet names a guard (`assertEvidenceReady`), grep for its
  call sites — "shipped + tested" ≠ "enforced".
- **Fan-out a staleness/supersession fix to EVERY evidence kind**: the
  per-def-current fix for artifacts must be mirrored for gates (and any future
  evidence row type). A rework/re-run leaves append-only history; readers MUST
  scope to the latest attempt, never any-row.
- **Sibling-sweep adjacent record sites**: when one branch of a multi-kind
  recorder (diff/commit_set/file) resolves a value correctly (`resolveBaseRef`),
  verify every sibling branch does too. Copy-adjacency hides drift.
- **Boundary validation is an allow-list, never a deny-list**: validate
  "is a non-empty relative file path", not "is not absolute and not `..`".
  Deny-lists silently admit the unenumerated case (empty string, `.`).
