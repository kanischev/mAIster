# Branch-divergence collisions (ADR + migration number) + frozen-spec drift — M27

**Date:** 2026-06-09 18:47
**Files:** web/lib/db/migrations/003[2-6]_m27_*.sql (renamed from 003[1-5]), web/lib/db/migrations/meta/_journal.json, docs/decisions.md + 27 ADR-referencing docs/code files, .ai-factory/specs/feature-m27-flow-studio-stage-1.md, docs/system-analytics/flow-studio.md, .ai-factory/plans/feature-m27-flow-studio-stage-1.md, web/lib/db/__tests__/migration-journal-integrity.test.ts (new)
**Severity:** high

## Problem

`/aif-review` of the M27 branch surfaced four merge/spec blockers:
1. The branch and `main` BOTH created `ADR-066` (different decisions) and BOTH
   created a `0031_*` migration — invisible until merge, where decisions.md and
   `_journal.json` collide and `db:migrate` breaks on a duplicate index.
2. Both sides rewrote the same authored-flow editor page (React Flow graph vs
   CodeMirror) — a guaranteed semantic conflict.
3. The frozen SDD §7.1.7 states `version_binding=latest` MUST resolve the
   newest published revision, but `resolveEffectiveFlowRevision` ignores the
   binding (returns the enabled pointer for both) — and `flow-studio.md` + the
   §9 test-matrix repeated the unimplemented claim while the actual test
   asserted the no-op.

## Root Cause

- **Parallel-branch number allocation.** ADR numbers and Drizzle migration
  indices are *globally sequential*. Two branches cut from the same base each
  grab "the next number" independently; nothing on either branch detects the
  clash because each branch's own gate is green. The clash only exists relative
  to the *other* branch, so a single-branch review never sees it.
- **Two-dot diff hid it.** `git diff main..HEAD` compares tip-to-tip, so main's
  9 newer commits showed up as branch "deletions" (shiki, tokens, etc.),
  drowning the real signal. The branch's true changes are `main...HEAD`
  (three-dot, = merge-base..HEAD).
- **Frozen-spec drift.** A deliberate Stage-1 simplification (collapse
  `latest`→enabled pointer) was documented only in a code docstring, never
  reconciled into the FROZEN spec / analytics doc / test-matrix — so three
  artifacts claimed behavior the code intentionally lacked.

## Solution

- **Reserve-the-other-branch's-slot renumber (branch-local, no merge).** ADRs
  `066→067…069→070` (main keeps 066); migrations `0031→0032…0035→0036` (main
  keeps idx 31). The branch journal carries an intentional **idx-31 gap** —
  safe because drizzle's `readMigrationFiles` ignores `idx` entirely (iterates
  `entries` by array order, resolves `${tag}.sql`, dedups by `when`). On merge,
  main's `0031`/`066` slot into the reserved gaps → contiguous.
- **ADR renumber tooling:** ordered highest-first substitution
  (`069→070; 068→069; 067→068; 066→067`, both `ADR-` and anchor `adr-`) so no
  cascade. Verified by the repo's own `validate-docs-adr-anchors.mjs`
  (182 anchor links resolved).
- **Spec amendment, not silent drift:** added an Amendment-log entry + tagged
  the newest-published path `(Phase 2 — deferred)` in spec §4.3/§7.1.7/§9 and
  `flow-studio.md`; the test already encoded Stage-1, so the §9 matrix was
  corrected to match the test (not vice-versa).
- **Editor:** recorded a merge directive (keep both — CodeMirror becomes the
  YAML-tab editor inside the branch's `flow-editor-tabs`), since a cross-branch
  conflict cannot be resolved without merging.

## Prevention

- **Always establish branch scope before reviewing/fixing:** run
  `git merge-base main HEAD` and review `main...HEAD` (three-dot). If
  `merge-base != main tip`, main diverged — two-dot stats are contaminated.
- **Check for number collisions proactively:** before merge, diff the new ADR
  numbers and migration `idx`/`tag` of the branch against `main`
  (`git show main:docs/decisions.md`, `git show main:.../meta/_journal.json`).
  A "no duplicate ADR number" assertion could augment
  `validate-docs-adr-anchors.mjs`.
- **Migration-journal bijection test added** (`migration-journal-integrity.test.ts`):
  every tag↔file, unique idx, unique tag — catches orphan/dup/renumber mistakes
  without a DB.
- **macOS BSD-grep gotcha (cost a wasted pass):** `grep -rlZE … | xargs -0 perl`
  malfunctioned on BSD grep (spurious match set + "File name too long", perl
  no-op). For glob-unsafe paths like `app/api/[slug]/…`, prefer
  `grep -rlE … | while IFS= read -r f; do perl -pi -e '…' "$f"; done`
  (quoted `"$f"` prevents `[slug]` shell-glob; one perl per file).
- **A frozen-spec deviation is a spec amendment, never a code-comment-only
  decision.** When code intentionally diverges from a MUST, amend the spec +
  every doc + the test-matrix in the same change, and ensure the named test
  asserts the new contract.
