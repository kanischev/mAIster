# ADR-116 composition editor: non-critical /aif-review findings

**Date:** 2026-06-29 01:37
**Files:** web/components/studio/files-manager.tsx, web/components/studio/package-composition.tsx, web/components/studio/__tests__/files-manager.dom.test.ts, web/components/studio/__tests__/package-composition.dom.test.ts, web/messages/en.json, web/messages/ru.json, docs/decisions.md, docs/system-analytics/local-packages.md, docs/system-analytics/flow-studio.md, .ai-factory/plans/feature-local-package-tabbed-editor.md
**Severity:** medium

## Problem

A `/aif-review` of the ADR-116 local-package composition editor branch surfaced
non-critical findings (the critical path-traversal/rename-desync was already
patched in `2026-06-29-01.03.md`). The user scoped `/aif-fix` to "the
non-critical findings", which per project rule means address ALL of them:

1. **(H2) Drag-only file move silently broken.** `FilesManager` drop targets
   read the dragged path from a `useRef` set in `onDragStart`. A ref mutation
   does not re-render, so on a drag with no prior click-select the targets stayed
   `disabled` (`source === null && dragged.current === null`) and the browser
   suppresses `onDrop` on a disabled button — the advertised drag-and-drop did
   nothing. Untested (only the click path had a test).
2. **(M1) `movePathInDraft`'s tested folder-move branch was unreachable** from the
   UI — folders rendered only as drop targets, never as drag sources.
3. **(H1) Blank-capability subagent scaffold showed the generic `errorInvalid`**,
   not a hint that a capability is required.
4. **(LOW) Footguns:** an MCP descriptor whose mcp-labels were absent fell through
   to the *frontmatter* editor (would mangle YAML on save); the kind `<select>`
   was `aria-label`ed "Add"; `FRONTMATTER_KIND_BY_COMPOSITION` carried dead
   `mcps`/`flows`/`skills` placeholder entries.
5. **(spec) Docs drift:** the headline `PackageSource` abstraction was documented
   as `{ manifest, inventory, listFiles, readFile }` but the code is
   `{ logLabel, spec, inventory, listFiles, readFile, loadFlow }`; two docs/plan
   blocks were stale (present-progressive tense under an "Implemented" tag;
   `flows/<name>.yaml` instead of the directory form `flows/<name>/flow.yaml`).

## Root Cause

H2 is the classic "mutable ref read during render" bug: a value that must drive
conditional rendering (`disabled`) was stored in a ref, so the render that should
have enabled the targets never happened. The footguns are silent-fallback
branches that mask a programming invariant instead of asserting it. The doc drift
is the usual "the abstraction was narrowed in code (`manifest` → `spec`,
`loadFlow` added for confinement) but the prose described the rejected shape".

## Solution

- **H2:** moved `dragged` from `useRef` to `useState` (+ `onDragEnd` cleanup) so a
  drag re-renders and the targets leave `disabled`; `onDrop` then fires.
- **M1:** made non-root folder targets `draggable` with `onDragStart`, exercising
  the existing tested folder-move (prefix-rewrite) branch.
- **H1:** a client-side guard returns `composition.create.errorCapabilityRequired`
  when `kind === "subagent"` and the capability is blank (new EN/RU key).
- **Footguns:** an MCP descriptor path now dispatches to `McpTemplateEditor`
  unconditionally (`filesLabels.mcp!` with a WHY comment — the studio/edit page
  always builds labels with `includeMcp=true`), never the frontmatter editor;
  the kind select uses a new `composition.create.kindLabel` ("Kind"/"Тип");
  `FRONTMATTER_KIND_BY_COMPOSITION` narrowed to a `Partial` of the three real
  frontmatter kinds with a `?? "rule"` fallback at the single index site.
- **Docs:** corrected the `PackageSource` shape in `decisions.md` (ADR-116 §1) and
  `local-packages.md`, fixed the two tense blocks and the two stale flow-path plan
  lines.
- **Tests:** added drag-only move + folder-drag tests (`files-manager`), and
  blank-capability + rename-collision-stays-open tests (`package-composition`).

## Prevention

- A value that gates conditional rendering (a `disabled`, a `className`, a branch)
  MUST live in state, never a ref — a ref read during render is a silent
  no-re-render bug. Reserve refs for focus/measurement/event-only reads.
- Any tested lib branch with no UI caller is either dead code or a missing
  affordance — wire it or delete it; don't leave it as latent "support".
- A `cond && labels.x ? <Right/> : <Wrong/>` dispatch where `<Wrong/>` would
  corrupt data is a footgun: make the destructive branch unreachable and assert
  the invariant (`x!` + WHY) instead of silently falling through.
- When an abstraction is narrowed in code (a field renamed, a method added for a
  security boundary), grep the docs for the old shape in the same pass — the
  headline-abstraction prose is the most-read and the most-likely-stale.
- The Next 71007 "serializable props" warnings on `"use client"` files are LSP
  heuristics, INVISIBLE-distinct from real build failures; the only authority is
  `pnpm build` (verified: full build exit 0, BUILD_ID generated).
