**Severity:** medium

## Problem

A pre-merge `/aif-verify` (6 adversarial reviewers + gates) on the M34
platform-agent branch surfaced 2 HIGH doc-contract defects, 8 MED gaps, and a
LOW cluster — no runtime/security blocker, but real defects in deliverables the
milestone owns:

- `docs/error-taxonomy.md` described the M34 agent surface with the wrong ADR
  ref (089/**088**), `(all Designed)` status, the removed `scope`/`project`
  frontmatter, and a `CONFLICT → deleting an agent` endpoint that does not exist.
- `CLAUDE.md` contradicted itself on the headline default this milestone changed
  (`MAISTER_MAX_CONCURRENT_RUNS` 6 at :156 vs "global cap = 3" at :366).
- `runs.agent_workspace` (migration `0052`, the central enforcement fix) was in
  ZERO db docs; `database-schema.md` still described the dead M24 `agent_schedules`.
- RD6 + an earlier patch promised the ephemeral `-ro` checkout a "GC sweep
  backstop" — none existed (`web/lib/gc/` had no `-ro` handling; the dir has no
  `workspaces` row, so the workspace GC never sees it → permanent leak if the
  best-effort terminal removal is ever missed).
- Three plan-named deliverables had no test (webhook route bounds/scope/dedup,
  `risk_tier=destructive` refusal, dirty `repo_read` baseline refusal).
- `tasks` POST parsed the body before auth (refuting an earlier patch's
  "auth-first on ALL changed routes" claim); disabling an attachment did not
  revoke its live agent tokens (only detach did); `recover.ts` duplicated the
  scheduler cap constant + helper; ~13 orphan i18n keys from the deleted
  agent add/edit modal remained.

## Root Cause

The host-catalog→package-source rework and the today's enforcement fix
(`agent_workspace`) shipped correct code but left analytics/ADR/db-doc prose
carrying pre-rework claims, promised a GC backstop that was never coded, and
skipped fan-out test coverage. The two auth/token gaps were narrow boundary
omissions (parse-before-auth on one of N reordered routes; revoke wired to the
delete path but not the disable path).

## Solution

- Doc corrections in place (error-taxonomy ADR/status/refusal rows; CLAUDE.md
  concurrency; `agent_workspace` added to runs-domain/agents-domain/
  database-schema citing migration `0052`; `agent_schedules` summary row to the
  M34 shape; ADR-089 project-scoped manual path + the third pre-spawn runner
  refusal + the webhook idempotency-key namespace; corrected the inaccurate
  "capability profiles merged" claim — flow bindings inherit prompt body only).
- New `runEphemeralAgentGcSweep` (`web/lib/gc/ephemeral-agent-gc.ts`) reaps
  `worktreesRoot()/<slug>/*-ro` dirs whose owning agent run is terminal/absent
  via path-guarded `removeOwnedWorktree`; wired into the `system_sweep` GC
  bundle with the same best-effort error contract as its siblings. Unit-tested
  (orphan vs live vs non-`-ro`, failure-continues, missing-subtree).
- Moved the `tasks` POST body parse behind `requireActiveSession` +
  `requireProjectAction` (zod failure re-thrown as `CONFIG`/422 to preserve
  behavior). `updateAgentLink({enabled:false})` now calls
  `revokeAgentProjectTokens` in the same tx. `recover.ts` imports
  `maxConcurrentRunsCap` from `@/lib/scheduler` (deleted the local copy).
- Pruned 13 verified-0-reference i18n keys from both catalogs (line-based,
  JSON-validated, EN/RU parity preserved at 1849 keys each).
- Three integration tests added by mirroring proven harnesses: webhook route
  (403/413/422/202 + dedup-409), destructive-refusal, dirty-baseline-refusal.

## Prevention

- When a rework flips an ownership/enforcement model, grep the canonical
  contract docs (`error-taxonomy.md`, ADRs, `db/*`) for the OLD nouns
  (`scope`, `MAISTER_AGENTS_ROOT`, dropped endpoints) and the wrong status tag.
- A documented backstop ("+ GC sweep") is a deliverable: if a resource is
  created off the `workspaces`-row path, it needs its own sweep — verify with
  `git diff main lib/gc/`, don't trust the prose.
- Auth-first is a per-route invariant: after reordering N routes, grep every
  changed route for `req.json()`/`.parse(` appearing above the authz call.
- A token-revocation guarantee stated for "detach OR disable" must be wired at
  BOTH mutation sites, not just one.
