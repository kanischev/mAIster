# Address remaining 8 findings from /aif-review on M3 (defense-in-depth + cleanups)

**Date:** 2026-05-26 12:53
**Files:** supervisor/src/types.ts, supervisor/src/spawn.ts, supervisor/src/cost.ts,
supervisor/src/main.ts, supervisor/src/__tests__/{types,spawn}.test.ts,
supervisor/test/fixtures/fake-acp.mjs, web/lib/supervisor-client.ts,
web/lib/__tests__/supervisor-client.test.ts, docs/supervisor.md
**Severity:** medium (overall — individual findings range from low to medium)

## Problem

A first round of /aif-fix only addressed 3 of 11 findings from /aif-review
on M3 (the Critical path-traversal + two cosmetic cleanups). The remaining
8 findings — defense-in-depth issues plus one performance and one ergonomic
gap — were left on the table. User flagged the gap and asked me to address
them before opening the PR.

Specifically:

1. `worktreePath` in `StartSessionRequestSchema` had no validation beyond
   `.min(1)`; flows into `cwd` of `child_process.spawn` (same class as the
   `runId`/`stepId` path-traversal fix, missed in round 1).
3. The line buffer in `spawn.ts` could grow unbounded if an agent emitted
   a single line >1 GB with no `\n` — OOM risk.
4. `cost.ts` ran `JSON.parse` + two recursive tree walks on every stdout
   line, even ones that obviously had no `usage` field.
6. `streamSession` in `web/lib/supervisor-client.ts` had no `AbortSignal`
   parameter — consumers (e.g., the Next.js Route Handler bridge in M7/M9)
   would have no clean way to cancel an in-flight SSE stream.
9. `main.ts` shutdown called `process.exit(0)` immediately after the last
   log line; pino's pretty-print transport runs in a worker, so the final
   logs could be truncated.
11. The per-session `EventEmitter` in `spawn.ts` used Node's default
    `maxListeners: 10`; with cost.ts + registry's auto-subscribe + N SSE
    consumers, an unbounded number of legitimate listeners would
    accidentally trip the leak-detection warning.

Two findings (#2 env-leakage to child, #10 stderr inheritance) were
intentionally NOT addressed with code changes — see "Solution" below for
the rationale.

Two findings (#5 dead `currentEvent`, #7 redundant `204` check) were
already fixed in the first round; not re-fixed here.

## Root Cause

Three distinct patterns drove the missed-findings list:

- **Boundary-validation drift** (#1): when adding the new Zod schema for
  `runId`/`stepId` in round 1, I focused on the field that surfaced in the
  review and didn't broaden the audit to all paths-into-`path.resolve`. The
  `worktreePath` finding was in the same review report, but I didn't carry
  the lesson across the entire schema in one pass.

- **Adversarial-input cost** (#3, #4, #11): when prototyping under the
  M3 "good-faith input" assumption (the supervisor is an internal-network
  service in POC topology), defenses against malformed or oversized stdout
  were treated as Phase-2 hardening. The review pointed out that those
  defenses are cheap enough to land now and that "internal service" is a
  thin assumption.

- **Lifecycle polish** (#6, #9): consumers and shutdown semantics weren't
  load-tested under cancellation or shutdown-with-buffered-logs scenarios
  during M3 implementation. The review caught the ergonomic gaps before any
  real consumer was wired up.

## Solution

**Code changes (6 findings):**

1. **worktreePath** — added a Zod `.refine()` that requires `startsWith("/")`
   AND no `..` segment in the split path. 3 new unit tests cover relative
   paths, `..` segments, and the happy case.

3. **Line-buffer cap (`MAX_LINE_BYTES = 1 MB`)** — in `spawn.ts`, the
   stdout `data` handler now checks `buffer.length` after each chunk;
   if it exceeds 1 MB, emit a truncated line (`buffer.slice(0, MAX_LINE_BYTES)`),
   log `WARN line-buffer-overflow`, and drop the rest of the buffer.
   1 new unit test drives the path via a `--giant-bytes N` flag added to
   `fake-acp.mjs`.

4. **Cost fast-path** — `extractCost` now short-circuits when the raw
   line doesn't contain the substring `"usage"`. Cheap (single `String#includes`),
   skips ~99% of lines, no behavior change for lines that DO have usage.

6. **`AbortSignal` on `streamSession`** — added `opts.signal?: AbortSignal`
   to the signature; forwards to `fetch(url, { headers, signal: opts.signal })`.
   1 new unit test asserts the signal is forwarded.

9. **pino flush before `process.exit`** — wrap `logger.flush()` in a
   promise and await before exit. Logs no longer get truncated mid-flight.

11. **`emitter.setMaxListeners(0)`** — applied right after the per-session
    `EventEmitter` is constructed in `spawn.ts`. Removes the artificial
    10-listener cap; legitimate listeners (cost + registry auto-subscribe
    + N SSE consumers) no longer trigger Node's leak-detection warning.

**Docs change (1 finding):**

2. **Env merge semantics** — added a paragraph to `docs/supervisor.md`
   explaining that `childEnv = { ...process.env, ...executor.env }`,
   why this is the right default (allows per-executor override via
   `maister.yaml`), and how to deny supervisor-process env from reaching
   a specific executor (unset at supervisor startup; do not rely on
   `executor.env` to shadow). Explicit Phase-2 hook for allow-list mode.

**Intentional defers (with rationale):**

8. **Dual pino instances** — Fastify's `loggerInstance` parameter changes
   the inferred `FastifyInstance` type to `<…, Logger<never, boolean>, …>`,
   which then doesn't match the `FastifyInstance` (default
   `FastifyBaseLogger`) that `registerRoutes` accepts. Working around it
   requires either generic-cascading the route registration helper or
   accepting a base-logger compromise — both bigger than the marginal cost
   of two transport workers in dev. Deferred to the M7/M9 work that
   touches Fastify-side concerns anyway.

10. **Inherited stderr** — child stderr goes to supervisor stderr via
    `stdio: [..., "inherit"]`. The right fix is piping stderr to the same
    `.log` file (so it ends up in `<step>.log` alongside stdout) AND
    decomposing structured-vs-unstructured output. Both belong in M7's
    structured ACP parsing work; doing it now adds a code path that M7
    will rewrite. Flagged in the round-1 patch's "Tags" too.

**Tests added (5):**

- `types.test.ts`: 3 new tests for `worktreePath` (relative, `..` segment, valid absolute).
- `spawn.test.ts`: 1 new test for the 1 MB line-buffer cap, using a new
  `--giant-bytes N` flag on `fake-acp.mjs`.
- `supervisor-client.test.ts`: 1 new test for `AbortSignal` forwarding to `fetch`.

**Final test count:** 116 (40 supervisor unit + 9 integration + 67 web
unit). Up from 111 before this patch; up from 105 before the first round
of /aif-fix.

## Prevention

- **Audit the WHOLE schema when one field gets a security regex.** Once
  you find one Zod field that flows into `path.*`, sweep every sibling
  field that touches the filesystem, subprocess argv, SQL, or HTML in the
  same pass. Don't ship in partial-coverage waves unless explicitly time-boxed.
- **Defense vs adversarial input isn't "Phase 2" — it's part of the
  shipping bar for services that accept HTTP.** Cap unbounded buffers
  (`MAX_LINE_BYTES`), fast-path expensive parsers (`includes` before
  `JSON.parse`), and uncap legitimate listeners (`setMaxListeners(0)`)
  in the same commit that introduces the data path. They're all
  one-line additions.
- **AbortSignal is the contract for any async generator that talks to a
  network resource.** Don't ship the consumer-facing API surface
  without it.
- **`pino.flush` is required before `process.exit` whenever the transport
  is async (workers, network destinations).** Make it part of the shutdown
  template, not a per-service decision.
- **Two-pass review on review feedback.** When /aif-review produces N
  findings, default to addressing ALL of them in /aif-fix unless the
  user explicitly scopes. Auto-narrowing without checking back is a
  silent failure mode that surfaces only at PR time.
