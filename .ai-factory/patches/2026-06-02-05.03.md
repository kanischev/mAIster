# Evidence guard keyed on node type, not the terminal transition; FSM transitions documented but unwired

**Date:** 2026-06-02 05:03
**Files:** web/lib/flows/graph/runner-graph.ts, web/lib/flows/graph/gates-exec.ts, web/lib/flows/graph/artifact-store.ts, web/lib/flows/graph/evidence-readiness.ts
**Severity:** high

## Problem

Two related gaps:
1. The review-evidence guard (`assertEvidenceReady(runId,"review")`) was
   conjoined with `node.source.node.type === "human"`, so any flow whose
   terminal node is non-human (`cli`/`check`/`judge`) reached `Review` with NO
   `requiredFor:[review]` enforcement. The contract is phrased on the review
   PHASE, but enforcement was bound to a node TYPE.
2. The artifact validity FSM documented `current → failed` and `(none) → skipped`
   transitions that no code ever wrote — the docs described behavior the engine
   did not have.

## Root Cause

1. The guard encoded an incidental assumption ("review nodes are human today")
   as the gate condition, instead of the actual contract surface (a terminal,
   non-rework transition that completes the run). Keying enforcement on a
   coarse, mutable property (node type) instead of the exact condition is the
   same class as a deny-list-by-shape guard: it silently admits cases the
   assumption did not anticipate.
2. Spec described a lifecycle the implementation never reached — `failArtifact`
   existed but was only called from a test; no path wrote `skipped`.

## Solution

1. Generalized the guard to fire on `!isRework && resolveTransition(node,
   outcome) === null` regardless of node type (review/approval may be human OR
   agent/cli/check). The opt-in vacuous-ready rule keeps non-evidence / 1.1.0
   flows unaffected.
2. Wired the FSM: `current → failed` — the §3.6 backstop now marks a prior
   attempt's still-current row `failed` when THIS attempt did not re-produce the
   declared output (so stale prior evidence can't satisfy readiness), and a
   blocking `artifact_required` gate marks a missing input's stale row `failed`.
   `(none) → skipped` — an unknown/unsupported gate kind with a declared output
   records a `skipped` row (producer `gate`), forward-compat for future kinds.
   Also fixed a stale docstring in `evidence-readiness.ts` (claimed no-evidence →
   not-ready; the impl is opt-in vacuously-ready).

## Prevention

- A guard enforcing a PHASE/transition contract must test the transition, not a
  node attribute that "usually" coincides with it. If the condition reads like
  `X.type === "human"` where the contract is "the run is completing," it admits
  every other producer of that transition.
- A documented state-machine transition with no code path that writes it is
  drift — either wire it or tag it `(Designed)`. Grep each FSM edge to the
  setter that writes that state before trusting the diagram.
