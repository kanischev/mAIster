## Problem

A read-only `/aif-verify` pass on `feature/m23-observatory-readonly` surfaced
quality/robustness gaps (no behaviour was broken; gates were green):

1. **Redaction (privacy)** — `redactSignalText` surfaces gate verdict reasons on
   the dashboard but let AWS access keys (`AKIA…`) and JWTs (`eyJ….….…`) pass
   through. Its bare-token-prefix and 32+hex branches had zero direct test
   coverage, so two of the three secret-classes that reach the shared UI were
   unverified.
2. **Drift hazard** — `labelsForTest()`, a hand-written full copy of the
   `ObservatoryLabels` map, was exported from a production component module and
   had already diverged from the real `messages/en.json` catalog.
3. **Accessibility** — heatmap drill-down links had no accessible name, so the
   correction-pressure severity (otherwise carried by background colour) was not
   announced.
4. **Self-contradicting artifact** — the prior review-fixes patch §3 claimed
   retries were excluded for human nodes, but code/tests/spec/ADR all count
   them; `/aif-evolve` could have mined a false rule from it.
5. **Thin tests** — the N+1 harness only asserted a relative equality; the EN/RU
   parity test only compared top-level keys; node-detail rework signals were
   run-scoped, not node-scoped; dead labels (`filters`, `days`) and the
   `ObservatoryArtifactSummary.runCount` field were never rendered.

## Root Cause

The feature was assembled across several partial contracts (redaction, i18n,
a11y, query batching). Each edge was individually plausible but the *verification
seams* were weak: redaction trusted a regex with no adversarial corpus; a test
fixture duplicated the source-of-truth catalog instead of deriving from it; and
worked-example/priority text in the plan was frozen as exact numbers that the
implementation later refined, so the plan drifted from `scoreCluster`.

## Solution

- **Redaction**: added JWT and AWS-access-key branches to `redactSignalText`,
  and a dedicated adversarial test covering bare `sk_`/`ghp_` tokens, a 32-hex
  digest, an AWS key, and a JWT (the previously-uncovered branches).
- **De-drift**: deleted `labelsForTest()` from `observatory-summary.tsx` and
  replaced it with `__tests__/labels.fixture.ts`, which *derives* the label map
  from `messages/en.json` via `labelsFromTranslations` + a stub translator — it
  can no longer drift.
- **A11y**: added an `aria-label` to heatmap drill-down links announcing node +
  correction rate; wired the dead `filters`/`days` labels as the filter form's
  accessible name and the lookback unit; rendered `runCount` in the artifact
  list.
- **Node-detail signals**: scoped rework clustering to `hitl.stepId === nodeId`
  so a drill-down stops surfacing sibling-step rework.
- **Tests**: made the EN/RU parity check recurse all nested keys; added an
  absolute query-count ceiling beside the relative equality; documented the
  flow-reconciliation coupling with a WHY comment.
- **Artifacts**: corrected the stale review-fixes patch §3 and the plan's
  priority formula to match the shipped `scoreCluster`.

Logging: no `[FIX]` console logging was added — these are redaction/i18n/a11y/
test changes with no new runtime branch, and the project bans `console` in
committed code (pino boundary). The existing `observatory-queries` pino logger
already covers the query path.

## Prevention

- Any code that renders external text (gate reasons, HITL metadata) on a shared
  surface needs an adversarial redaction corpus per secret-class, not a single
  happy-path assertion — every regex branch must be exercised.
- Test fixtures for i18n/label maps must be *derived* from the real catalog, never
  hand-copied; a copied map is a guaranteed future drift.
- Freeze formulas in the as-built doc/ADR qualitatively; keep exact coefficients
  only in code + unit tests so plan prose can't drift from the implementation.
- A self-improvement patch is itself an artifact a verify pass checks — its
  narrative must match the shipped code, or downstream automation learns a false
  rule.
