# Flow Studio: move AI assistant from a bottom panel to a mutually-exclusive right drawer

**Date:** 2026-06-28 19:20
**Files:** web/components/studio/local-package-editor.tsx, web/components/flows/flow-editor-tabs.tsx
**Severity:** medium

## Problem

The Flow Studio editor stacked the graph canvas on top of an AI-assistant panel
in the same vertical column. The graph's own chrome (title row + node/gate
toolbars + tabs) is ~275px FIXED, so on any non-huge viewport the canvas and the
assistant fought for height: the panel either covered the graph, crushed React
Flow to 0px, or clipped its own input. Multiple height-tuning attempts
(drag-resize, min/max floors, scroll) only moved the symptom around — the
stacked layout was the root cause.

## Root Cause

Two tall, fixed-chrome regions (graph + chat) cannot coexist in one bounded
vertical column. Any `flex` split between them starves one. The fix is layout,
not sizing: the chat must not share the vertical axis with the graph.

## Solution

The AI assistant moved out of the bottom panel into a RIGHT drawer that shares
the single existing right slot with the node-properties inspector — mutually
exclusive (the user's requested model):

- New `aiOpen` state + an "Ask AI" (`✦ ИИ`) toggle button in the top toolbar.
- The canvas column is now plain `flex-1` full-height — the graph fills all
  vertical space (React Flow went from 0/84px to ~200px+ on a short viewport).
- The right `<aside>` holds EITHER the inspector portal target (`inspectorEl`)
  OR the chat drawer (`w-[clamp(420px,34vw,560px)]`), toggled by `aiOpen`.
- Mutual exclusion: `FlowEditorTabs` gained an `onSelectNode` prop that forwards
  FlowGraphEditor's already-existing `onSelectNode` up to the shell; selecting a
  node (`id !== null`) sets `aiOpen=false`, handing the slot back to the
  inspector. Re-open with the toolbar button.
- Both subtrees stay MOUNTED and toggle via `hidden` (not unmount): the chat
  keeps its live ACP run + draft text across open/close, and React Flow's
  `createPortal` inspector target is never torn down.

`onSelectNode` was already plumbed FlowGraphEditor→(unused); only one hop
(FlowEditorTabs forward) had to be added — no new selection state.

## Prevention

- Two regions that each carry tall FIXED chrome must not share one bounded axis.
  Put them on different axes (here: graph = full column, chat = side drawer) or
  give one its own scroll container that does not steal from the other.
- A drawer that must preserve live state (a running session, unsent input) across
  open/close must stay mounted and hide via `hidden` — never conditionally
  unmount it, or the state resets.
- When two surfaces share one slot, define the precedence explicitly and bubble
  the trigger that flips it (here: node-select → inspector wins) rather than
  letting both try to own the slot.
