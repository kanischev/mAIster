# Long agent turns hit undici fetch headers timeout

**Date:** 2026-06-26 14:42
**Files:** web/lib/supervisor-client.ts, web/lib/__tests__/supervisor-client.test.ts
**Severity:** high

## Problem

Flow runs with an AI node failed after about five minutes with
`EXECUTOR_UNAVAILABLE`, even though the supervisor session and ACP adapter were
still alive. The failed node duration was consistently around 301 seconds, and
the supervisor logs showed `sse-disconnect`, `ACP connection closed`, and an
intentional session delete after the web tier abandoned the request.

## Root Cause

`sendPrompt` and `streamSession` used Node's global `fetch` for web-to-supervisor
HTTP. The supervisor does not send `/sessions/:id/prompt` response headers
until the ACP turn completes, and SSE streams can be quiet between events. The
global fetch path inherits undici's default five-minute headers/body timeout, so
the web tier converted the timeout into `EXECUTOR_UNAVAILABLE` and then cleaned
up the otherwise live supervisor session.

## Solution

Added a dedicated package `undici` `Agent` with `headersTimeout: 0` and
`bodyTimeout: 0`, and routed only the open-ended supervisor calls through
package `undici.fetch` with that dispatcher:

- `sendPrompt` for long ACP turns.
- `streamSession` for quiet SSE streams.

Other supervisor requests still use global fetch defaults. The helper logs
`[FIX:supervisor-long-lived-fetch]` through the existing pino logger boundary,
with structured `url` and `ctx` fields. Tests now mock package `undici.fetch`
and assert that `sendPrompt` and `streamSession` receive the dispatcher while
ordinary calls keep using `globalThis.fetch`.

## Prevention

- Review any new web-to-supervisor call that intentionally waits on agent work
  or an SSE/streaming response; use the long-lived dispatcher only for that
  class.
- Do not use `setGlobalDispatcher` for this because Node's built-in fetch and
  the package `undici` dispatcher can be incompatible.
- Keep fixed request ceilings on explicit watchdogs, budgets, and run
  time-limit policy, not on the HTTP transport for live ACP turns.
- Regression tests should assert the transport choice, not sleep for five
  minutes.
