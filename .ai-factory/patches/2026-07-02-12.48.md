# Fixture-sync clobbered the triager's flagged-task hold guard (fixture had diverged from the maister-plugins deliverable)

**Date:** 2026-07-02 12:48
**Files:** maister-plugins/packages/core/maister-agents/triager.md (sibling repo), web/lib/agents/__tests__/fixtures/core-package/maister-agents/triager.md, web/lib/agents/__tests__/triager-definition.test.ts
**Severity:** high

## Problem

Codex adversarial review (no-ship, [high]): the triager persona lost the
Step-1 early-stop guard for `triage_status='flagged'` tasks. Without it, a
flagged (human-held) task is re-triaged on ANY human or system
`task.comment_added` event — the event trigger only self-excludes the
agent's OWN comments, and neither the trigger nor the ext triage route has
a flagged-state guard, so the agent can re-comment, re-flag, or stamp a new
verdict/enqueue over a human-held task. The auto-launch tick's give-up
comment would even re-trigger it systematically.

## Root Cause

Two-source drift plus an unverified overwrite. The platform-repo fixture
(`fixtures/core-package/.../triager.md`) is contractually byte-identical to
the maister-plugins deliverable, but the guard had been added ONLY to the
fixture (during ADR-112 hardening) and never back-ported to the
deliverable — the contract was already broken before this session, and the
SHIPPED persona never had the guard at all. During the ADR-121 instruction
update the deliverable was edited and `cp`-synced OVER the fixture without
first diffing the fixture's full body (only its frontmatter was read), so
the fixture-only guard was silently clobbered. Classic mirror-pair hazard:
when two copies must be identical, syncing in either direction without a
diff destroys whichever side was ahead.

## Solution

- Restored the canonical guard hunk (recovered via
  `git show HEAD:web/lib/agents/__tests__/fixtures/.../triager.md`) into the
  maister-plugins deliverable's Step 1 — a full old-fixture vs old-deliverable
  diff proved the guard was the ONLY divergent hunk — then re-synced the
  fixture byte-identical (md5-verified).
- New regression test in triager-definition.test.ts asserts the parsed
  prompt still contains the flagged early-stop text and the no-op wording —
  the exact regression class that occurred (prompt-level invariant dropped
  during a sync). Enforcement point is the prompt, so the definition test is
  the honest level; the wire test cannot exercise agent reasoning (its own
  NOTE says so).
- Gates: triager-definition 4/4, triager-package + triager-wire integration
  9/9 (real PG), eslint 0, parseAgentDefinition green on the deliverable.

## Prevention

- Before `cp`-syncing a contractual mirror pair (fixture ↔ package
  deliverable), ALWAYS `diff` the two current bodies first; if they diverge,
  reconcile the delta explicitly — never assume the declared invariant holds.
- Prompt-level invariants (agent personas) deserve pinning tests: a
  `toContain` on the load-bearing guard text turns silent prose loss into a
  red test.
- When a review claims "X was deleted", verify WHERE X previously lived —
  here the guard was fixture-only, so the real defect (shipped persona never
  had it) was worse than the reported regression.
