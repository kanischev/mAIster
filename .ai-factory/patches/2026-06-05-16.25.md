## Problem

A code review of the M24 scheduler + M25 catalog branch produced findings. Two
earlier `/aif-fix` runs (patches 12.21, 15.18) had already landed most fixes,
but: (a) the `/api/cron/gc` compatibility route silently ran the keepalive +
reconcile sweeps (it could move runs to `Crashed`); (b) a thrown GC sub-sweep
was swallowed by `runGcSweeps` (`Promise.allSettled` → zero summaries), so the
route returned 200 on a real failure; (c) there was still no UI/API to author
scheduler jobs; (d) the attempt id used `md5(random()…)` instead of a UUID; (e)
catalog routes carried a redundant `instanceof SyntaxError` ternary.

## Root Cause

- The M24 refactor routed `/api/cron/gc` through the full `runSystemSweep`
  composition for "code reuse", conflating GC with live state-transition sweeps.
- `runGcSweeps()` is designed to never throw (allSettled), so wrapping it in a
  try/catch in the compat path could never surface a thrown sub-sweep — the
  observability contract (207 on partial failure) silently broke.
- "Managed via UI" was implemented as tick-time seeding (`ensureDefaultSchedulerJobs`),
  not an operator surface.

## Solution

- Extracted `runGcBundle()` that calls `runWorkspaceGcSweep` + `runRevisionGcSweep`
  + `runCapabilitiesCleanupSweep` directly, each with its own try/catch (mirrors
  the pre-M24 GC route). `runGcCompatibilitySweep` uses the bundle only;
  `runSystemSweep` = keepalive + reconcile + bundle. GC cron no longer reconciles.
- Added the full admin CRUD: `lib/scheduler/job-admin.ts` (+ zod schema), routes
  under `/api/admin/scheduler-jobs[/{jobId}]` (`requireGlobalRole("admin")`), an
  `/admin/scheduler` page + view-only table + create/edit modal, role-gated nav,
  EN/RU i18n, and OpenAPI. `agent_tick` create defaults `maxFailures` from the
  shared `schedulerAgentTickMaxFailures()` (DRY with the disable path).
- Declared the partial unique index `…_active_draft_uq` in `schema.ts` (was only
  in the migration → drift). Switched attempt id to `gen_random_uuid()::text`.
  Hardened `http_ping` with a URL `http(s)` scheme check. Dropped the redundant
  catalog-route ternary (the shared error mapper already handles SyntaxError).
- Kept the per-kind budget enforcement (user superseded the earlier "don't
  limit" answer once it was a working, tested feature).

## Prevention

- Before editing on a branch that other `/aif-fix` runs may have touched,
  re-read the ACTUAL current file state — a review snapshot goes stale fast
  (jobs.ts had grown 367→500 lines between review and fix).
- A sweep aggregator that uses `Promise.allSettled` swallows throws; if a caller
  must report partial failure, call the sub-sweeps directly or have the
  aggregator return per-sweep failure signals — never wrap allSettled in
  try/catch and expect to catch anything.
- In Vitest, do NOT combine `vi.resetModules()` with a dynamically-imported SUT
  when the test constructs a domain error (e.g. `MaisterError`) and passes it
  through a mock: the SUT's fresh module realm makes `instanceof` false, so
  `isMaisterError` returns false and the status maps to 500 instead of 409.
- When a user decision predates a context change (dead code became a working
  feature), re-confirm before reverting working, tested code.
