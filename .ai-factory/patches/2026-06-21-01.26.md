# M36 local-package adversarial-review fixes (race + symlink-leaf + import materialization)

**Date:** 2026-06-21 01:26
**Branch/worktree:** feature/flow-studio-package-viewer (.claude/worktrees/flow-studio-package-viewer)
**Files:**
- web/lib/local-packages/service.ts (ensureDefaultLocalPackage, listFiles, cleanCopyExcludingGit)
- web/lib/local-packages/paths.ts (resolveWithinWorkingDir leaf guard)
- web/app/api/studio/local-packages/[id]/import/route.ts (pre-materialize caps)
- tests: confinement.test.ts (+2), service.integration.test.ts (+two-racer), import/__tests__/route.test.ts (new)
**Severity:** critical + high

## Problem

A Codex adversarial review of the merged M36 local-package feature found 3 issues
the earlier security-review pass missed:

1. **[critical] Concurrent default-package creation deletes the winner's repo.**
   `ensureDefaultLocalPackage` derived its slug via the racy read-then-act
   `uniqueSlugForName`, scaffolded `workingDir`, then claimed the row. Two
   concurrent first-element-forks for the same project computed the SAME slug Ôćĺ
   SAME `workingDir`; the insert loser then `rm`-ed that shared dir ÔÇö the
   WINNER's repo ÔÇö losing the forked content. (It could also raise a stray
   `23505` on the slug-unique constraint, since the `onConflictDoNothing` arbiter
   only covered `(project_id) WHERE is_default`.)
2. **[high] Working-dir read follows a symlink LEAF outside confinement.** The
   earlier confinement fix walks ANCESTORS, but `readFileContent` Ôćĺ
   `fsReadFile(abs)` still followed a symlink whose LEAF pointed outside;
   `listFiles` used `stat` (follows); `cleanCopyExcludingGit` (fork/export)
   copied symlinks verbatim. The read/list/export/fork class was open.
3. **[high] Import caps run after the upload is materialized.** The route called
   `File.arrayBuffer()` (folder mode via `Promise.all`, archive single) BEFORE
   any `MAISTER_IMPORT_MAX_*` cap, so a member could force large in-memory
   allocations ahead of the rejection. The earlier fix capped DECOMPRESSION, not
   the raw multipart materialization.

## Root Cause

1. `uniqueSlugForName` is SELECT-all-then-pick ÔÇö not concurrency-safe. The author
   anticipated the insert race (onConflictDoNothing) but assumed each racer owned
   a DISTINCT working dir; under a slug collision they share one, so the loser's
   "roll back my orphan" deletes the winner's adopted dir.
2. Confinement was hardened only on the not-yet-existing-intermediate (write/mkdir)
   vector. An EXISTING leaf that is itself a symlink is resolved and then followed
   by the fs read/copy ÔÇö a different escape than a symlinked intermediate.
3. Caps were enforced inside the import LIBRARY (after the route handed it fully
   materialized bytes). `File.size` is available without reading, but the route
   never consulted it before `arrayBuffer()`.

## Solution

1. `ensureDefaultLocalPackage` now derives a per-attempt UNIQUE slug
   (`${slugifyName(name)}-${uuid8}`) ÔÇö never the shared deterministic slug. Two
   racers get distinct working dirs: the loser rolls back only its OWN orphan,
   and its insert hits ONLY the `(project_id)` arbiter (clean no-op, no stray
   23505). The user-facing name stays clean; the slug is an internal handle.
2. `resolveWithinWorkingDir` (central) now `realpath`s the FINAL resolved path
   when it exists and requires the real target to stay under the root ÔÇö closing
   symlink-LEAF read/write/copy escapes (a not-yet-existing leaf realpath-throws
   Ôćĺ skipped, correct for new files/imports). `listFiles` switched `stat`Ôćĺ`lstat`
   (symlink leaves are not regular files Ôćĺ skipped). `cleanCopyExcludingGit`'s
   `cp` filter now also rejects symlinks (lstat per source) so forks/exports
   never carry an escaping symlink.
3. The import route validates DECLARED sizes BEFORE materializing: folder mode
   checks `fileParts.length` ÔëĄ maxEntries, each `file.size` ÔëĄ maxFileBytes, and
   the running total ÔëĄ maxBytes; archive mode checks the single blob's `.size` ÔëĄ
   maxBytes (compressed cap). Rejection is a 422 before any `arrayBuffer()`.

## Prevention

- **A read/confinement boundary must guard the LEAF, not just intermediates.**
  When hardening path confinement, realpath the final existing target (and switch
  enumerations to `lstat`), and sweep EVERY consumer of the sink ÔÇö read, list,
  export, fork, copy ÔÇö not only the write path the first finding cited.
- **Claim a unique identity BEFORE filesystem side-effects.** Any "scaffold then
  insert-or-rollback" flow must give each attempt a unique path (uuid), so a
  concurrent loser's cleanup can never touch the winner's resource. Read-then-act
  slug/id helpers are not concurrency-safe; never derive a shared mutable path
  from them ahead of the DB claim. Add a two-racer integration test.
- **Enforce size/count limits at the REQUEST boundary, on declared metadata,
  before materializing.** `File.size`/`length` are free; check them before
  `arrayBuffer()`/`Promise.all`. A cap inside a downstream library does not
  protect the process from the bytes the route already read into memory.
