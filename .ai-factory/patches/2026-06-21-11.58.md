# Run-bound credentials, side-table-row-as-signal, and a renumber grep-to-zero gap

**Date:** 2026-06-21 11.58
**Files:** web/lib/runs/bound-run.ts, web/lib/runs/run-status-sets.ts, web/app/api/v1/ext/runs/{delegate,plan,collect,cancel,promote,rework,message}/route.ts, web/lib/orchestrator/cascade.ts, web/lib/agents/launch.ts, docs/{CLAUDE,error-taxonomy,system-analytics/orchestrator}.md, docs/decisions.md
**Severity:** high

## Problem

A Codex adversarial review (which challenges invariants under edge/terminal
conditions, not just implementation defects) found two confirmed-real holes that
a normal /aif-review (focused on auth-first / two-phase / concurrency) and six
review subagents all MISSED:

1. **A run-bound ext token kept working after its run terminalized.** The
   orchestrator token (`orchestrator-run:<id>`) was revoked ONLY on the normal
   graph-exit path; abandon/crash/stop flip the run terminal without revoking,
   and the seven run-bound routes loaded the bound parent by id+project but
   never checked its `status`. A leftover/copied token could delegate / cancel /
   promote / rework under a TERMINAL tree for the 48h TTL.

2. **A finalization decision keyed on a side-table row's existence broke for the
   shared case.** `finalizeAgentRun` decides Review-vs-Done from
   `workspaceRows.length > 0`. A shared-worktree REUSER child intentionally gets
   NO `workspaces` row (the allocator owns the UNIQUE `worktree_path`), so it
   finalized `Done` not `Review` — its diff unreviewable / strandable.

3. **(meta) A renumber grep-to-zero missed a file.** The earlier
   ADR-095/096/097→098/099/100 renumber detected shared files to touch by
   grepping for the lowercase ANCHOR slug (`adr-095-orchestrator`). A glossary
   line in `docs/CLAUDE.md` used the BARE `ADR-095/096` with no anchor and no
   `M36`, so it was never flagged and shipped pointing at main's unrelated ADRs.

## Root Cause

1. **The token's existence was treated as the authorization, and its revocation
   lived on one of N terminal paths.** Two separate bugs that compound: a
   credential whose lifetime is not bounded to its subject's lifetime on EVERY
   terminal edge, AND consumers that trust the credential without re-validating
   the subject. A TTL is not a security boundary.

2. **A proxy signal (the workspace row) stood in for the real predicate ("did
   this child produce a writable diff").** The proxy held for the per-run case
   and silently inverted for the shared case. Nobody asked "what if two children
   share one row's worth of tree?"

3. **Grep-to-zero was run over a PRE-FILTERED file list, and the filter pattern
   was narrower than the replacement set.** Detection used anchor slugs; the
   thing being renumbered also appears as bare `ADR-NNN`. A filter narrower than
   the edit set silently drops cases.

## Solution

1. New `resolveActiveBoundRun(db, boundRunId, projectId)` — one place that loads
   the bound run and returns a result-union `{ok:false, PRECONDITION}` when it is
   missing or `isTerminalRunStatus`. Wired into all 7 routes as the first check
   after the run-binding null-guard (result-union, not throw, because the load
   sits outside each route's MaisterError-mapping try/catch and `handleExt`
   re-throws → 500). Token revocation added to `cascadeAbandonRunTree` (the one
   teardown all of abandon/stop/drop/crash route through), before its
   nothing-to-cascade early-return so a childless tree still revokes.

2. Owner chose fail-closed: GATE `workspace_mode: shared` + writable `worktree`
   at launch (CONFIG) until the review-ownership model is designed. The dormant
   shared-allocation + serialized-writer code stays for the redesign.

3. Fixed the one missed glossary line; re-ran a whole-tree scan for
   orchestrator-context `ADR-09[567]` (excluding main's topics) → zero real
   remaining.

## Prevention

- **A run-bound (or resource-bound) credential needs BOTH:** revocation on EVERY
  terminal transition of its subject (centralize the terminal teardown so there
  is ONE place to revoke), AND consumer-side re-validation of the subject's live
  state on each use. Never let "the token still verifies" imply "the subject is
  still active." Treat the credential TTL as a backstop, not a boundary.
- **Don't key a state decision on the presence/absence of a side-table row when
  that row can be shared or omitted.** Derive the decision from the authoritative
  per-subject column (here `runs.agent_workspace`), or refuse the configuration
  that makes the proxy ambiguous. Ask "what if two subjects share one row?"
- **Grep-to-zero means the WHOLE tree and ALL forms of the token, with detection
  ≥ the edit set.** For ADR/migration renumbers: scan bare `ADR-NNN`, the
  lowercase `adr-nnn-` anchor slug, `NNNN_name`, and bare `` `NNNN` `` across the
  entire repo (incl. `CLAUDE.md`, `*.md` glossaries) — never over a list
  pre-filtered by a single one of those forms.
- **Run an adversarial / invariant-challenging review on security-or-state-machine-
  heavy changes.** Defect-focused review and an invariant-challenging review find
  different bug classes; the cheap multi-angle pass is worth it before merge.
