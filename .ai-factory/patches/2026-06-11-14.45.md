# Codex adversarial-review fix pass on M31: pre-authz launch leak, inbox membership scoping, renumber stragglers

**Date:** 2026-06-11 14:45
**Files:** web/lib/services/runs.ts, web/lib/queries/inbox.ts, web/app/(app)/page.tsx, web/app/(app)/projects/[slug]/page.tsx, web/lib/services/__tests__/runs-launch-gate.test.ts, web/lib/queries/__tests__/inbox.integration.test.ts, CLAUDE.md, .ai-factory/ROADMAP.md, docs/database-schema.md, docs/db/erd.md, docs/db/runs-domain.md, docs/api/external/operations.openapi.yaml
**Severity:** high

## Problem

Three findings from a post-implementation Codex adversarial review of the
M31 social-board branch, all validated against the code:

1. **launchRun evaluated launchability before `ctx.authorize`.** An
   authenticated non-member POSTing a foreign `taskId` to `/api/runs`
   received the task's launchability classification, the project's
   archived state, and — new on this branch — the `KEY-N` identifiers of
   open blocker tasks in the PRECONDITION error, all before the project
   authorization check ran.
2. **Portfolio social inbox ignored current membership.** `getUnreadInboxCount`
   / `getInboxItems` filtered only by `(recipient_type, recipient_id)`, so a
   user removed from a project kept seeing new fanout items (task titles,
   `KEY-N`, project names) from it; items from archived projects also
   surfaced. The sibling `getCrossProjectHitlInbox` on the same page already
   filtered by membership + `archived_at`.
3. **ADR/migration renumber left stragglers.** After the 075→078 / 0040→0041
   renumber, `CLAUDE.md` ("M31, ADR-075"), `.ai-factory/ROADMAP.md`
   ("ADR-075, migration `0040_social_board.sql`"), short-form ``migration
   `0040` `` refs in docs/database-schema.md, docs/db/{erd,runs-domain}.md and
   docs/api/external/operations.openapi.yaml — plus two missed
   Designed→Implemented status flips in the ERD intros — stayed stale.

## Root Cause

1. The authorize call is ctx-injected and sat mid-function, so precondition
   gates accreted *before* it: M28 added the classifier gate ahead of
   `ctx.authorize`, and M31 enriched that gate's error with cross-task
   identifiers without re-checking what already ran pre-authz. There was no
   stated invariant "nothing user-distinguishable before authorize", and no
   test pinned the ordering.
2. Fanout rows are written at event time and never cleaned on membership
   revocation, so the read side must re-derive visibility — but the new
   queries treated recipient-scoping as authorization and did not mirror the
   established `getCrossProjectHitlInbox` visibility pattern sitting on the
   same portfolio page.
3. The renumber sweep was driven by a curated file list (docs/, web/, mcp/)
   instead of a repo-wide grep-to-zero, and only matched the long form
   (`0040_social_board`) — short-form ``migration `0040` `` refs and files
   outside the list (`CLAUDE.md`, `.ai-factory/ROADMAP.md`) never entered
   the sweep.

## Solution

1. Reordered `launchRun`: task lookup → project lookup → `ctx.authorize`
   → archived check → latest-flow-run + open-blockers + classifier gate.
   Non-members now get the bare authz error; classification/blocker detail
   is post-authz only. Added a unit test that injects a throwing `authorize`
   on a non-launchable task and asserts the authz error (not PRECONDITION)
   surfaces. Existing gate/branch/materialize fakes were positionally
   unaffected (runs + task_relations dispatch by table identity).
2. Added `globalRole` to all three inbox queries; non-admin recipients get
   an inner join on `project_members (project_id, user_id)`, and all roles
   get `projects.archived_at IS NULL` — exactly mirroring
   `getCrossProjectHitlInbox`. The unused-but-exported sibling
   `getUnreadInboxCountsByProject` got the same treatment (sibling-sweep
   rule: same sink class, same pass). Integration tests now seed users +
   memberships and pin revoked-membership hiding, admin bypass, and archived
   exclusion.
3. Fixed all twelve stale references and the two missed status flips;
   re-grepped repo-wide for `ADR-075`, `0040_social` and `` `0040` `` (minus
   webhooks/plans/patches) to zero.

## Prevention

- **Authz-first invariant for ctx-authorize services:** `ctx.authorize`
  must run immediately after the minimal lookups that resolve the project
  id; every other precondition, classification, or enriched error goes
  after it. When touching launch/precondition code, check the authorize
  call's position and keep an ordering test (throwing-authorize sentinel)
  in the gate suite.
- **Fanout tables are not authorization.** Any read model over per-recipient
  fanout rows (inbox_items and future notification surfaces) must re-derive
  visibility at read time: membership join for non-admin + archived-project
  exclusion, mirroring `getCrossProjectHitlInbox`. Review checklist for new
  portfolio/cross-project queries: "where is the membership filter?"
- **Renumber = grep-to-zero, never a file list.** When renumbering
  ADRs/migrations, finish with repo-wide greps on ALL forms — `ADR-NNN`,
  `NNNN_name`, and bare `` `NNNN` `` — excluding only plans/patches archives,
  and require zero hits. `CLAUDE.md` and `.ai-factory/ROADMAP.md` are part
  of the surface even though they live outside docs/.
- **Status flips travel with the renumber:** Designed→Implemented sweeps
  must include the db/ ERD intro paragraphs, not just the schema tables.
