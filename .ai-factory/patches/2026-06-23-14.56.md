# M39 Stream-A review fixes: fork authz gate, ReactFlow nodeTypes stability, dir-retirement doc drift

**Date:** 2026-06-23 14:56
**Files:** web/app/api/studio/packages/[ref]/fork/route.ts, web/app/api/studio/packages/[ref]/fork-element/route.ts, web/components/flows/flow-graph-editor.tsx, web/lib/local-packages/{fork,service,validate}.ts, web/components/studio/{use-new-local-package.ts,packages-list,local-packages-list,package-detail}.tsx, docs/{system-analytics/agents.md,api/web.openapi.yaml,database-schema.md,decisions.md,flow-dsl.md,screens/studio/editor.md} + 4 test files
**Severity:** high

## Problem

`/aif-review` of the M39 Stream-A branch surfaced three merge-blocking classes:

1. **Auth under-gate (security).** `POST /api/studio/packages/{ref}/fork` and `ŌĆ”/fork-element` gated with `requireSession()` while EVERY sibling studio mutation used `requireGlobalRole("member")`. `requireSession` only checks authentication ŌĆö so a `viewer`, an inactive account, or a password-must-change user could create local packages + write working dirs. The route comments even claimed "member-accessible", which the code did not enforce.
2. **ReactFlow `nodeTypes` recreated per selection (correctness/perf regression).** `flow-graph-editor.tsx` moved `selectedNodeId`/`tooltipsById` into the `nodeTypes` factory + its memo deps, so a new `nodeTypes` object was minted on every node click ŌĆö the documented ReactFlow footgun that remounts every node renderer (same family as this milestone's "reseed-remount drops the connect modal" bug). The sibling read-only viewer added in the same branch did it correctly.
3. **Doc drift from a directory retirement.** M39 converged the platform-agent runtime dir from root `agents/<stem>.md` to `maister-agents/<stem>.md`, but six doc surfaces still asserted the old path (one was an internal contradiction in the same file), and the fork OpenAPI was tagged `Designed` though shipped.

## Root Cause

1. **Auth helper semantics not checked at the call site.** `requireSession` is deliberately permissive (it must let the change-password page clear its own gate ŌĆö see authz.ts comment). It is NOT an action gate. The author reached for the lightest-looking helper whose NAME read as "is logged in" without confirming it enforced account-active + role. The "member-accessible" comment was aspirational, never matched to code.
2. **External component state closed over instead of threaded through data.** Selection/tooltip are per-node render inputs; closing them into the node-renderer factory couples renderer IDENTITY to selection state. ReactFlow keys node components by `nodeTypes` identity, so any per-interaction change forces a remount. The fix is to keep the factory stable and pass per-node state via `node.data`.
3. **A dir retirement is a contract-WIDE rename, but only the code + primary doc were swept.** "Code lands, docs follow on the cited lines" leaves sibling surfaces (a mermaid `scan installed_path/agents/*.md`, a narrative DB `sourcePath` comment, an OpenAPI endpoint description, `flow-dsl.md`) describing the dead path. They drift independently of the prose the author was looking at.

## Solution

1. Swapped both fork routes to `requireGlobalRole("member")` (which chains `requireActiveSession` ŌåÆ account-active + password-gate + role), and rewrote the comments to state the gate + WHY. Lib-level fork tests were unaffected (they call the service directly); no route-level fork test existed.
2. Made `makeEditorNodeView(labels.graph)` stable (`nodeTypes` memo dep `[labels.graph]` only), seeded `selected`+`tooltip` into the initial node `data`, and added a `useEffect` that syncs `data.selected` on `selectedNodeId` change (no-op when unchanged). Mirrors the verified `board/flow-graph-view.tsx` pattern.
3. Ran a `grep`-to-zero across the whole `docs/` tree for stale platform-agent `agents/<stem>.md` claims (NOT just the cited lines) ŌĆö fixed 9 current-tense occurrences across 5 files, left ADR-089's historical body + ADR-105's transition text intact, and added a one-line supersession pointer at ADR-089. Flipped the four `ADR-105 ŌĆö Designed` fork tags to `Implemented` and corrected the now-stale `requireSession` description. Confirmed the new ADR anchor resolves (561, was 560).

Also addressed the non-blocking review findings in the same pass (default-to-all rule): extracted `loadSourceElement`/`copyElementInto` from the two duplicated element-fork functions; added rollback-on-throw to `forkPackageToLocal`/`createLocalPackage` (no orphan dir on a mid-create failure); extracted a shared `useNewLocalPackage` hook; reconciled `inferFileKind` so a capability subagent is not mislabeled `readme`; simplified the `validate.ts` kind dispatch (dropped a duplicated regex + a redundant conjunct); added the two MAJOR-gap tests (subagent editor branch, manifest/subagent dispatch + `initialSelectedPath`) + `listAllLocalPackages` coverage + 3 node-tooltip branch tests; added missing `: ReactElement` return types.

## Prevention

- **Auth gate at every state-changing route must be `requireGlobalRole`/`requireProjectAction`/`requireActiveSession`, NEVER bare `requireSession`.** `requireSession` is for the password-change escape hatch only. When you see it on a mutating route, treat it as a finding. Cross-check the route against its siblings ŌĆö a single route using a weaker gate than its neighbors is the tell.
- **ReactFlow `nodeTypes`/`edgeTypes` must be referentially stable.** Per-node render state (selected, status, tooltip, current) goes through `node.data`, never closed into the factory or its memo deps. Review check: grep changed `useMemo<NodeTypes>` deps for anything that changes per-interaction.
- **A dropped/renamed dir/route/status/field is a `grep`-to-zero sweep of the WHOLE `docs/` tree, not the cited lines.** Mermaid node labels, narrative DB comments, OpenAPI endpoint descriptions, and cross-domain docs (`flow-dsl.md`) drift independently. Record the grep in the close-out. Leave ADR history immutable; annotate with a supersession pointer instead. Re-run `validate:docs:adr:all` ŌĆö a new anchor link must resolve.
- **Shipped code must be tagged `Implemented`, not `Designed`** ŌĆö when a milestone flips status, grep the OpenAPI + screens + analytics for the ADR number, not just the system-analytics prose.
