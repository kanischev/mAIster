# Codex adversarial-review fixes for M39 Stream-A local packages (F1-F3; F4 blocked)

**Date:** 2026-06-23 18:38
**Files:** web/lib/local-packages/service.ts, web/lib/local-packages/fork.ts,
web/lib/local-packages/manifest.ts, web/lib/config.schema.ts,
web/app/api/studio/local-packages/[id]/cut-version/route.ts (+ 6 test files)
**Severity:** critical

## Problem

A Codex adversarial review of the local-package authoring branch returned **No ship** with
1 critical + 3 high findings:
- **F1 (critical):** two concurrent same-name `createLocalPackage`/`forkPackageToLocal` calls
  derived the SAME slug → SAME working dir; the insert loser's `catch`/`!row` `rm(workingDir)`
  deleted the **winner's** repo (active DB row → missing repo). Data loss.
- **F2 (high):** `scaffoldWorkingDir` wrote the display name into `maister-package.yaml`'s `name`
  (violates `capabilityRefIdSchema`) + an empty `flows: []` (violated `.min(1)`), committed at
  git-init (bypassing the commit gate, which only validates *changed* paths) → invalid baseline
  that later failed cut-version; flow element forks copied a flow without adding it to `flows[]`.
- **F3 (high):** the cut-version route exported the **live working dir** (uncommitted/invalid WIP)
  with no clean-tree, commit-gate, or lock check — contradicting "commit state is the validation gate".
- **F4 (high):** package-root `maister-agents/` are inventoried (Studio-visible) but never registered
  (registration scans the per-flow revision path), so they are not launchable.

## Root Cause

- **F1:** scaffold-then-insert ordering + a deterministic `slug → workingDir` map. The unique
  constraint fired AFTER both racers had written the shared dir, so the loser's rollback hit a
  path it did not exclusively own. (`ensureDefaultLocalPackage` had already been fixed for the
  identical race — the create/fork siblings were missed.)
- **F2:** the scaffold conflated the human display name with the manifest id; the install schema
  required >=1 flow but empty/draft packages are legitimate (owner D2).
- **F3:** the route trusted the working dir as if it were committed state.
- **F4:** ADR-105 A4 relocated platform agents to the **package root**, but the registration AND
  the launch resolver (`lib/agents/effective.ts`) were left keyed per **flow revision**
  (`<flowRefId>:<stem>`, `.md` read from the flow's `installedPath`).

## Solution

- **F1 — insert-first (owner D1):** claim the unique slug via the DB INSERT *before* any fs work
  (`insertLocalPackageRow` shared by create + fork; 23505 → CONFLICT). The race loser fails having
  touched nothing; a post-insert scaffold failure rolls back only the caller's own row+dir
  (`rollbackLocalPackageRow`). Friendly slugs preserved (no UUID tail / temp-dir rename).
- **F2.a:** scaffold writes `name: <slug>` + `metadata.title: <display>` (`serializeScaffoldManifest`).
  **F2.b:** flow element forks register the copied flow in `flows[]` (`appendManifestFlow` +
  `registerFlowElementInManifest`). **F2.c (D2):** relax `maisterPackageManifestSchema.flows` to
  `.default([])` — empty packages are valid.
- **F3 (D3):** `assertPackageCuttable` (clean-tree gate + full-package validation over ALL paths)
  called by the cut-version route before any export.
- **F4:** NOT implemented — blocked (see Prevention). F1-F3 verified: typecheck clean, 4762 unit +
  19 local-package integration tests green.

## Prevention

- **Changing an identifier scheme is a grep-every-CONSUMER task, not just every writer.** The plan
  pre-committed to "grep every reader of `agents.id` FIRST" before F4. That gate fired: the launch
  resolver + attach gate + available-list all *parse* the id prefix as a project-configured
  `flowRefId` and read the `.md` from that flow revision's path. A `<packageName>:<stem>` id would
  have **registered a catalog row that can never launch or attach** — a plausible-looking false fix.
  When an id's prefix carries meaning that downstream code decodes, enumerate every decoder before
  picking a new scheme.
- **A relocation (A4: agents → package root) must move EVERY coupled consumer in the same pass** —
  inventory collection was moved, but registration + effective-resolution + attach were not, leaving
  a visible-but-unusable surface. Grep the old location's readers when moving a canonical path.
- **Relaxing a SHARED schema has blast radius:** dropping `flows.min(1)` changed the git-package
  contract too and broke 3 tests (`packages/manifest`, local `manifest`/`validate`,
  `package-manifest-form`) that encoded the old "reject empty flows" rule. Grep fixture/contract
  tests for the old invariant when loosening a schema.
- **Sibling sweep:** F1's race + F2's scaffold bug each existed at multiple sites
  (`createLocalPackage` + `forkPackageToLocal`; `scaffoldWorkingDir` via create + `ensureDefault`;
  both element-fork entry points). Fix the class, not the cited line.
