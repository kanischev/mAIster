# M19 reconcile/GC: route DTO contract drift + non-recoverable Crashed discard gap

**Date:** 2026-06-02 02:33
**Files:** web/app/api/runs/[runId]/recover/route.ts, web/app/api/cron/gc/route.ts, web/components/runs/run-recover-actions.tsx, web/app/(app)/runs/[runId]/page.tsx, web/app/api/runs/[runId]/recover/__tests__/route.integration.test.ts, web/app/api/cron/gc/__tests__/route.integration.test.ts, web/e2e/_seed/seed-e2e.ts, web/e2e/m19-reconcile-gc.spec.ts
**Severity:** medium

## Problem

Codex adversarial review of the M19 branch surfaced three real defects:

- **F1 — non-recoverable Crashed run had no UI discard path.** The run-detail
  page gated the *entire* `RunRecoverActions` block on `detail.recoverable`, so
  a Crashed run with no `acpSessionId` (or a session-less current node) rendered
  neither Recover nor Discard. Discard is the only UI entry into the GC
  countdown — such a run was stranded, removable only by the cron sweep.
- **F2 — cron GC route returned a shape the OpenAPI did not declare.** The route
  returned the internal nested `{workspace, revision}` sub-summaries; the
  published `GcSweepSummary` contract is the flat `{worktreesPreserved,
  worktreesRemoved, revisionsRemoved, errors}`. It also returned 200 even when
  the workspace sweep CAUGHT per-row failures (`skippedUnpreserved`/`failed`),
  so a cron monitor reading the status code would miss a partial cleanup
  failure.
- **F3 — recover route error states were not `MaisterErrorBody`.** Non-success
  recover states (`discard-only`/`conflict`/`unresumable`/`transient`) returned
  an ad-hoc body, so clients could not branch on the typed `code` (ADR-008
  closed union) and only had the HTTP status to go on — even though the OpenAPI
  409/410/503 entries already declared `MaisterErrorBody`.

## Root Cause

All three are **the same class of defect: the implementation and its published
contract drifted, and the test suite asserted the drifted shape instead of the
contract.**

- F1: "recoverable" was overloaded as the visibility gate for *both* actions,
  when it should only gate Recover. The Discard affordance has a different
  invariant (available for every Crashed run). The e2e only seeded a
  *recoverable* Crashed run, so the missing-discard path was never exercised.
- F2: the route was written against the internal sweep return types, never
  re-projected onto the OpenAPI DTO, and the e2e asserted `toHaveProperty
  ("workspace")` — codifying the wrong shape so the drift stayed green.
- F3: the success path was typed/tested first; the error path was added later
  with a hand-rolled body and a status-only test assertion, so the missing
  `code` field passed CI.

## Solution

- **F1**: `RunRecoverActions` now takes `canRecover` and renders Recover only
  when true; Discard renders unconditionally. The page always mounts the
  component (passing `canRecover={detail.recoverable}`) and shows a
  `run-not-recoverable` hint when not recoverable.
- **F2**: the cron route projects both sub-sweeps onto the flat `GcSweepSummary`
  and pushes an `errors[]` entry — forcing 207 — when a sweep threw OR returned
  `skippedUnpreserved > 0` / `failed > 0`. `skippedReferenced` is NOT an error
  (a still-referenced revision is correctly retained).
- **F3**: non-success states map to typed `{code, message}` bodies
  (`CONFLICT`/`CHECKPOINT`/`EXECUTOR_UNAVAILABLE`) matching the OpenAPI
  409/410/503 `MaisterErrorBody` entries.
- **Tests (regression guards)**: recover route integration asserts the typed
  `code` per error state; cron route integration asserts the flat 200 DTO and
  adds a 207 case for caught per-row failures; the **e2e `(d)` assertion was
  corrected** from the stale `{workspace, revision}` shape to the flat contract
  (a sibling that would have failed otherwise); a **non-recoverable Crashed run
  fixture** was added to the e2e seed plus e2e `(a3)` proving Discard is visible
  and Recover is hidden.

## Prevention

- When a route has a published OpenAPI schema, the handler MUST return that
  schema's shape — re-project internal types onto the DTO at the boundary, never
  leak the internal shape. Treat the spec as the source of truth (docs/CLAUDE.md
  R3).
- A test that asserts a response shape MUST assert the **contract** shape, not
  whatever the code currently emits — otherwise the test cements the drift. When
  changing a DTO, grep for every test asserting the old keys (here: e2e `(d)`
  asserting `workspace`/`revision`) and update them in the same pass.
- A UI affordance with a different invariant than its sibling (Discard vs
  Recover) MUST be gated by its own condition, not piggy-backed on the sibling's
  gate. Seed the *negative* case (non-recoverable Crashed) so the affordance is
  actually exercised.
- A status code that a monitor reads (cron 200 vs 207) MUST reflect CAUGHT
  partial failures, not only thrown exceptions.
