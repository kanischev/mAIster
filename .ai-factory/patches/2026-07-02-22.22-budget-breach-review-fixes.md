# Budget breach review fixes

- Composite HITL recovery checks must run before normal run-state availability
  checks. Otherwise staged retries after terminalization or preservation can be
  blocked by the very state transitions they need to recover from.
- Preserve-before-terminal composites should avoid generic stop dispatchers.
  Use defensive session checkpointing, preserve the work product, then archive
  or terminalize so retries stay answerable.
- Contract DTOs need one canonical shape across read models, OpenAPI, UI, and
  docs. Progress DTO keys and legacy request payloads are compatibility
  surfaces, not internal implementation details.
- Route and integration tests must exercise real preflight and lifecycle
  boundaries for restart and park; stubs should not encode behavior that the
  production workbench service does not have.
