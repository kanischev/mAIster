# Agent L3 enforcement keyed off the catalog index, not the run's effective workspace

**Date:** 2026-06-13 16:55
**Files:** web/lib/db/schema.ts, web/lib/db/migrations/0052_yummy_saracen.sql, web/lib/agents/launch.ts, web/lib/agents/__tests__/dirty-watchdog.integration.test.ts, web/lib/__tests__/reconcile-sweep.integration.test.ts, supervisor/test/fixtures/mock-acp-compatibility.mjs, supervisor/src/__tests__/adapter-compatibility.integration.test.ts, docs/system-analytics/social-board.md
**Severity:** high

## Problem

Three findings from a pre-merge `/aif-verify` on the M34 platform-agent branch:

- **(M1, correctness)** `finalizeAgentRun` gated the L3 dirty-watchdog AND the
  ephemeral `-ro` worktree cleanup on `loadAgentWorkspaceContext` → `agents.workspace`,
  which is the catalog INDEX (a projection of the package's NEWEST installed
  revision). Launch, L1 `readOnlySession`, L2 materialization, and the session
  cwd all used the project's PINNED (effective) definition instead. When a
  package changed an agent's `workspace:` axis across versions and a project
  stayed pinned to the older `repo_read` revision, the terminal block
  `if (wsCtx?.workspace === "repo_read")` was skipped — so a repo_read run that
  dirtied the parent checkout was NOT quarantined, and its ephemeral checkout
  leaked (no GC backstop for `-ro` dirs).
- **(M2, docs drift)** `social-board.md` still asserted the `agent` social actor
  was "never written today / (Implemented)" and its activity-event-kind list
  omitted `triage_set` / `triage_requeued` / `agent_quarantined` — both false as
  of M34, which writes the agent actor via ephemeral agent tokens.
- **(M3, missing promised tests)** Two load-bearing paths the plan named as
  deliverables had no test: the reconcile no-worktree agent classification
  (`reconcile.ts:483-486`) and the supervisor `readOnlySession` wire-level
  auto-deny round-trip (only the pure decision fn was covered).

## Root Cause

`finalizeAgentRun` re-derived the run's workspace axis from a MUTABLE catalog
projection at terminal time, rather than acting on what the run actually
launched with. The catalog index and the project's pinned (effective)
definition are two different facts; enforcement must key off the latter, and
the latter is only knowable from the run itself once spawned. The docs drift was
carried forward from the Stage-1 (M31) baseline text when M34 turned the agent
actor on. The test gaps were fan-out coverage the implementation skipped.

## Solution

- Added `runs.agent_workspace` (migration `0052_yummy_saracen.sql`,
  `text` enum `none|repo_read|worktree`), set at the single agent-run insert in
  `launchAgentRun` from `ctx.effective.parsed.workspace`. `finalizeAgentRun`
  now gates on `row.agentWorkspace ?? wsCtx?.workspace` — the persisted value
  wins; the `??` keeps legacy/seeded rows working, so no test-seed churn.
- Corrected `social-board.md`: the agent actor goes live in M34 via
  `socialActorForToken`, and the activity-kind list now carries the three M34
  kinds.
- Added a divergence integration test (index says `worktree`, run says
  `repo_read` → still quarantined), a reconcile integration test (no-worktree
  agent run → candidate, skipped, NOT crashed; asserts `candidates >= 1` so it
  cannot pass vacuously), and a supervisor wire test that drives a real adapter
  with `readOnlySession: true` and asserts a write-class permission is
  auto-denied inline (`selected:deny`, no `session.permission_request` event,
  `pendingPermissions.size === 0`) and a read-class one auto-approved.

## Prevention

- Terminal/enforcement logic must act on what a run DID (snapshotted on the run
  row at spawn), never re-derive it from a mutable catalog/projection that can
  drift after launch. When a launch reads field X from an "effective"/pinned
  source, the terminal path must read the SAME source — persist it if the
  source is mutable.
- When a feature flips a dormant capability on (here: the polymorphic `agent`
  actor), grep the older-stage analytics for "never / not yet / MUST NOT" claims
  and status tags that the flip invalidates.
- A test that asserts "did NOT happen" (not crashed, not quarantined) must also
  assert the path was actually exercised (e.g. `candidates >= 1`) so it cannot
  pass vacuously.
