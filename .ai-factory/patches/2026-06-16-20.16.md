# Scratch `Stop & archive` combined op had no direct test (runKind-composition gap)

**Date:** 2026-06-16 20:16
**Files:** web/lib/workbench-lifecycle/__tests__/workbench-stop.integration.test.ts, web/lib/workbench-lifecycle/service.ts
**Severity:** low

## Problem

The combined `stopThenArchive` op ("Stop & archive") accepts both `flow` and
`scratch` runs and is reachable from the rail ⋯ action-sheet for either. But
no test exercised the **scratch** path end-to-end. Coverage was:

- `stopThenArchive` unit tests (`service.test.ts`) — all built a `flow` ctx.
- `stopScratchWorkbench` integration test — exercised the scratch stop
  primitive *alone*, never composed with the archive half.

So the scratch composition (scratch stop → reload → archive dispatch) shipped
proven only by reasoning, not by an executed test. `/aif-verify` reported "all
green" on plan-task + gate completion; the gap surfaced only under the
adversarial `/aif-review` lens.

## Root Cause

When a handler **dispatches on an enum** (`stopRunByKind` switches on
`runs.run_kind`), each arm is a distinct execution path. The test suite covered
the `flow` arm of the *combined* op and the `scratch` arm of the *primitive*,
and the composition "looked covered" because both halves were individually
green. Per-phase / per-unit greens hide the cross-arm composition: the scratch
arm of `stopThenArchive` reloads ctx via a run-kind-agnostic `loadContext`
(`service.ts` `loadLifecycleContext`) and relies on `policy.ts` gating worktree
actions by **status** (`WORKTREE_ACTION_STATUSES`), not run kind — a real
invariant that nothing asserted.

## Solution

Added an integration test seeding a live scratch run **with** a workspace and
calling `stopThenArchive(runId, { deps })`. Wiring that makes the hybrid work:

- `depsFromOptions` is all-or-nothing (`options?.deps ?? defaults`), so the test
  passes a **full** `WorkbenchLifecycleDeps`.
- The scratch arm calls `stopScratchWorkbench` via dynamic import → it hits the
  **real** container DB through the already-mocked `getDb` (not injected deps),
  so the stop→`Review` transition is exercised for real.
- The injected `loadContext` runs **real** `pool.query`s (so the post-stop
  reload genuinely sees `Review`), while the git/claim ops
  (`preserveWorktree` / `recordArchive` / `claim`+`finalizeLifecycleOperation`)
  are spies — the seeded `/tmp` paths are not real git repos.
- Asserts: real-DB `runs.status` + `scratch_runs.dialog_status` = `Review`;
  `preserveWorktree` + `recordArchive` called; `claimLifecycleOperation` called
  with `operation: "archive"`; result carries `archivedBranch` +
  `supervisorStopped: false`.

Also (secondary, observability): `stopAgentAfterAuth` now captures
`finalizeAgentRun`'s `{ finalized }` and `log.info`s when it was a no-op (a lost
live→terminal CAS race) — the outcome is still correct (`Abandoned`), but the
no-op is no longer silent.

## Prevention

- **Test every arm of an enum-dispatched handler's composition, not just one
  arm + the shared primitive.** When a combined/aggregating op fans out on
  `run_kind` / a type discriminant, each arm that reaches a different downstream
  (here: scratch-stop-then-archive vs flow-stop-then-archive) needs its own
  end-to-end assertion. "Half A is tested + half B is tested" ≠ "A∘B is tested".
- For hybrid integration tests that must touch real DB state but stub
  side-effecting deps: drive the real transition through the un-injected path
  (dynamic-imported service + mocked `getDb`) and back `loadContext` with real
  queries so reloads observe the committed state; spy only the git/FS/claim
  boundary.
- Reviewer checklist: for a new `settings.runner`-style or run-kind allow-list,
  grep the test dir for each allow-listed value and confirm a test names it.
