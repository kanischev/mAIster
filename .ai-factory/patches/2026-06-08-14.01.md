# Analytics-doc drift in a freshly-authored doc: acp-runners.md misdescribed the create-response shape and the dup-id mechanism

**Date:** 2026-06-08 14:01
**Files:** docs/system-analytics/acp-runners.md, docs/decisions.md, supervisor/src/spawn.ts, web/components/scratch/scratch-dialog.tsx, .ai-factory/plans/feature-scratch-workspace-dialogs.md
**Severity:** low

## Problem

Two docs authored in the same branch as the code they describe contradicted that code,
while tagging themselves "Implemented":

- `docs/system-analytics/acp-runners.md` (new this branch) showed the create flow as
  `R->>DB: SELECT id (duplicate check)` then `R-->>M: 201 { runner }`. The code
  (`web/app/api/admin/acp-runners/route.ts` POST) does NOT pre-`SELECT`; it returns
  `{ ok, id }` and the OpenAPI explicitly does NOT echo the row. The Edge-cases entry
  likewise said the dup-id 409 was "enumerated by pre-insert existence check".
- `docs/decisions.md` "TODO (tracked doc defects)" claimed the `web.openapi.yaml`
  GET/POST/PATCH acp-runner shapes were still drifted and "left for a dedicated
  contract-sync pass" — but the same branch's `web.openapi.yaml` had already synced
  them (`{ok,id}` / `{ok}` / `defaultRunnerId`+`adapters`+`sidecars`). The TODO was
  self-contradicting.

Plus three `--resume` comment stragglers left after the `--resume`→`session/resume`
rename, all in files this branch otherwise rewrote: `spawn.ts` (said "via session/load",
also the wrong protocol term), `scratch-dialog.tsx`, and the scratch backlog plan.

## Root Cause

The docs were written from the spec / an earlier draft and trusted their own
"Implemented" status tag, rather than being re-derived from the route handler at HEAD.
Two mechanism decisions changed *during* implementation and did not propagate to every
surface:

1. Dup-id handling moved from a read-then-write `SELECT` to a race-safe
   `onConflictDoNothing().returning()` (the team even added a two-racer testcontainer
   test for it) — but `acp-runners.md` still depicted the rejected `SELECT` design.
2. Resume moved from a `--resume` CLI flag to the ACP `session/resume` protocol call —
   the rename hit ~20 doc/CLAUDE/OpenAPI surfaces but missed three in-branch code
   comments / a backlog bullet.

The `decisions.md` TODO described a drift that a later commit in the same branch had
already closed; "tracked defect" notes were not re-checked against the branch's own
final state.

## Solution

- Re-derived the `acp-runners.md` create sequence from `route.ts`: readiness computed
  before the insert, `INSERT ... onConflictDoNothing().returning({ id })`, empty
  `returning()` → 409, success → `201 { ok, id }`. Rewrote the Edge-cases dup-id entry
  to name the race-safe insert and why a pre-`SELECT` is wrong (concurrent POSTs → raw
  `23505` → 500).
- Converted the stale `decisions.md` TODO into a `(resolved 2026-06-08)` entry matching
  the local "(resolved)" convention, recording that the OpenAPI sync already shipped on
  this branch.
- Fixed `session/load`→`session/resume` in `spawn.ts` and the two `--resume` comment
  stragglers (`scratch-dialog.tsx`, scratch plan). Left the correct "adapters ignore
  `--resume` on argv" reference intact.

Verified with `pnpm validate:docs` (35/35 mermaid blocks, 221 ADR anchors) and a
straggler re-grep.

## Prevention

- When authoring or landing a `system-analytics/*.md` doc, reconcile EACH sequence
  step and response shape against the route handler at HEAD — never trust the doc's own
  "Implemented" tag (skill-context rule #7). A `201 { runner }` claim must be checked
  against the actual `NextResponse.json(...)` return.
- A mechanism rename (e.g. `--resume`→`session/resume`) must `grep` ALL surfaces,
  including code comments, backlog plans, and pseudocode — not just docs + CLAUDE.md.
  A single sweep over `web/components`, `supervisor/src`, and `.ai-factory/plans` would
  have caught the three stragglers.
- Before adding a "tracked doc defect" TODO, confirm the defect isn't already fixed
  elsewhere in the same change; re-check resolved TODOs when the change that fixes them
  lands.
