## Key decisions (baked, do not re-derive)

1. **Field name `packages[]`** in `maister.yaml` (user-renamed from `flow_packages[]`): `{id, source, version, path?}`. Additive — `flows[]`/`capability_imports[]` untouched.
2. **New module dir `web/lib/packages/`** (`manifest.ts`, `install.ts`, P2: `catalog.ts`, `attach.ts`) — keeps `flows.ts` focused.
3. **Revision inheritance**: `installFlowPlugin`/`installRevision` (web/lib/flows.ts:999/594) and `installCapabilityRevision` (web/lib/capabilities/import.ts:399) gain optional `resolvedRevisionOverride`. Package install computes ONE revision — git: `rev-parse HEAD` of the tag clone; local: `localDirectoryContentDigest(pkgRoot)` (flows.ts:407) — and passes it to every sub-install. Fixes the capability local-rev asymmetry (today: hash of source STRING, import.ts:58–65) and keeps `runs.flow_revision` pinning immutable for git packages.
4. **P0 test fixtures = v1-verbatim snapshot** at `web/test-fixtures/aif-flows/` (the 5 flow dirs + schemas only, ~30 KB; NO capability bundle; provenance README pointing at maister-plugins). Tests exercising engine behavior keep passing unchanged; the package contract guard for v2 lives in maister-plugins (P3 brief). No assertion churn.
5. **Discovery** = `git ls-remote --tags <url>` (filter `<name>/v*`) + depth-1 default-branch clone to tmp for `packages/*/maister-package.yaml` scan; result cached as jsonb on the source row + `last_checked_at`; per-source failure degrades to stale cache + WARN, never blocks the page. NO new scheduler job kind (avoids the 4-point kind-registration fan-out) — startup debounce in `web/instrumentation.ts` (staleness > `MAISTER_PACKAGE_DISCOVERY_STALE_HOURS`, env int hours, default 24, zod-parsed once) + manual refresh button.
6. **Attach atomicity** (multi-store rule): install/intent rows happen BEFORE (idempotent, content-addressed); the attach itself = ONE `db.transaction` writing flows rows + capability rows + `project_package_attachments` + FK group links; `maister.yaml` write-back happens AFTER commit (fs side-effect, best-effort: failure → WARN + UI notice, DB stays truth). Crash windows: (a) install done / attach tx not committed → revision orphan, harmless, GC'able; (b) tx committed / yaml not written → stale yaml, healed by next attach/detach/upgrade write-back or manual edit. Detach = symmetric single tx behind a live-run guard (PRECONDITION if any non-terminal run pins a member revision).
7. **Package-level trust**: one operator decision per package revision; the same tx fans `trust_status`/`exec_trust` onto member `flow_revisions` + `capability_imports` rows. `setup.sh` NEVER runs at install — only via the existing `runRevisionSetup`/`trustExecutable` path after the trust flip (physical fetch≠execute separation preserved, flows.ts:727, import.ts:394).
8. **Typed ingestion on attach**: package manifest inventory (skills/agents walked at install into `package_installs.manifest`); `mcps[]` templates → project MCP catalog entries flagged `package-provided` (env values `env:NAME` refs only), removed on detach; `restrictions[]` (manifest v1 section `{id, paths[]}`) → `capability_records` kind=`restriction` with `material.paths` per attached project (unlocks `must_not_touch` → aif v2.1.0, out of this plan).
9. **yaml write-back is comment-preserving**: parse with `yaml` Document API, mutate only `packages[]`, serialize, `atomicWriteText` (web/lib/atomic.ts). The dirty file is the user's to commit.
10. **Local package versions are first-class**: install from a local dir → version label `local-<digest12>`, catalog-marked `local`, attachable like any version. UI local-path attach is `manageCatalog`/admin-gated; file:// has no path allowlist today (existing posture for flows[]) — documented boundary, not widened silently.
11. **Numbers**: ADR-**087**, migration **0047** (main = f34769fd owns ADR-086 + 0046). T0.1 rebases first so numbering is taken against live main; later sibling merges may force the standard renumber dance (journal `when` + snapshot transplant gotchas).
12. **Identifiers** (trust-boundary rule) for every new route: `slug` = url-param → project row; `sourceId`/`packageInstallId` = body ids resolved to server rows (404 on miss, project-scoped joins); `localPath` (admin attach-local only) = body-controlled abs path, zod abs-path shape + role gate, used only as fs read root for install (same trust class as registration's `path`). NO body-controlled value ever composes a cache/symlink path — those derive from server-validated `flowRefId`/`resolvedRevision` as today.
13. **Deployment wiring = exactly one env var** (owner decision 2026-06-12): `MAISTER_PACKAGE_DISCOVERY_STALE_HOURS` (default 24) → `.env.example` committed default + web service `environment:` block in `compose.yml` (+ `compose.production.yml` passthrough if needed; the repo has NO `compose.override.yml`) + `docs/configuration.md` env table — dedicated task T3.11 per skill-context rule. No new ports, sidecars, binaries, or webhook event types (package lifecycle events = future work).
