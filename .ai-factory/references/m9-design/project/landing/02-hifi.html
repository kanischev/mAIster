§ Product spine

Backlog to merge.
Humans, agents, and the path in between.

The spine isn't a pipeline — it's a configurable surface. Each step accepts a human or an agent (or both). HITL is decided per flow, per task, not at one fixed point. Solo today; teams and full agentization next.

§ Four ways to collaborate

Humans, agents,
and every combination of both.

A control plane that treats agents as first-class teammates, not a service queue. Every interaction has a clear surface, an artifact, and a queue.

↔

Human ↔ Human

Team backlog. Peer review on a diff. Hand-off comments on a HITL ticket. Tickets queue across projects.

→

Human → Agent

Launch a task with prompt + flow + executor. Approve a permission. Send corrections back to the work loop.

→

Agent → Human

NeedsInput artifacts. Schema-driven forms in the run page. Inbox badge across projects. Keep-alive + checkpoint/resume.

↔

Agent ↔ Agent Phase 2

Agent-to-agent hand-off inside a flow. Sub-agents spawned per step. Reviewer / log / dependency watchers.

§ Nine pieces of glue

Connective tissue for the tools you already use.

No new IDE. No new agent. No new task tracker. MAIster is the layer between them.

Multi-project portfolio
One grid of every active workspace across every registered project. Filter by status. "Needs you (N)" badge counts pending HITL across all projects, all flows.

vendor-neutral · ACP 0.22.1
Configurable HITL
Per-step, per-flow, per-task. Schema-driven forms, not chat. Keep-alive + checkpoint/resume so an active review never times out mid-thought.
30-min keep-alive · 24h TTL
Diff review & merge
Raw git diff, accept / request-changes, --no-ff to main. Conflicts abort to manual resolve. No auto-clobber.
POC · ADR-012
Live SSE everything
Every agent stdout line streams over SSE, piped to disk in parallel. Last-Event-ID reconnect works without replaying from RAM. Per-run cost accounting in cost.jsonl.
ADR-007 · bounded memory
Project-tightened agents runtime
Each project gets its own agent runtime — skills, MCPs, rules and tools pinned per repo. Agents don't drift across projects.
